# TASK-0004 — Implement Project Types

## Goal

Implement the shared domain model for Atlas Projects.

A Project represents a complete translation project within a Workspace. It contains pages, assets, language configuration, project metadata, and project settings.

This task defines only the data structures.

No runtime logic, validation, serialization, or file operations should be implemented.

---

# Background

A Workspace can contain multiple Projects.

A Project acts as the root object for all translation work, including pages, assets, glossary references, and export configuration.

Projects are stored using the Atlas Document format.

# Project

The Project interface represents a complete translation project.

Suggested properties:

- id
- workspaceId
- name
- description
- version
- metadata
- settings
- language
- status
- assets
- pageIds
- createdAt
- updatedAt

Do not include actual Page objects.

Use identifiers only.

---

# ProjectMetadata

Contains descriptive information.

Suggested fields:

- author
- publisher
- series
- volume
- chapter
- tags
- notes

All metadata should be optional unless required by the application.

---

# ProjectSettings

Contains project-level preferences.

Suggested fields:

- defaultFontFamily
- defaultFontSize
- defaultAlignment
- defaultBubbleMode
- defaultTranslationEngine
- defaultOCRProvider
- enableTranslationMemory
- enableAutoSave

This task only defines the structure.

---

# ProjectLanguage

Represents language configuration.

Suggested fields:

- sourceLanguage
- targetLanguage

Both fields should use the shared LanguageCode type.

---

# ProjectStatus

Represents the lifecycle of a project.

Suggested values:

- Draft
- Ready
- Translating
- Reviewing
- Completed
- Archived

Use a TypeScript enum or string literal union according to the project's coding standard.

---

# ProjectAsset

Represents external resources used by the project.

Suggested fields:

- id
- type
- name
- path
- mimeType
- size
- checksum

Assets may include:

- Images
- Fonts
- Dictionaries
- Templates
- Export files
