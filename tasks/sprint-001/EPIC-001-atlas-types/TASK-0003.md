# TASK-0003 — Implement Workspace Types

## Goal

Implement the domain types related to the Atlas Workspace.

A Workspace is the highest-level container in Atlas Studio.

It contains projects, shared resources, user preferences, and workspace metadata.

This task defines only the data model.

No business logic or validation should be implemented.

---

# Background

The Workspace is the entry point of every Atlas Studio session.

It represents a collection of projects and shared resources.

Every project belongs to exactly one workspace.

---

# Scope

Included:

- Workspace interface
- Workspace metadata
- Workspace settings
- Recent project references
- Shared resources

Excluded:

- Project implementation
- Page implementation
- File loading
- Validation
- Business logic

---

# Deliverables

Create the following structure.

```text
packages/

atlas-types/

src/

workspace/

    Workspace.ts
    WorkspaceMetadata.ts
    WorkspaceSettings.ts
    RecentProject.ts

index.ts
```

---

# Workspace

Create the main Workspace interface.

Properties should include:

- id
- name
- description
- version
- createdAt
- updatedAt
- settings
- metadata
- recentProjects

Do not include project objects yet.

Use project identifiers only.

---

# WorkspaceMetadata

Contains descriptive information.

Suggested fields:

- author
- organization
- tags
- notes

Metadata should remain optional where appropriate.

---

# WorkspaceSettings

Contains workspace-level preferences.

Examples:

- defaultLanguage
- targetLanguage
- autosave
- theme
- defaultFontFamily
- defaultFontSize

Only define the structure.

No default values.

---

# RecentProject

Represents a recently opened project.

Suggested fields:

- projectId
- projectName
- lastOpened

---

# Public API

Export every workspace type from:

```text
src/index.ts
```

# Technical Requirements

The implementation must:

- Use existing common types.
- Reuse UUID.
- Reuse Timestamp.
- Reuse LanguageCode.
- Avoid duplicated types.
- Introduce no runtime code.
