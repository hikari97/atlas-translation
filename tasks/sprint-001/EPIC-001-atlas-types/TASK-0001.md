# TASK-0001 — Initialize atlas-types Package

## Goal

Create the initial structure for the `atlas-types` package.

This task establishes the package foundation but does **not** implement any domain models.

---

# Background

`atlas-types` is the foundational package for the entire Atlas Studio ecosystem.

Every package, application, plugin, and service depends on the types defined here.

Because of this, the package must remain lightweight, dependency-free, and framework-agnostic.

---

# Scope

Included:

- Package initialization
- Folder structure
- package.json
- tsconfig.json
- tsup configuration
- ESLint configuration (if applicable)
- src/index.ts
- README.md

Excluded:

- Interfaces
- Enums
- Domain Models
- Validation
- Business Logic

---

# Deliverables

The following structure should exist after this task:

```text
packages/

atlas-types/

README.md
package.json
tsconfig.json
tsup.config.ts

src/
    index.ts

tests/
```

---

# Requirements

The package must:

- Compile successfully.
- Export an empty public API.
- Use TypeScript strict mode.
- Produce ESM and CommonJS builds.
- Generate declaration files.
- Be independent from React.
- Be independent from Node.js APIs.
- Have zero runtime dependencies.

---

# Technical Requirements

TypeScript:

- strict = true
- declaration = true
- declarationMap = true

Build:

- tsup

Output:

```text
dist/

index.js
index.cjs
index.d.ts
```
