# TASK-0002 — Implement Common Types

## Goal

Implement the foundational shared types used across the entire Atlas Studio ecosystem.

These types represent generic concepts that are not tied to any specific domain model.

---

# Background

Before implementing domain-specific models such as Workspace, Project, or Bubble, Atlas Studio requires a set of reusable primitive types.

These types will be referenced throughout every package and application.

---

# Scope

Included:

- Primitive aliases
- Geometry types
- Common utility types
- Language types
- Timestamp types

Excluded:

- Workspace
- Project
- Page
- Bubble
- Layer
- Translation
- Style
- Plugin

---

# Deliverables

Create the following directory structure.

```text
packages/

atlas-types/

src/

    UUID.ts
    Timestamp.ts

    LanguageCode.ts

geometry/

    Point.ts

```

---

# Requirements

Implement the following reusable types.

## UUID

Represents a unique identifier.

Example:

```ts
export type UUID = string;
```

---

## Identifier

Generic identifier.

Used for:

- Workspace ID
- Project ID
- Bubble ID
- Plugin ID

---

## Timestamp

Represents an ISO-8601 timestamp.

Example:

```ts
export type Timestamp = string;
```

---

## LanguageCode

Represents a supported language.

Examples:

- en
- id
- ja
- ko
- zh-CN
- zh-TW

Do not hardcode every language yet.

Use:

```ts
export type LanguageCode = string;
```

---

## Point

Represents a coordinate.

```ts
export interface Point {
  x: number;

  y: number;
}
```

---

## Size

Represents width and height.

```ts
export interface Size {
  width: number;

  height: number;
}
```

---

## Rectangle

Represents a rectangular area.

```ts
export interface Rectangle {
  x: number;

  y: number;

  width: number;

  height: number;
}
```

---

## Polygon

Represents polygon coordinates.

```ts
export interface Polygon {
  points: Point[];
}
```

---

# Public API

Export every type from:

```text
src/index.ts
```

Example:

```ts
export * from './common/UUID';

export * from './geometry/Point';
```

---

# Technical Requirements

The package must:

- Use strict TypeScript.
- Avoid runtime code.
- Export types only.
- Remain dependency-free.

---

# Acceptance Criteria

The task is complete when:

- All common types compile successfully.
- All exports are available through `src/index.ts`.
- No circular imports exist.
- No runtime dependencies are introduced.
