# TASK-0004 — Implement Shared Enums

## Summary

Implement the shared enumerations used throughout Atlas Studio.

These enums define common states, statuses, modes, and categories shared across domain models.

No business logic should be implemented.

---

# Goal

Provide a centralized collection of reusable enums that ensure consistency across every Atlas Studio package.

---

# Background

Many domain models require standardized values.

Examples include:

- Project status
- Bubble type
- Layer type
- Export format
- OCR direction
- Translation status

Without shared enums, different packages may introduce inconsistent values.

---

# Scope

## Included

- Common enums
- Workflow enums
- Editor enums
- Export enums
- Translation enums
- OCR enums
- Status enums
- Barrel exports

---

## Excluded

- Validation

# Technical Requirements

- Use native TypeScript `enum` or `const enum` only where appropriate.
- Prefer string values for serialization.
- Every enum must have a descriptive name.
- Export all enums through `enums/index.ts`.
- Export the enums module from the package root.
