# TASK-0008 — Implement Layer Domain

## Summary

Implement the complete Layer domain model.

A Layer represents an editable element within a Page. Layers provide the foundation for organizing visual content, text, masks, annotations, and future editor features.

This task defines the data model only.

No rendering or editing logic should be implemented.

---

# Goal

Create a reusable, framework-independent Layer domain that supports non-destructive editing throughout Atlas Studio.

---

# Background

Each Page may contain multiple Layers.

Examples include:

- Original Image Layer
- Clean Image Layer
- Bubble Layer
- Text Layer
- Mask Layer
- Annotation Layer
- Guide Layer

Layers provide the structure required for future editing, rendering, export, and history tracking.

# Domain Responsibilities

The Layer domain describes:

- Layer identity
- Layer ordering
- Layer visibility
- Layer locking
- Layer opacity
- Layer transform
- Layer metadata
- References to related objects

The Layer domain must not contain implementation logic.

---

# Technical Requirements

The implementation must:

- Use interfaces only.
- Prefer readonly properties.
- Reuse Common Types.
- Reuse Geometry Types.
- Reuse Shared Enums.
- Avoid runtime dependencies.
- Export all models through `layer/index.ts`.
- Export the Layer module from the package root.
