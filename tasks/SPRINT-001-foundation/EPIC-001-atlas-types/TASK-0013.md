# TASK-0013 — Implement Asset Domain

## Summary

Implement the complete Asset domain model.

The Asset domain represents all reusable resources managed by Atlas Studio, including images, fonts, AI models, templates, brushes, and future resource types.

Only data models should be implemented.

No storage, caching, downloading, uploading, or file system logic should be implemented.

---

# Goal

Create a reusable, framework-independent Asset domain that standardizes how resources are described throughout Atlas Studio.

---

# Background

Assets are shared resources used by Projects, Pages, and the Editor.

Examples include:

- Images
- Fonts
- Brush presets
- Templates
- AI models
- Export presets
- User resources

The Asset domain describes the metadata and relationships of these resources without defining how they are stored or loaded.

# Domain Responsibilities

The Asset domain describes:

- Asset identity
- Asset metadata
- Resource location
- Resource origin
- Version information
- Integrity information
- Permissions
- Tags
- References to related objects

The Asset domain must remain implementation-independent.

---

# Technical Requirements

The implementation must:

- Use interfaces only.
- Prefer readonly properties.
- Reuse Common Types.
- Reuse Shared Enums.
- Avoid runtime dependencies.
- Export every model through `asset/index.ts`.
- Export the Asset module from the package root.
