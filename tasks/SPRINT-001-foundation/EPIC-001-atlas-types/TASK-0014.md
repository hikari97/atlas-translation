# TASK-0014 — Implement Image Domain

## Summary

Implement the complete Image domain model.

The Image domain describes image resources used throughout Atlas Studio, including original pages, processed images, thumbnails, previews, exported images, and intermediate rendering outputs.

Only data models should be implemented.

No image processing, rendering, decoding, encoding, or file management logic should be implemented.

---

# Goal

Create a reusable and framework-independent Image domain that standardizes image metadata across the Atlas Studio platform.

---

# Background

Images are fundamental resources in Atlas Studio.

Every imported manga, comic, or webtoon page is represented by an Image.

The Image domain should support:

- Original images
- Processed images
- Cleaned images
- Preview images
- Thumbnail images
- Exported images
- Temporary rendering images

The domain represents image metadata only.

# Domain Responsibilities

The Image domain describes:

- Image identity
- Image dimensions
- Resolution
- Aspect ratio
- File format
- Color profile
- Thumbnail information
- Image metadata
- Image statistics
- References to related assets

The Image domain must not implement image manipulation.

---

# Technical Requirements

The implementation must:

- Use interfaces only.
- Prefer readonly properties.
- Reuse Common Types.
- Reuse Geometry Types.
- Reuse Asset Domain.
- Reuse Shared Enums.
- Avoid runtime dependencies.
- Export every model through `image/index.ts`.
- Export the Image module from the package root.
