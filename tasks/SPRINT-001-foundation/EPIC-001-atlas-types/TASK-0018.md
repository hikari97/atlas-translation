# TASK-0018 — Implement Editor Domain

## Summary

Implement the complete Editor domain model.

The Editor domain defines the state of the Atlas Studio editor, including active tools, viewport configuration, interaction modes, editing preferences, and editor session information.

Only data models should be implemented.

No rendering, user interaction handling, command execution, or UI logic should be implemented.

---

# Goal

Create a reusable and framework-independent Editor domain that provides a consistent editor state across all Atlas Studio applications.

---

# Background

The Editor is the central workspace where users interact with Pages, Layers, Bubbles, and Translations.

The Editor domain should support:

- Tool management
- Viewport state
- Zoom
- Pan
- Active page
- Active layer
- Active bubble
- Editor preferences
- Session state

The domain represents editor state only.

# Domain Responsibilities

The Editor domain describes:

- Editor identity
- Current editor state
- Active tool
- Active page
- Active layer
- Active bubble
- Viewport configuration
- Zoom level
- Pan position
- Editor preferences
- References to related objects

The Editor domain must not contain rendering or interaction logic.

---

# Technical Requirements

The implementation must:

- Use interfaces only.
- Prefer readonly properties.
- Reuse Common Types.
- Reuse Geometry Types.
- Reuse Shared Enums.
- Reuse Workspace, Project, Page, Layer, Bubble, and Translation references where appropriate.
- Avoid runtime dependencies.
- Export every model through `editor/index.ts`.
- Export the Editor module from the package root.
