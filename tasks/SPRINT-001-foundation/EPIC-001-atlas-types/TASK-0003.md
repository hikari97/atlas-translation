# TASK-0003 — Implement Geometry Types

## Summary

Implement the shared geometry models used throughout Atlas Studio.

Geometry types define positions, dimensions, bounding boxes, polygons, and transformations used by the editor, OCR, rendering, and image processing systems.

This package contains **only data models**.

No geometry calculations or algorithms should be implemented.

---

# Goal

Provide a complete set of reusable geometry types that can be shared across every Atlas Studio package.

These models must remain immutable, lightweight, framework-independent, and free of runtime dependencies.

---

# Background

Almost every feature in Atlas Studio depends on geometry.

Examples include:

- Bubble Detection
- OCR Regions
- Text Placement
- Image Crop
- Canvas Selection
- Layer Positioning
- Export Regions

Centralizing geometry models ensures consistency throughout the platform.

---

# Scope

## Included

- Point
- Size
- Rectangle
- Bounding Box
- Polygon
- Circle
- Line
- Matrix
- Transform
- Vector
- Geometry barrel exports

---

## Excluded

- Geometry calculations
- Collision detection
- Rotation algorithms
- Matrix multiplication
- Image processing
- Rendering logic

These belong to future packages.

# Technical Requirements

The implementation must:

- Use interfaces for domain models.
- Use readonly properties whenever appropriate.
- Avoid classes.
- Avoid methods.
- Avoid business logic.
- Export every type through `geometry/index.ts`.
- Export the geometry module through the package root.
