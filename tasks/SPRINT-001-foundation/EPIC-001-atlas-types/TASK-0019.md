# TASK-0019 — Implement Selection Domain

## Summary

Implement the complete Selection domain model.

The Selection domain defines how entities are selected within Atlas Studio. It provides a generic model for selecting Pages, Layers, Bubbles, Images, Assets, and future editable objects.

Only data models should be implemented.

No hit-testing, mouse interaction, keyboard shortcuts, or editor behavior should be implemented.

---

# Goal

Create a reusable and framework-independent Selection domain that provides a consistent representation of selected objects across the Atlas Studio platform.

---

# Background

Selection is used by nearly every editor feature.

Examples include:

- Selecting a Bubble
- Selecting multiple Layers
- Selecting an Image Region
- Selecting text objects
- AI-selected objects
- Plugin-selected objects

The Selection domain must support both single and multi-selection while remaining implementation-independent.

# Domain Responsibilities

The Selection domain describes:

- Selection identity
- Selected objects
- Single selection
- Multi-selection
- Selection groups
- Selection bounds
- Selection filters
- References to selected entities

The Selection domain must not implement interaction behavior.

---

# Technical Requirements

The implementation must:

- Use interfaces only.
- Prefer readonly properties.
- Reuse Common Types.
- Reuse Geometry Types.
- Reuse Shared Enums.
- Reuse references from Workspace, Project, Page, Layer, Bubble, Asset, and Image where appropriate.
- Avoid runtime dependencies.
- Export every model through `selection/index.ts`.
- Export the Selection module from the package root.
