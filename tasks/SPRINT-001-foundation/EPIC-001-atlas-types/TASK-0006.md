# TASK-0006 — Implement Project Domain

## Summary

Implement the complete Project domain model.

A Project represents a translation project inside a Workspace. It contains pages, translation settings, metadata, statistics, and project-specific resources.

This task defines the data model only.

---

# Goal

Create a complete, reusable, and framework-independent Project domain model that can be shared across every Atlas Studio package.

---

# Background

A Project is the primary working unit in Atlas Studio.

Every imported manga, comic, or webtoon chapter exists as a Project.

A Project manages:

- Pages
- Assets
- Translation settings
- Metadata
- Statistics
- Export configuration

Future packages such as `atlas-document`, `atlas-core`, and the frontend editor will depend on these definitions.

---

## Included

---

# Domain Responsibilities

The Project domain describes:

- Project identity
- Display information
- Language configuration
- Translation settings
- Export preferences
- Statistics
- References to pages

The Project domain must not contain behavior or implementation logic.

---

# Technical Requirements

The implementation must:

- Use interfaces for domain models.
- Prefer readonly properties.
- Reuse common types.
- Reuse shared enums.
- Avoid runtime dependencies.
- Export all models through `project/index.ts`.
- Export the project module from the package root.
