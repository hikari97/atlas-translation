# TASK-0002 — Implement Common Types

## Summary

Implement the shared primitive types and utility types used throughout Atlas Studio.

These types will become the foundation for every domain model and must remain stable, lightweight, and framework-independent.

---

# Goal

Create a reusable collection of common types that can be imported by every package without introducing runtime dependencies.

---

# Background

Every domain model (Workspace, Project, Page, Bubble, etc.) requires shared primitive types such as identifiers, timestamps, nullable values, and utility generics.

Centralizing these definitions avoids duplication and ensures consistency across the platform.

---

# Scope

## Included

- Primitive type aliases
- Identifier types
- Timestamp types
- Nullable utilities
- Optional utilities
- Generic helper types
- Barrel exports

---

## Excluded

- Geometry
- Workspace
- Project
- Page
- Bubble
- Translation
- Enums

These belong to future tasks.

---

# Technical Requirements

The implementation must:

- Have zero runtime dependencies.
- Use TypeScript type aliases where appropriate.
- Use interfaces only when extensibility is required.
- Avoid classes.
- Export every public type through `common/index.ts`.
- Export `common/index.ts` through the package root.

---

# Public API

The package should expose:

```ts
export * from './common';
```

The exact implementation may evolve, but the public API must remain clean and stable.
