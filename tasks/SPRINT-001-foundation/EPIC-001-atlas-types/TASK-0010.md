# TASK-0010 — Implement Translation Domain

## Summary

Implement the complete Translation domain model.

The Translation domain represents every translated text associated with a Bubble, including the original text, translated text, review status, AI provider information, glossary references, and translation metadata.

Only data models should be implemented.

No translation engine, AI logic, prompt generation, or business logic should be implemented.

---

# Goal

Create a reusable and provider-independent Translation domain that can be shared across every Atlas Studio package.

---

# Background

The Translation domain is one of the core domains of Atlas Studio.

Every detected Bubble may contain one or more translations during its lifecycle.

The Translation domain must support:

- Machine Translation
- Human Translation
- Review Workflow
- Translation History
- Multiple AI Providers
- Future Translation Memory integration

The Translation domain must remain implementation-independent.

# Domain Responsibilities

The Translation domain describes:

- Translation identity
- Original text
- Translated text
- Source language
- Target language
- Translation provider
- Review status
- Confidence score
- Metadata
- References to Bubble and Project

The Translation domain must never implement behavior.

---

# Technical Requirements

The implementation must:

- Use interfaces only.
- Prefer readonly properties.
- Reuse Common Types.
- Reuse Shared Enums.
- Avoid runtime dependencies.
- Export every model through `translation/index.ts`.
- Export the Translation module from the package root.
