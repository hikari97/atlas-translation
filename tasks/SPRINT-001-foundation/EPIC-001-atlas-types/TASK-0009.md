# TASK-0009 — Implement Bubble Domain

## Summary

Implement the complete Bubble domain model.

The Bubble domain represents speech bubbles, narration boxes, thought bubbles, sound effects, and other editable text regions detected on a comic page.

This task defines the complete Bubble data model only.

No OCR, AI, rendering, or editing logic should be implemented.

---

# Goal

Create a reusable and extensible Bubble domain model that can be shared across every Atlas Studio package.

The model must support both automatic AI detection and manual editing.

---

# Background

The Bubble domain is the central component of Atlas Studio.

Nearly every workflow depends on Bubble objects:

- Bubble Detection
- OCR
- Translation
- Image Cleaning
- Typography
- Rendering
- Export

Because of this, the Bubble domain must remain stable, implementation-independent, and reusable.

# Domain Responsibilities

The Bubble domain describes:

- Bubble identity
- Bubble geometry
- Bubble category
- OCR result
- Translation result
- Typography settings
- Confidence score
- Metadata
- Relationships

The Bubble domain must never implement behavior.

---

# Technical Requirements

The implementation must:

- Use interfaces only.
- Prefer readonly properties.
- Reuse Common Types.
- Reuse Geometry Types.
- Reuse Shared Enums.
- Avoid runtime dependencies.
- Export every model through `bubble/index.ts`.
- Export the Bubble module from the package root.
