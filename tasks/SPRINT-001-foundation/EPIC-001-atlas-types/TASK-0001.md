# TASK-0001 — Initialize atlas-types Package

## Summary

Initialize the `atlas-types` package as the foundational package of Atlas Studio.

This task creates the package structure, build configuration, TypeScript configuration, and public API entry point.

No domain models or business logic should be implemented.

---

# Goal

Create a production-ready TypeScript package that will serve as the shared type library for the entire Atlas Studio platform.

The package must compile successfully and expose an empty but functional public API.

---

# Background

`atlas-types` is the lowest-level package in the Atlas Studio dependency graph.

Every other package depends on it.

Because of this, the package must remain lightweight, framework-independent, and free from runtime dependencies.

This task establishes the package foundation before any domain models are introduced.

---

# Scope

## Included

- Package initialization
- Folder structure
- TypeScript configuration
- Build configuration
- Package metadata
- Public API entry point
- Development tooling configuration

---

## Excluded

- Workspace types
- Project types
- Page types
- Bubble types
- Translation types
- Enums
- Geometry
- Validation
- Business logic
- Tests

These will be implemented in future tasks.

# Technical Requirements

The package must:

- Use TypeScript Strict Mode.
- Support ESM output.
- Support CommonJS output.
- Generate declaration files (`.d.ts`).
- Have zero runtime dependencies.
- Be framework-independent.
- Be compatible with Node.js LTS.
