# TASK-0021 — Implement Export Domain

## Summary

Implement the complete Export domain model.

The Export domain defines reusable models describing export requests, export options, export outputs, metadata, and supported export targets throughout Atlas Studio.

Only data models should be implemented.

No export engine, file generation, compression, rendering, or storage logic should be implemented.

---

# Goal

Create a reusable and framework-independent Export domain that standardizes export operations across all Atlas Studio packages.

---

# Background

Atlas Studio supports exporting projects into multiple formats.

Examples include:

- PNG
- JPEG
- WEBP
- PDF
- CBZ
- Atlas Project Package

The Export domain should describe **what** will be exported, **how** it should be exported, and **what** the resulting artifact represents.

# Domain Responsibilities

The Export domain describes:

- Export identity
- Export target
- Export format
- Export configuration
- Export metadata
- Generated artifact
- Export result
- References to exported resources

The Export domain must not implement export behavior.

---

# Technical Requirements

The implementation must:

- Use interfaces only.
- Prefer readonly properties.
- Reuse Common Types.
- Reuse Asset Domain where appropriate.
- Reuse Image Domain where appropriate.
- Reuse Shared Enums.
- Avoid runtime dependencies.
- Export every model through `export/index.ts`.
- Export the Export module from the package root.
