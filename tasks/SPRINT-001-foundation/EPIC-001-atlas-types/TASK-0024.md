Validate the overall quality of the `atlas-types` package.

# Validation Checklist

Verify the following:

## Build

- Package builds successfully.
- Declaration files are generated.
- No build warnings.
- No TypeScript errors.

---

## Public API

Verify:

- Every domain is exported.
- No internal modules are exposed.
- Import paths are stable.
- Subpath exports resolve correctly.

---

## Type Safety

Verify:

- Strict mode passes.
- No implicit any.
- No duplicate declarations.
- No conflicting type names.

---

## Architecture

Verify:

- No circular dependencies.
- Domain boundaries are respected.
- No runtime dependencies introduced.
- Package structure matches the architecture documentation.

---

## Compatibility

Verify imports such as:

```ts id="cvtqwp"
import type { Bubble } from '@atlas/types';

import type { Project } from '@atlas/types/workspace';

import type { Canvas } from '@atlas/types/editor';

import type { Asset } from '@atlas/types/resource';

import type { WorkflowDefinition } from '@atlas/types/platform';
```

Every import should resolve correctly.

# Technical Requirements

The validation process must:

- Use TypeScript strict mode.
- Validate generated declaration files.
- Verify all barrel exports.
- Confirm package exports are consistent.
