# TASK-0011 — Implement Typography Domain

## Summary

Implement the complete Typography domain model.

The Typography domain defines all text appearance and layout properties used throughout Atlas Studio. It provides a reusable model for rendering translated text consistently across different editors, rendering engines, and export targets.

Only data models should be implemented.

No text rendering, font loading, layout calculation, or editor behavior should be implemented.

---

# Goal

Create a reusable and framework-independent Typography domain that standardizes text appearance throughout Atlas Studio.

---

# Background

Typography is responsible for how translated text is displayed.

The Typography domain should support:

- Comic dialogue
- Narration boxes
- Sound effects (SFX)
- Titles
- Captions
- Future vertical and right-to-left layouts

Typography should be reusable by Bubble, Editor, Export, and Rendering packages.

# Domain Responsibilities

The Typography domain describes:

- Text appearance
- Text alignment
- Text spacing
- Text rotation
- Stroke
- Shadow
- Decorations
- Transform
- References to related objects

The Typography domain must not implement rendering behavior.

---

# Technical Requirements

The implementation must:

- Use interfaces only.
- Prefer readonly properties.
- Reuse Common Types.
- Reuse Geometry Types where appropriate.
- Reuse Shared Enums.
- Avoid runtime dependencies.
- Export every model through `typography/index.ts`.
- Export the Typography module from the package root.
