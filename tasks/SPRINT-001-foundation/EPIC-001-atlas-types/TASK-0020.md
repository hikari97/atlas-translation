# TASK-0020 — Implement Canvas Domain

## Summary

Implement the complete Canvas domain model.

The Canvas domain defines the editable workspace used throughout Atlas Studio. It describes the coordinate system, viewport, workspace boundaries, grid configuration, guides, snapping options, and visual overlays.

Only data models should be implemented.

No rendering, drawing, interaction handling, or layout calculations should be implemented.

---

# Goal

Create a reusable and framework-independent Canvas domain that provides a consistent workspace model for all Atlas Studio applications.

---

# Background

Every editing operation occurs inside a Canvas.

The Canvas domain should support:

- Workspace boundaries
- Coordinate systems
- Zoom
- Pan
- Grid
- Guides
- Snapping
- Rulers
- Safe areas
- Overlay configuration

The Canvas domain defines workspace metadata only.

# Domain Responsibilities

The Canvas domain describes:

- Canvas identity
- Coordinate system
- Workspace boundaries
- Zoom level
- Pan offset
- Viewport configuration
- Grid settings
- Guide lines
- Ruler settings
- Overlay configuration
- Snapping configuration

The Canvas domain must not implement rendering or interaction behavior.

---

# Technical Requirements

The implementation must:

- Use interfaces only.
- Prefer readonly properties.
- Reuse Common Types.
- Reuse Geometry Types.
- Reuse Shared Enums.
- Reuse Editor references where appropriate.
- Avoid runtime dependencies.
- Export every model through `canvas/index.ts`.
- Export the Canvas module from the package root.
