# TASK-0013 — Implement Document Diff Framework

## Summary

Implement the document diff framework for Atlas Studio.

The framework compares two document snapshots and produces a structured change set describing additions, removals, updates, and moves.

Only the diff framework should be implemented.

No merge engine, conflict resolution, undo/redo logic, or synchronization should be implemented.

---

# Goal

Create a reusable framework capable of comparing document snapshots in a consistent and deterministic way.

The resulting change set will be consumed by other packages such as history, synchronization, collaboration, and undo/redo.

---

# Background

Several Atlas Studio features require comparing document states.

Examples include:

- Undo / Redo
- History
- Autosave
- Collaboration
- Synchronization
- Version comparison

Rather than requiring every subsystem to implement its own comparison logic, the document package should expose a common diff framework.

# Responsibilities

The Document Diff Framework is responsible for:

- Comparing two document snapshots.
- Detecting created nodes.
- Detecting removed nodes.
- Detecting updated nodes.
- Detecting moved nodes.
- Producing deterministic change sets.

The framework is **not** responsible for:

- Applying changes.
- Conflict resolution.
- History management.
- Synchronization.
- Business logic.

# Technical Requirements

- Use strict TypeScript.
