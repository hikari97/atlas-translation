# TASK-0008 — Implement Subscription Index

## Summary

Implement the Subscription Index responsible for optimizing subscription lookup within the Atlas Studio Event Framework.

The Subscription Index maintains read-optimized indexes over the immutable Subscription Registry.

It accelerates event publication without owning subscription lifecycle.

---

# Goal

Provide efficient lookup structures that enable fast subscription discovery for large numbers of registered subscriptions.

The Subscription Index is an internal optimization layer and is not part of the public Event API.

---

# Background

The Subscription Registry stores all subscriptions.

Scanning the entire registry for every published event does not scale.

The Subscription Index maintains secondary indexes that allow rapid retrieval of candidate subscriptions.

The index is rebuilt or updated whenever the Subscription Registry changes.

# Responsibilities

The Subscription Index is responsible for:

- Building indexes from registered subscriptions.
- Providing efficient lookup structures.
- Supporting multiple indexing strategies.
- Keeping indexes synchronized with the Subscription Registry.

The Subscription Index is NOT responsible for:

- Registering subscriptions.
- Matching subscriptions.
- Executing subscribers.
- Event publication.
- Business logic.

---

# Index Model

```text
Subscription Registry

↓

Subscription Index

├── Event Type Index
├── Priority Index
├── Namespace Index

↓

Subscription Matcher
```

Indexes reference Subscription objects rather than duplicating them.

---

# Public API

Expected exports

```ts
SubscriptionIndex;

DefaultSubscriptionIndex;

SubscriptionIndexBuilder;
```

Although exported, the index is intended primarily for framework internals.

---

# Design Principles

The Subscription Index should be:

- Read optimized
- Deterministic
- Replaceable
- Framework independent
- Easy to rebuild

Indexes should contain references only and never own Subscription instances.

---

# Technical Requirements

The implementation must:

- Depend only on:

```
@atlas/types

@atlas/document

@atlas/command
```

- Use TypeScript strict mode.
- Avoid runtime dependencies.
- Avoid circular dependencies.
- Support incremental updates.
- Support complete index rebuilds.

---
