# TASK-0017 — Implement Event Journal

## Summary

Implement the Event Journal responsible for recording published events within the Atlas Studio Event Framework.

The Event Journal provides an append-only record of publication activity.

It is intended for diagnostics, replay preparation, auditing, and observability.

The Event Journal is **not** the source of truth for application state.

---

# Goal

Provide a reusable append-only journal that records publication metadata and event payloads without affecting event execution.

---

# Background

The Atlas Studio Event Framework supports optional event recording.

Recorded events may later be used for:

- Replay
- Diagnostics
- Debugging
- Audit
- Metrics
- Tracing

The journal records publication history only.

# Responsibilities

## EventJournal

Responsible for:

- Recording publication history.
- Reading journal entries.
- Enumerating entries.
- Providing append-only semantics.

Not responsible for:

- Event publication.
- Event replay.
- Subscriber execution.
- Business logic.
- Domain persistence.

---

## JournalWriter

Responsible for:

- Appending new journal entries.
- Preserving ordering.
- Guaranteeing append-only behavior.

---

## JournalReader

Responsible for:

- Reading journal entries.
- Filtering entries.
- Enumerating recorded events.

# Design Principles

The Event Journal should be:

- Append-only
- Immutable
- Deterministic
- Replaceable
- Framework independent

Journal entries must never be modified after they are recorded.

---

# Technical Requirements

The implementation must:

Depend only on:

```
@atlas/types

@atlas/document

@atlas/command
```

- Use TypeScript strict mode.
- Avoid runtime dependencies.
- Avoid circular dependencies.
- Support future persistent repositories.
- Preserve publication order.

---

# Supported Operations

The journal must support:

- Append
- Read
- Enumerate
- Filter
- Count
- Clear (development/testing only)

Future extensions may include:

- File-backed repository
- Database repository
- Cloud repository
- Distributed journal
