# TASK-0007 — Implement Subscription Registry

## Summary

Implement the Subscription Registry responsible for managing all event subscriptions within the Atlas Studio Event Framework.

The Subscription Registry is the authoritative source of registered subscriptions.

It is responsible only for registration and lookup.

It does not perform subscription matching or subscriber execution.

---

# Goal

Provide a reusable registry capable of storing, retrieving, and managing event subscriptions.

The registry serves as the foundation for the Subscription Matcher and Event Bus.

---

# Background

Subscribers are never registered directly with the Event Bus.

Instead, subscribers are wrapped in immutable Subscription objects and stored inside the Subscription Registry.

The Event Bus consults the registry to obtain candidate subscriptions for a published event.

# Responsibilities

The Subscription Registry is responsible for:

- Registering subscriptions.
- Removing subscriptions.
- Retrieving subscriptions.
- Enumerating registered subscriptions.
- Preventing duplicate registrations.

The Subscription Registry is NOT responsible for:

- Matching subscriptions.
- Executing subscribers.
- Event publication.
- Pipeline execution.
- Replay.
- Retry.

---

# Registry Model

```text
Subscription

↓

Subscription Registry

↓

Lookup

↓

Subscription Collection
```

The registry stores immutable Subscription objects.

# Design Principles

The Subscription Registry should be:

- Strongly typed
- Deterministic
- Framework independent
- Replaceable
- Testable

Subscriptions remain immutable after registration.

---

# Technical Requirements

The implementation must:

- Depend only on:

```
@atlas/types

@atlas/document

@atlas/command
```

- Use TypeScript strict mode.
- Avoid runtime dependencies.
- Avoid circular dependencies.
- Support multiple subscriptions for the same event.
- Prevent duplicate subscription identifiers.

---

# Registry Operations

Supported operations include:

- Register
- Unregister
- Get by identifier
- List all subscriptions
- Clear registry

Lookup returns registered subscriptions without filtering or prioritization.
