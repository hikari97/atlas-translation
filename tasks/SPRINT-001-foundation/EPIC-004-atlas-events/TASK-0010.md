# TASK-0010 — Implement Event Bus

## Summary

Implement the Event Bus for the Atlas Studio Event Framework.

The Event Bus is the only public entry point responsible for publishing events.

The Event Bus delegates all orchestration to the internal Event Publication Coordinator.

It should remain a thin façade.

---

# Goal

Provide a stable, minimal, and extensible Event Bus API that allows publishers to publish events without knowing how event delivery is implemented.

---

# Background

The Event Bus separates publishers from subscribers.

Publishers should never interact directly with:

- Subscription Registry
- Subscription Matcher
- Event Pipeline
- Replay
- Metrics
- Tracing

The Event Bus hides all implementation details.

---

# Responsibilities

## EventBus

Responsible for:

- Receiving published events.
- Creating publication context.
- Delegating publication.
- Returning EventPublicationResult.

Not responsible for:

- Matching subscriptions.
- Executing subscribers.
- Recording history.
- Metrics.
- Replay.

---

## EventPublicationCoordinator

Responsible for:

- Orchestrating publication.
- Invoking Subscription Matcher.
- Executing Event Pipeline.
- Coordinating publication lifecycle.
- Returning publication results.

Not responsible for:

- Public API.
- Subscription registration.
- Business logic.

# Public API

Expected exports

```ts
EventBus;

DefaultEventBus;
```

Internal exports

```ts
EventPublicationCoordinator;

DefaultEventPublicationCoordinator;
```

The coordinator is intended for framework composition and testing, not as the primary API for consumers.

---

# Design Principles

The Event Bus should be:

- Minimal
- Stateless
- Strongly typed
- Framework independent
- Easy to replace

The coordinator should encapsulate orchestration logic.

---

# Technical Requirements

The implementation must:

Depend only on:

```
@atlas/types

@atlas/document

@atlas/command
```

- Use TypeScript strict mode.
- Avoid runtime dependencies.
- Avoid circular dependencies.
- Support synchronous and asynchronous publication.
