# TASK-0003 — Define Event Context

## Summary

Define the Event Context used throughout the Atlas Studio Event Framework.

The Event Context encapsulates all information required during the lifecycle of an event publication.

It serves as the shared execution context for the Event Bus, Pipeline, Behaviors, and Subscribers.

This task defines contracts only.

No runtime implementation should be introduced.

---

# Goal

Provide a strongly typed execution context that enables event publication, tracing, replay, and future extensibility.

---

# Background

Publishing an event requires more than the event payload itself.

The framework needs contextual information such as:

- Correlation identifiers
- Causation identifiers
- Publisher information
- Execution metadata
- Shared services

This information must travel together during event processing.

# Responsibilities

The Event Context is responsible for:

- Carrying the Event instance.
- Providing execution metadata.
- Preserving tracing information.
- Sharing contextual information across the pipeline.

The Event Context is NOT responsible for:

- Publishing events.
- Executing subscribers.
- Storing events.
- Business logic.

---

# Context Model

Every Event Context should contain:

- Event
- CorrelationId
- CausationId
- PublisherId
- Timestamp
- Metadata
- Optional services

# Design Principles

The Event Context should be:

- Immutable where possible.
- Strongly typed.
- Serializable.
- Extensible.
- Framework independent.

It should evolve without breaking existing contracts.
