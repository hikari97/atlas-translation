# TASK-0002 — Define Event Contracts

## Summary

Define the core Event contracts for the Atlas Studio Event Framework.

Events represent immutable facts that have already occurred.

They are published through the Event Bus and consumed by one or more Event Subscribers.

This task defines contracts only.

No runtime implementation should be introduced.

---

# Goal

Provide strongly typed event contracts that establish the foundation for the Event Framework.

These contracts become the public API for every package that publishes or subscribes to events.

---

# Background

Unlike Commands, Events are notifications.

Commands ask the system to perform work.

Events announce that work has already happened.

One Event may be consumed by many independent subscribers.

# Responsibilities

The Event contract is responsible for:

- Representing immutable events.
- Providing event identity.
- Providing metadata.
- Providing event version information.
- Providing event timestamps.

The Event contract is NOT responsible for:

- Event publication.
- Event subscription.
- Event dispatch.
- Event persistence.
- Business logic.

---

# Event Model

Every Event should contain:

- Event identifier
- Event name
- Event version
- Timestamp
- Metadata
- Payload

Events must be immutable after creation.

# Design Principles

Events should be:

- Immutable
- Serializable
- Strongly typed
- Versioned
- Framework independent

Events represent facts, not requests.
