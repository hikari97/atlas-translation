# TASK-0025 — Build Unit Test Suite

## Summary

Create a comprehensive unit test suite for the `atlas-command` package.

Unit tests must validate the behavior of every public component in isolation.

Tests should verify contracts, edge cases, and failure scenarios without depending on external packages or runtime infrastructure.

---

# Goal

Provide a reliable automated test suite that guarantees the correctness of the Command Framework's core components.

---

# Background

The `atlas-command` package contains multiple independent components:

- Command
- Handler
- Command Bus
- Resolver
- Dispatcher
- Pipeline
- Pipeline Behaviors
- Transactions
- History
- Undo / Redo
- Composite Commands
- Macro Commands

Each component must be tested independently before integration testing begins.

# Testing Principles

Every test should follow:

```text
Arrange

↓

Act

↓

Assert
```

or

```text
Given

↓

When

↓

Then
```

Each test should verify one behavior only.

---

# Technical Requirements

Tests must:

- Run deterministically.
- Be isolated.
- Avoid shared state.
- Use mocks instead of real implementations.
- Execute without network or filesystem dependencies.
- Run under TypeScript strict mode.
