# TASK-0009 — Implement Command Resolver

## Summary

Implement the Command Resolver responsible for locating the appropriate handler descriptor for a command.

The resolver bridges the gap between the Command Registry and the Handler Registry.

It performs lookup only.

It never creates handlers.

It never executes handlers.

---

# Goal

Create a reusable resolver that maps command types to registered handler descriptors.

The resolver becomes the foundation for the Command Dispatcher.

---

# Background

Commands describe intentions.

Handlers perform work.

Registries store metadata.

The resolver connects these pieces.

Applications and the Command Bus should not perform handler lookup directly.

# Responsibilities

The Command Resolver is responsible for:

- Resolving handler descriptors from commands.
- Querying the Handler Registry.
- Returning deterministic results.
- Producing meaningful resolution errors.

The resolver is NOT responsible for:

- Creating handlers.
- Executing handlers.
- Running middleware.
- Managing transactions.
- Publishing events.

---

# Resolution Flow

```text
Command

↓

Command Type

↓

Handler Registry

↓

Handler Descriptor
```

The resolver stops here.

# Design Principles

The resolver should be:

- Stateless
- Deterministic
- Strongly typed
- Fast
- Framework independent

The resolver should never cache handler instances.
