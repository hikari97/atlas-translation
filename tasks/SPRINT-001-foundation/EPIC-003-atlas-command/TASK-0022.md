# TASK-0022 — Implement Macro Commands

## Summary

Implement Macro Commands that encapsulate a reusable sequence of commands.

A Macro Command extends the Composite Command model by supporting recording, persistence, and replay.

Macro Commands enable users, plugins, and automation systems to execute predefined command sequences as a single logical operation.

---

# Goal

Provide a reusable automation mechanism that integrates seamlessly with the Atlas Studio Command Framework.

Macro Commands should support:

- Recording
- Serialization
- Replay
- Persistence
- Plugin extensibility

without changing the execution pipeline.

---

# Background

Many workflows consist of repeating the same sequence of commands.

Examples include:

- Applying a document template
- Bulk formatting
- Batch editing
- Import automation
- Plugin-defined workflows

Rather than recreating these sequences manually, Atlas Studio allows them to be recorded as Macro Commands.

# Responsibilities

Macro Commands are responsible for:

- Recording command sequences.
- Storing execution metadata.
- Replaying recorded command sequences.
- Supporting serialization.

Macro Commands are NOT responsible for:

- Executing commands directly.
- Validation.
- Transactions.
- Event publication.
- Business logic.

Execution remains the responsibility of the Command Framework.

---

# Execution Model

```text
MacroCommand

↓

CompositeCommand

↓

Command A

↓

Command B

↓

Command C
```

Macro Commands behave exactly like Composite Commands during execution.

The additional capabilities are:

- Recording
- Replay
- Serialization

# Design Principles

Macro Commands should be:

- Immutable after recording.
- Serializable.
- Replayable.
- Strongly typed.
- Framework independent.

A Macro Command may contain Composite Commands.

---

# Technical Requirements

The implementation must:

- Depend only on:

```
@atlas/types

@atlas/document
```

- Use TypeScript strict mode.
- Avoid runtime dependencies.
- Avoid circular dependencies.
- Preserve command ordering.
- Support serialization and deserialization contracts.
