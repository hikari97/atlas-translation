# TASK-0003 — Define Command Context

## Summary

Define the `CommandContext` contract used throughout the Atlas Studio Command Framework.

The context provides runtime information required during command execution while remaining independent of command execution logic.

This task defines only the contract.

No runtime implementation is introduced.

---

# Goal

Create a strongly typed execution context that becomes the shared contract between:

- Command Bus
- Command Handlers
- Middleware
- Transactions
- Future extensions

---

# Background

Every command execution requires access to runtime information.

Examples include:

- Current document
- Current workspace
- Current project
- Current user
- Cancellation state
- Execution metadata

Rather than passing numerous parameters between components, Atlas Studio standardizes them within a single `CommandContext`.

This improves consistency and extensibility.

# Responsibilities

The CommandContext contract is responsible for:

- Representing runtime execution state.
- Providing shared execution information.
- Supporting future extensibility.
- Remaining implementation-independent.

The context is **not** responsible for:

- Executing commands.
- Resolving dependencies.
- Managing transactions.
- Publishing events.
- Performing validation.

---

# Design Principles

The context should be:

- Immutable by default.
- Lightweight.
- Framework-independent.
- Serializable when appropriate.
- Strongly typed.

The context should avoid exposing implementation details.
