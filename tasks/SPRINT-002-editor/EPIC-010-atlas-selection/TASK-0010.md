# TASK-0010 — Implement Selection Validator

## Summary

Implement the `SelectionValidator`, a reusable abstraction responsible for validating selection requests before they are applied within Atlas Studio.

The Selection Validator evaluates whether a proposed selection operation is permitted according to configurable validation rules. It supports item validation, selection limits, type restrictions, custom validation providers, and future extensible validation policies.

The Selection Validator is responsible only for validating selection requests. It does not modify selection state, render visual feedback, execute commands, or implement business logic.

---

# Goal

Provide a reusable, extensible, and framework-independent selection validation abstraction capable of validating selection operations through a stable public API.

---

# Background

Not every object should always be selectable.

Typical validation rules include:

- Locked objects
- Hidden objects
- Disabled objects
- Read-only items
- Maximum selection limits
- Type restrictions
- Plugin-defined validation rules

Instead of embedding validation logic throughout the editor, Atlas Studio should expose a unified Selection Validator abstraction.

# Responsibilities

The Selection Validator is responsible for:

- Validating selection requests.
- Applying validation rules.
- Returning immutable validation results.
- Supporting custom validation providers.
- Preserving deterministic validation behavior.

The Selection Validator is **not** responsible for:

- Updating selection state.
- Rendering selection overlays.
- Executing commands.
- Managing selection history.
- Business logic.
