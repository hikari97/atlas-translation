# TASK-0008 — Implement Toggle Selection

## Summary

Implement the `ToggleSelection`, a reusable abstraction responsible for toggling the selection state of individual items within Atlas Studio.

The Toggle Selection abstraction provides a consistent mechanism for adding an unselected item to the current selection or removing an already selected item without affecting other selected items. It serves as the foundation for Ctrl+Click, Command+Click, touch-assisted selection, and plugin-defined toggle workflows.

The Toggle Selection abstraction is responsible only for representing toggle selection behavior. It does not render selection visuals, modify documents, execute commands, or implement business logic.

---

# Goal

Provide a reusable, extensible, and framework-independent toggle selection abstraction capable of switching selection state through a stable public API.

---

# Background

Modern editors commonly support toggling individual items inside an existing selection.

Typical examples include:

- Ctrl + Click
- Command + Click
- Layer toggle selection
- Asset toggle selection
- Tree node toggle selection
- Plugin-defined toggle operations

Instead of requiring every subsystem to implement toggle logic independently, Atlas Studio should expose a unified Toggle Selection abstraction.

# Responsibilities

The Toggle Selection abstraction is responsible for:

- Toggling individual items.
- Preserving existing selections.
- Removing selected items.
- Adding unselected items.
- Providing immutable toggle metadata.

The Toggle Selection abstraction is **not** responsible for:

- Rendering selection overlays.
- Managing range selection.
- Updating documents.
- Executing commands.
- Business logic.
