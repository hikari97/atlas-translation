# TASK-0032 — Implement Screen Reader Metadata

## Summary

Implement the `ScreenReaderMetadata`, a reusable abstraction responsible for exposing selection information to assistive technologies within Atlas Studio.

The Screen Reader Metadata abstraction provides normalized accessibility metadata describing the current selection, navigation context, hierarchy position, and selection state. It enables screen readers and other assistive technologies to accurately communicate editor state without depending on rendering implementations.

The Screen Reader Metadata abstraction is responsible only for producing accessibility metadata. It does not render UI, control screen readers, execute commands, or implement business logic.

---

# Goal

Provide a reusable, extensible, and framework-independent metadata abstraction capable of exposing selection semantics to assistive technologies through a stable public API.

---

# Background

Screen readers require structured metadata rather than visual information.

Typical examples include:

- Current selected object
- Multiple selected objects
- Parent hierarchy information
- Position within a collection
- Group membership
- Selection count
- Plugin-defined accessibility metadata

Instead of requiring each package to generate accessibility information independently, Atlas Studio should expose a unified Screen Reader Metadata abstraction.

# Responsibilities

The Screen Reader Metadata abstraction is responsible for:

- Producing accessibility labels.
- Producing accessibility descriptions.
- Providing hierarchy metadata.
- Providing collection metadata.
- Supporting future accessibility providers.

The Screen Reader Metadata abstraction is **not** responsible for:

- Rendering accessibility overlays.
- Managing screen readers.
- Updating document state.
- Executing commands.
- Business logic.
