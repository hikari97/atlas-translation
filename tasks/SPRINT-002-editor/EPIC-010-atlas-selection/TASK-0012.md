# TASK-0012 — Implement Range Anchor

## Summary

Implement the `RangeAnchor`, a reusable abstraction responsible for managing the anchor point of a range selection within Atlas Studio.

The Range Anchor represents the fixed starting point from which contiguous selections are calculated. It provides a consistent reference for range-based interactions such as Shift+Click, Shift+Arrow, keyboard navigation, timeline selections, and hierarchical selection workflows.

The Range Anchor is responsible only for representing and managing the anchor state. It does not calculate ranges, render selection overlays, execute commands, or modify application state.

---

# Goal

Provide a reusable, extensible, and framework-independent Range Anchor abstraction capable of maintaining the reference point for range selections through a stable public API.

---

# Background

Nearly every editor that supports range selection relies on an anchor item.

Typical examples include:

- Shift + Click
- Shift + Arrow
- Layer panel selection
- File explorer selection
- Timeline selection
- Tree selection
- Plugin-defined range workflows

Without a dedicated anchor abstraction, each subsystem would need to manage anchor state independently.

# Responsibilities

The Range Anchor is responsible for:

- Maintaining the current anchor item.
- Updating anchor references.
- Clearing anchor state.
- Providing immutable anchor metadata.
- Supporting future anchor strategies.

The Range Anchor is **not** responsible for:

- Calculating selection ranges.
- Rendering selection overlays.
- Executing commands.
- Updating documents.
- Business logic.
