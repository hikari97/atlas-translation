# TASK-0029 — Implement Select All

## Summary

Implement the `SelectAll`, a reusable abstraction responsible for selecting every selectable entity within a defined scope inside Atlas Studio.

The Select All abstraction provides a deterministic mechanism for collecting every eligible selectable item within a container, document, viewport, hierarchy, or custom scope. It serves as the foundation for editor commands such as **Ctrl+A**, "Select All Layers", "Select All Objects", and plugin-defined bulk selection workflows.

The Select All abstraction is responsible only for determining the complete selection set. It does not render selection visuals, modify document content, execute commands, or implement business logic.

---

# Goal

Provide a reusable, extensible, and framework-independent Select All abstraction capable of selecting every eligible entity within a defined scope through a stable public API.

---

# Background

Selecting every available object is one of the most common editor operations.

Typical examples include:

- Ctrl + A
- Select all layers
- Select all objects
- Select all artboards
- Select all assets
- Select all nodes
- Plugin-defined select-all behavior

Instead of allowing each subsystem to implement its own bulk selection algorithm, Atlas Studio should expose a unified Select All abstraction.

# Responsibilities

The Select All abstraction is responsible for:

- Resolving selection scope.
- Discovering selectable entities.
- Producing immutable bulk selection results.
- Supporting custom scope providers.
- Preserving deterministic ordering.

The Select All abstraction is **not** responsible for:

- Rendering selection overlays.
- Executing editor commands.
- Updating documents.
- Managing history.
- Business logic.
