# TASK-0026 — Implement Focus Navigation

## Summary

Implement the `FocusNavigation`, a reusable abstraction responsible for navigating focus between selectable entities within Atlas Studio.

The Focus Navigation abstraction provides deterministic movement of the active focus independently of selection state. It supports keyboard navigation, accessibility workflows, hierarchical navigation, canvas navigation, and plugin-defined focus behaviors while maintaining immutable navigation state.

The Focus Navigation abstraction is responsible only for focus movement and focus state. It does not modify document contents, render focus indicators, execute commands, or implement business logic.

---

# Goal

Provide a reusable, extensible, and framework-independent focus navigation abstraction capable of moving focus predictably through selectable entities using a stable public API.

---

# Background

Modern editors distinguish between **focused items** and **selected items**.

Typical examples include:

- Arrow key navigation
- Tab navigation
- Layer panel navigation
- Asset browser navigation
- Tree navigation
- Canvas focus movement
- Plugin-defined focus behavior

Instead of allowing every subsystem to implement independent focus management, Atlas Studio should expose a unified Focus Navigation abstraction.

# Responsibilities

The Focus Navigation abstraction is responsible for:

- Moving focus.
- Tracking the active focused item.
- Preserving immutable focus metadata.
- Supporting custom navigation providers.
- Maintaining deterministic focus behavior.

The Focus Navigation abstraction is **not** responsible for:

- Rendering focus indicators.
- Selecting items.
- Updating documents.
- Executing commands.
- Business logic.
