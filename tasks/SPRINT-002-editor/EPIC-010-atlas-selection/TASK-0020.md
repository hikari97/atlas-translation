# TASK-0020 — Implement Marquee Auto Scroll

## Summary

Implement the `MarqueeAutoScroll`, a reusable abstraction responsible for calculating automatic scrolling behavior while performing marquee selection within Atlas Studio.

The Marquee Auto Scroll abstraction determines when automatic scrolling should occur as the pointer approaches the edge of a scrollable viewport. It calculates scrolling direction and velocity while remaining independent from rendering, viewport implementations, and document management.

The Marquee Auto Scroll abstraction is responsible only for calculating auto-scroll requests. It does not modify viewport position, render UI, execute commands, or implement business logic.

---

# Goal

Provide a reusable, extensible, and framework-independent auto-scroll abstraction capable of supporting continuous marquee selection beyond the visible viewport through a stable public API.

---

# Background

Modern editors automatically scroll when the pointer reaches the viewport edge during marquee selection.

Typical examples include:

- Canvas selection
- Layer panel selection
- Timeline selection
- Asset browser selection
- Node editor selection
- Spreadsheet selection
- Plugin-defined marquee workflows

Instead of allowing each subsystem to implement its own scrolling heuristics, Atlas Studio should expose a unified Marquee Auto Scroll abstraction.

# Responsibilities

The Marquee Auto Scroll abstraction is responsible for:

- Detecting viewport edge proximity.
- Calculating scroll direction.
- Calculating scroll velocity.
- Producing immutable scroll requests.
- Supporting future scrolling strategies.

The Marquee Auto Scroll abstraction is **not** responsible for:

- Moving the viewport.
- Rendering.
- Hit testing.
- Updating selection state.
- Business logic.
