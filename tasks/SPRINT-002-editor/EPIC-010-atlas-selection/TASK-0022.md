# TASK-0022 — Implement Child Selection

## Summary

Implement the `ChildSelection`, a reusable abstraction responsible for selecting immediate child entities within hierarchical document structures in Atlas Studio.

The Child Selection abstraction provides a deterministic mechanism for navigating from a parent entity to one or more direct children while preserving immutable selection state and hierarchical relationships. It can be reused by layers, groups, folders, scene graphs, artboards, component trees, and plugin-defined hierarchies.

The Child Selection abstraction is responsible only for resolving and selecting immediate child entities. It does not recursively traverse descendants, render selection overlays, execute commands, or modify application state.

---

# Goal

Provide a reusable, extensible, and framework-independent child selection abstraction capable of navigating hierarchical structures through a stable public API.

---

# Background

Hierarchical editors frequently require selecting child objects.

Typical examples include:

- Select first child
- Select direct child layer
- Enter group
- Enter component
- Navigate scene graph
- Folder navigation
- Plugin-defined hierarchy traversal

Instead of requiring each subsystem to implement child traversal independently, Atlas Studio should expose a unified Child Selection abstraction.

# Responsibilities

The Child Selection abstraction is responsible for:

- Resolving immediate child entities.
- Selecting direct children.
- Preserving hierarchy semantics.
- Producing immutable selection results.
- Supporting future hierarchy providers.

The Child Selection abstraction is **not** responsible for:

- Recursive traversal.
- Rendering selection overlays.
- Updating document structures.
- Executing commands.
- Business logic.
