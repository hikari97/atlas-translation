# TASK-0011 — Implement Range Selection

## Summary

Implement the `RangeSelection`, a reusable abstraction responsible for representing contiguous selections within Atlas Studio.

The Range Selection abstraction provides a deterministic mechanism for selecting a continuous range of items between an anchor item and an active item. It supports list views, tree views, layer panels, asset browsers, timelines, and plugin-defined selection workflows.

The Range Selection abstraction is responsible only for calculating contiguous selection ranges. It does not render selection overlays, modify documents, execute commands, or implement business logic.

---

# Goal

Provide a reusable, extensible, and framework-independent range selection abstraction capable of representing contiguous selections through a stable public API.

---

# Background

Range selection is one of the most common selection mechanisms in modern desktop applications.

Typical examples include:

- Shift + Click
- Shift + Arrow
- Layer range selection
- File explorer selection
- Timeline selection
- Tree view selection
- Plugin-defined range selection

Instead of requiring every subsystem to calculate contiguous ranges independently, Atlas Studio should expose a unified Range Selection abstraction.

# Responsibilities

The Range Selection abstraction is responsible for:

- Calculating contiguous selections.
- Tracking anchor and active items.
- Producing immutable range results.
- Supporting ordered collections.
- Supporting future range strategies.

The Range Selection abstraction is **not** responsible for:

- Rendering selection overlays.
- Updating documents.
- Executing commands.
- Managing selection history.
- Business logic.
