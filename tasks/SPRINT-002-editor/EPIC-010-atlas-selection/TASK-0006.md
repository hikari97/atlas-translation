# TASK-0006 — Implement Single Selection

## Summary

Implement the `SingleSelection`, a reusable abstraction responsible for representing and managing the selection of a single item within Atlas Studio.

The Single Selection abstraction provides a deterministic model for selecting exactly one item at a time. When a new item is selected, the previous selection is replaced automatically. It serves as the default selection strategy across editors, inspectors, trees, lists, layers, and plugin-defined components.

The Single Selection abstraction is responsible only for representing single-selection state. It does not render visual feedback, modify documents, execute commands, or implement business logic.

---

# Goal

Provide a reusable, extensible, and framework-independent single selection abstraction capable of representing one active selected item through a stable public API.

---

# Background

Single selection is the most common selection strategy used throughout graphical applications.

Typical examples include:

- Selecting one layer
- Selecting one object
- Selecting one node
- Selecting one artboard
- Selecting one asset
- Plugin-defined selections

Instead of allowing every subsystem to implement its own single-selection behavior, Atlas Studio should expose a unified Single Selection abstraction.

# Responsibilities

The Single Selection abstraction is responsible for:

- Representing one selected item.
- Replacing the current selection.
- Clearing the active selection.
- Providing immutable selection metadata.
- Supporting future selection extensions.

The Single Selection abstraction is **not** responsible for:

- Rendering selection overlays.
- Managing multiple selections.
- Updating documents.
- Executing commands.
- Business logic.
