# TASK-0002 — Implement Selection Session

## Summary

Implement the `SelectionSession`, a reusable abstraction responsible for representing the lifecycle of an active selection operation within Atlas Studio.

A Selection Session encapsulates a single selection interaction from initialization through completion or cancellation. It provides a consistent interaction model that can be reused by pointer selection, keyboard selection, marquee selection, programmatic selection, and plugin-defined selection workflows.

The Selection Session is responsible only for representing selection interaction state. It does not modify documents, render selection overlays, or execute editor commands.

---

# Goal

Provide a reusable, extensible, and framework-independent Selection Session abstraction capable of representing active selection interactions through a stable public API.

---

# Background

Every selection operation has a lifecycle.

Typical examples include:

- Mouse click selection
- Keyboard selection
- Touch selection
- Marquee selection
- Multi-selection
- Plugin-defined selection

Instead of allowing every subsystem to manage its own selection lifecycle, Atlas Studio should expose a unified Selection Session abstraction.

# Responsibilities

The Selection Session is responsible for:

- Representing an active selection operation.
- Tracking selection lifecycle.
- Recording session metadata.
- Providing immutable session information.
- Supporting future selection strategies.

The Selection Session is **not** responsible for:

- Rendering selection overlays.
- Managing selection collections.
- Updating documents.
- Executing commands.
- Business logic.
