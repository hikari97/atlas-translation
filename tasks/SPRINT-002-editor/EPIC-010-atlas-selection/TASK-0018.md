# TASK-0018 — Implement Marquee Hit Test

## Summary

Implement the `MarqueeHitTest`, a reusable abstraction responsible for determining which selectable items intersect with a marquee selection rectangle within Atlas Studio.

The Marquee Hit Test evaluates geometric intersections between a normalized marquee rectangle and selectable item bounds. It provides deterministic, framework-independent hit detection that can be reused by canvases, layer views, node editors, timelines, asset browsers, and plugin-defined selection workflows.

The Marquee Hit Test is responsible only for geometric intersection evaluation. It does not render selection overlays, modify selection state, execute commands, or implement business logic.

---

# Goal

Provide a reusable, extensible, and framework-independent hit testing abstraction capable of determining which selectable items intersect a marquee selection region through a stable public API.

---

# Background

Marquee selection depends on efficient geometric hit testing.

Typical examples include:

- Canvas object selection
- Layer thumbnail selection
- Node graph selection
- Timeline clip selection
- Asset browser selection
- Plugin-defined selectable objects

Rather than allowing each subsystem to implement its own hit testing algorithm, Atlas Studio should expose a unified Marquee Hit Test abstraction.

# Responsibilities

The Marquee Hit Test is responsible for:

- Detecting rectangle intersections.
- Detecting complete containment.
- Returning immutable hit results.
- Supporting future hit-testing strategies.
- Providing deterministic geometric evaluation.

The Marquee Hit Test is **not** responsible for:

- Rendering selection overlays.
- Updating selections.
- Executing commands.
- Managing sessions.
- Business logic.
