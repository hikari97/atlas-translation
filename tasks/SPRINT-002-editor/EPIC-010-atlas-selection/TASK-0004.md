# TASK-0004 — Implement Selection State

## Summary

Implement the `SelectionState`, a reusable abstraction responsible for representing the current state of the selection system within Atlas Studio.

The Selection State provides an immutable snapshot describing the overall selection status, including whether a selection exists, the current selection mode, selection count, timestamps, and additional metadata. It serves as the canonical state representation shared across the selection subsystem.

The Selection State is responsible only for describing selection state. It does not modify selections, perform rendering, execute commands, or implement business logic.

---

# Goal

Provide a reusable, extensible, and framework-independent Selection State abstraction capable of representing the current selection status through a stable public API.

---

# Background

Every selection system requires a centralized state representation.

Typical examples include:

- No active selection
- Single selection active
- Multiple selection active
- Updating selection
- Clearing selection
- Programmatic selection
- Plugin-defined selection state

Rather than allowing every subsystem to expose its own state representation, Atlas Studio should expose a unified Selection State abstraction.

# Responsibilities

The Selection State is responsible for:

- Representing current selection status.
- Exposing selection mode.
- Tracking selection count.
- Providing immutable state metadata.
- Supporting future state extensions.

The Selection State is **not** responsible for:

- Managing selections.
- Rendering overlays.
- Updating documents.
- Executing commands.
- Business logic.

- Immutable
- Strongly typed
- Lightweight
