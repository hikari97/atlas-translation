# TASK-0019 — Implement Marquee Filter

## Summary

Implement the `MarqueeFilter`, a reusable abstraction responsible for filtering marquee hit-test results before they become an active selection within Atlas Studio.

The Marquee Filter evaluates hit-test results against configurable filtering rules, allowing only eligible items to participate in the final selection. It supports visibility checks, lock state filtering, selectable flags, custom predicates, and plugin-defined filtering strategies.

The Marquee Filter is responsible only for filtering hit-test results. It does not perform hit testing, render selection overlays, modify documents, or execute application logic.

---

# Goal

Provide a reusable, extensible, and framework-independent filtering abstraction capable of producing valid marquee selection candidates through a stable public API.

---

# Background

Not every object intersecting a marquee rectangle should become selected.

Typical examples include:

- Hidden layers
- Locked objects
- Disabled nodes
- Non-selectable guides
- Helper objects
- Internal editor objects
- Plugin-defined filtering rules

Instead of embedding filtering logic into hit testing, Atlas Studio should expose a dedicated Marquee Filter abstraction.

# Responsibilities

The Marquee Filter is responsible for:

- Filtering hit-test results.
- Applying configurable filter rules.
- Returning immutable filtered results.
- Supporting custom filter providers.
- Preserving deterministic filtering behavior.

The Marquee Filter is **not** responsible for:

- Rendering.
- Hit testing.
- Updating selection state.
- Executing commands.
- Business logic.
