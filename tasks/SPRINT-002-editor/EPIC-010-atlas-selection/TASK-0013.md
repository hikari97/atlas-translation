# TASK-0013 — Implement Range Expansion

## Summary

Implement the `RangeExpansion`, a reusable abstraction responsible for expanding and reducing an existing range selection within Atlas Studio.

The Range Expansion abstraction extends an active range by updating its boundary while preserving the original anchor. It enables continuous range manipulation through pointer interactions, keyboard navigation, timeline editing, tree navigation, and plugin-defined workflows.

The Range Expansion abstraction is responsible only for expanding an existing range. It does not manage anchor creation, render selection overlays, execute commands, or modify application state.

---

# Goal

Provide a reusable, extensible, and framework-independent Range Expansion abstraction capable of extending contiguous selections through a stable public API.

---

# Background

Range selections are rarely static.

Typical examples include:

- Holding Shift while pressing Arrow keys
- Dragging across ordered lists
- Timeline range expansion
- Layer panel navigation
- Tree navigation
- Spreadsheet selection
- Plugin-defined expansion behaviors

Instead of recalculating range behavior independently across subsystems, Atlas Studio should expose a unified Range Expansion abstraction.

# Responsibilities

The Range Expansion abstraction is responsible for:

- Expanding existing ranges.
- Reducing existing ranges.
- Updating active boundaries.
- Producing immutable range results.
- Supporting future expansion strategies.

The Range Expansion abstraction is **not** responsible for:

- Creating anchors.
- Rendering selection overlays.
- Executing commands.
- Updating documents.
- Business logic.
