# TASK-0009 — Implement Invert Selection

## Summary

Implement the `InvertSelection`, a reusable abstraction responsible for inverting the current selection within Atlas Studio.

The Invert Selection abstraction provides a consistent mechanism for selecting every unselected item while deselecting every currently selected item inside a given selection scope. It serves as the foundation for advanced editing workflows involving layers, objects, assets, tree structures, and plugin-defined selection behaviors.

The Invert Selection abstraction is responsible only for calculating inverted selection state. It does not render selection overlays, modify documents, execute commands, or implement business logic.

---

# Goal

Provide a reusable, extensible, and framework-independent invert selection abstraction capable of producing inverted selections through a stable public API.

---

# Background

Many editors support an "Invert Selection" operation.

Typical examples include:

- Invert selected layers
- Invert selected objects
- Invert selected assets
- Invert selected tree nodes
- Invert selected artboards
- Plugin-defined selection inversion

Instead of allowing every subsystem to implement its own inversion algorithm, Atlas Studio should expose a unified Invert Selection abstraction.

# Responsibilities

The Invert Selection abstraction is responsible for:

- Producing inverted selections.
- Preserving selection scope.
- Returning immutable selection results.
- Supporting future inversion strategies.
- Providing selection metadata.

The Invert Selection abstraction is **not** responsible for:

- Rendering selection overlays.
- Updating documents.
- Executing commands.
- Managing selection history.
- Business logic.
