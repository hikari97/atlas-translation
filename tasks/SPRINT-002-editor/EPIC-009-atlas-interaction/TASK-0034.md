# TASK-0034 — Implement Rotate Gesture

## Summary

Implement the `RotateGesture`, a reusable abstraction responsible for representing rotation-based gesture interactions within Atlas Studio.

The Rotate Gesture detects angular changes between two or more active pointers and exposes normalized rotation information that can be consumed by higher-level systems such as editors, canvases, design tools, and plugins.

The Rotate Gesture is responsible only for describing rotation state and metadata. It does not rotate objects, update transformations, or perform rendering.

---

# Goal

Provide a reusable, extensible, and framework-independent rotation gesture abstraction capable of representing rotation gestures through a stable public API.

---

# Background

Rotation gestures are commonly used in design, illustration, and graphical editing applications.

Typical examples include:

- Rotating selected objects
- Rotating images
- Rotating vector elements
- Rotating artboards
- Two-finger rotation
- Stylus rotation
- Plugin-defined rotation gestures

Rather than allowing each subsystem to calculate rotation independently, Atlas Studio should expose a unified Rotate Gesture abstraction.

# Responsibilities

The Rotate Gesture is responsible for:

- Detecting rotation gestures.
- Calculating rotation angles.
- Calculating angle delta.
- Tracking gesture lifecycle.
- Providing immutable rotation metadata.

The Rotate Gesture is **not** responsible for:

- Rotating objects.
- Applying transformations.
- Rendering animations.
- Executing commands.
- Business logic.
