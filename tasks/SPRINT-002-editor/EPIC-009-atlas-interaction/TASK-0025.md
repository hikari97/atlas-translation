# TASK-0025 — Implement Auto Scroll

## Summary

Implement the `AutoScroll`, a reusable abstraction responsible for describing automatic scrolling behavior during active interaction sessions.

The Auto Scroll component enables continuous scrolling when the pointer approaches the edge of a scrollable viewport while dragging, selecting, resizing, or performing other pointer-based interactions.

The Auto Scroll abstraction is responsible only for calculating scrolling intent and metadata. It does not directly manipulate scroll containers or render visual feedback.

---

# Goal

Provide a reusable, extensible, and framework-independent auto scroll abstraction capable of describing automatic scrolling behavior through a stable public API.

---

# Background

Large editors frequently require automatic scrolling when the pointer reaches the viewport edge.

Typical examples include:

- Dragging objects beyond the visible area
- Extending text selections
- Moving canvas elements
- Reordering long lists
- Dragging tree nodes
- Plugin-defined interactions

Rather than embedding scrolling behavior into every interaction, Atlas Studio should expose a unified Auto Scroll abstraction.

# Responsibilities

The Auto Scroll component is responsible for:

- Detecting viewport edge proximity.
- Calculating scroll direction.
- Calculating scroll speed.
- Tracking scroll lifecycle.
- Providing immutable scroll metadata.

The Auto Scroll component is **not** responsible for:

- Performing scrolling.
- Rendering scroll indicators.
- Updating layouts.
- Executing commands.
- Business logic.

# Public API

Example:

```ts

  readonly state: AutoScrollState;

  readonly direction: AutoScrollDirection;

  readonly speed: number;
```

# Design Principles

The Auto Scroll component should be:

- Framework independent
- Immutable
- Strongly typed
- Lightweight
- Extensible

Auto scrolling should produce scroll intent only and remain independent from viewport implementations.
