# TASK-0038 — Implement Interaction Events

## Summary

Implement the `InteractionEvents` abstraction, a reusable event model responsible for publishing and subscribing to normalized interaction events within Atlas Studio.

The Interaction Events component provides a consistent event contract for interaction producers and consumers. It enables communication between interaction modules while remaining independent from rendering, application state, document manipulation, and business logic.

The Interaction Events abstraction is responsible only for defining and dispatching interaction events. It does not execute commands, perform rendering, or implement application-specific workflows.

---

# Goal

Provide a reusable, extensible, and framework-independent interaction event system capable of transporting normalized interaction events through a stable public API.

---

# Background

Every interaction generated by the interaction layer eventually becomes an event consumed by higher-level systems.

Typical consumers include:

- Editor
- Workspace
- Selection Manager
- Command System
- Plugin API
- Diagnostics

Instead of allowing each subsystem to define incompatible event contracts, Atlas Studio should expose a unified Interaction Events abstraction.

# Responsibilities

The Interaction Events component is responsible for:

- Publishing normalized interaction events.
- Registering event listeners.
- Removing event listeners.
- Preserving event ordering.
- Providing immutable event metadata.

The Interaction Events component is **not** responsible for:

- Executing commands.
- Managing editor state.
- Rendering UI.
- Persisting events.
- Business logic.

# Public API

Example:

```ts
interface InteractionEventBus {
  subscribe(listener: InteractionEventListener): void;

  unsubscribe(id: string): void;

  publish(event: InteractionEvent): void;
```

Example event:

```ts
interface InteractionEvent {
```

- Immutable
