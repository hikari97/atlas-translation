# TASK-0020 — Implement IME Interaction

## Summary

Implement the `IMEInteraction`, a reusable abstraction responsible for representing Input Method Editor (IME) composition sessions within Atlas Studio.

The IME Interaction provides a normalized representation of text composition across different operating systems and languages. It enables higher-level systems to observe composition lifecycle events without depending on platform-specific IME implementations.

The IME Interaction is responsible only for representing composition state and lifecycle. It does not modify documents, execute commands, or render composition UI.

---

# Goal

Provide a reusable, extensible, and framework-independent IME interaction abstraction capable of representing composition sessions through a stable public API.

---

# Background

Modern editors must support international text input.

Typical IME scenarios include:

- Chinese input
- Japanese input
- Korean input
- Vietnamese input
- Emoji composition
- Speech-to-text composition
- Future input methods

Instead of exposing browser-specific composition events directly, Atlas Studio should expose a unified IME Interaction abstraction.

# Responsibilities

The IME Interaction is responsible for:

- Representing composition sessions.
- Tracking composition lifecycle.
- Providing immutable composition metadata.
- Supporting international input methods.
- Normalizing platform-specific composition behavior.

The IME Interaction is **not** responsible for:

- Inserting text into documents.
- Rendering composition windows.
- Executing commands.
- Managing editor state.
- Business logic.

# Public API

Example:

```ts
interface IMEInteraction {
  readonly composition: IMEComposition;
```

- Immutable

- Extensible
