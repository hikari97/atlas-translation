# TASK-0028 — Implement Resize Preview

## Summary

Implement the `ResizePreview`, a reusable abstraction responsible for describing the visual preview metadata of an active resize operation.

The Resize Preview provides a normalized description of the temporary dimensions and position that should be displayed during a resize session. It contains presentation metadata only and delegates all rendering responsibilities to higher-level packages such as `atlas-ui` and `atlas-renderer`.

The Resize Preview does not modify layouts, update document state, or perform rendering.

---

# Goal

Provide a reusable, extensible, and framework-independent resize preview abstraction capable of describing resize preview information through a stable public API.

---

# Background

During resize operations, users expect immediate visual feedback before the final dimensions are committed.

Typical resize previews include:

- Selection outline
- Bounding box
- Ghost rectangle
- Resize handles
- Temporary dimensions
- Plugin-defined previews

Rather than embedding preview behavior into every resizable component, Atlas Studio should expose a unified Resize Preview abstraction.

# Responsibilities

The Resize Preview is responsible for:

- Representing preview dimensions.
- Representing preview position.
- Tracking preview lifecycle.
- Providing immutable preview metadata.
- Supporting future preview implementations.

The Resize Preview is **not** responsible for:

- Rendering.
- Updating layouts.
- Performing resize operations.
- Executing commands.
- Business logic.
