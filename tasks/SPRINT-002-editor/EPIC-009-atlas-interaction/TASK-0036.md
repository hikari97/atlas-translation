# TASK-0036 — Implement Multi Pointer

## Summary

Implement the `MultiPointerManager`, a reusable abstraction responsible for managing multiple active pointer interactions simultaneously within Atlas Studio.

The Multi Pointer Manager coordinates pointer lifecycles originating from touch, mouse, stylus, pen, and future input devices. It provides a unified view of active pointers that can be consumed by gesture recognizers, drag sessions, resize sessions, and other interaction systems.

The Multi Pointer Manager is responsible only for tracking pointer state and relationships. It does not recognize gestures, execute commands, modify application state, or perform rendering.

---

# Goal

Provide a reusable, extensible, and framework-independent multi-pointer management abstraction capable of tracking concurrent pointer interactions through a stable public API.

---

# Background

Modern interaction systems frequently require more than one active pointer.

Typical examples include:

- Pinch-to-zoom
- Rotate gestures
- Two-finger pan
- Stylus with touch support
- Multi-touch editing
- Collaborative interaction hardware
- Future pointer technologies

Instead of allowing each subsystem to manage active pointers independently, Atlas Studio should expose a centralized Multi Pointer Manager.

# Responsibilities

The Multi Pointer Manager is responsible for:

- Registering active pointers.
- Removing completed pointers.
- Tracking pointer relationships.
- Providing immutable pointer metadata.
- Supporting concurrent interaction sessions.

The Multi Pointer Manager is **not** responsible for:

- Recognizing gestures.
- Rendering cursors.
- Executing commands.
- Updating layouts.
- Business logic.

# Public API

Example:

```ts
interface MultiPointerManager {
  register(pointer: PointerEvent): void;

  update(pointer: PointerEvent): void;

  release(pointerId: string): void;

  getPointers(): readonly ActivePointer[];
}
```

Example active pointer:

```ts
interface ActivePointer {
  readonly id: string;

  readonly type: string;

  readonly x: number;

  readonly y: number;

  readonly pressure: number;
}
```

The Multi Pointer Manager should be:

- Immutable

- Lightweight
- Extensible
