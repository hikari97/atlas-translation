# TASK-0026 — Implement Resize Session

## Summary

Implement the `ResizeSession`, a reusable abstraction responsible for representing an active resize operation within Atlas Studio.

A Resize Session encapsulates the complete lifecycle of a resize interaction, including initialization, continuous updates, completion, and cancellation. It provides a unified interaction model for resizing editor elements, floating windows, panels, viewports, selections, and plugin-defined objects.

The Resize Session is responsible only for representing resize interaction state. It does not modify layouts, update dimensions, perform rendering, or execute application logic.

---

# Goal

Provide a reusable, extensible, and framework-independent resize session abstraction capable of representing resize operations through a stable public API.

---

# Background

Resizing is a fundamental interaction in graphical editors.

Typical resize operations include:

- Resizing floating panels
- Resizing editor panes
- Resizing selected objects
- Resizing images
- Resizing widgets
- Resizing containers
- Plugin-defined resize operations

Rather than allowing every subsystem to implement its own resize lifecycle, Atlas Studio should expose a unified Resize Session abstraction.

# Responsibilities

The Resize Session is responsible for:

- Representing an active resize operation.
- Tracking resize lifecycle.
- Recording initial bounds.
- Recording current bounds.
- Providing immutable resize metadata.

The Resize Session is **not** responsible for:

- Updating layouts.
- Applying constraints.
- Locking aspect ratios.
- Rendering previews.
- Business logic.
