# TASK-0009 — Implement Pointer Router

## Summary

Implement the `PointerRouter`, a reusable routing component responsible for dispatching normalized pointer events to the appropriate interaction session.

The Pointer Router decouples pointer event production from event consumption by determining which interaction session should receive a pointer event. It must remain independent from rendering, document manipulation, and business logic.

The Pointer Router does not perform hit testing or pointer capture. It only routes already-normalized pointer events.

---

# Goal

Provide a reusable, extensible, and framework-independent routing mechanism capable of delivering pointer events to registered interaction sessions through a stable public API.

---

# Background

A single pointer event may be consumed by different interaction systems depending on the current interaction state.

Examples include:

- Pointer selection
- Drag operation
- Resize operation
- Drawing operation
- Gesture recognition
- Custom interaction

Instead of coupling input devices directly to consumers, Atlas Studio should route events through a centralized Pointer Router.

# Responsibilities

The Pointer Router is responsible for:

- Receiving normalized pointer events.
- Resolving the destination interaction session.
- Dispatching events.
- Maintaining routing consistency.
- Supporting future routing extensions.

The Pointer Router is **not** responsible for:

- Performing hit testing.
- Capturing pointers.
- Tracking pointers.
- Rendering.
- Business logic.

# Design Principles

The Pointer Router should be:

- Framework independent
- Strongly typed
- Event driven
- Extensible
- Lightweight

Routing decisions should remain deterministic and independent from rendering or platform-specific APIs.
