# TASK-0019 — Implement Shortcut Dispatcher

## Summary

Implement the `ShortcutDispatcher`, a reusable abstraction responsible for dispatching recognized keyboard shortcuts to registered consumers.

The Shortcut Dispatcher acts as the bridge between the interaction layer and higher-level systems by forwarding validated shortcut events. It does not resolve shortcut definitions, execute commands, or manage application state.

The dispatcher must remain independent from rendering, document manipulation, and business logic.

---

# Goal

Provide a reusable, extensible, and framework-independent dispatcher capable of forwarding normalized shortcut interactions through a stable public API.

---

# Background

Keyboard shortcuts are one of the primary interaction mechanisms in Atlas Studio.

Typical shortcut consumers include:

- Command System
- Editor
- Workspace
- Selection
- Plugin System
- Future extensions

Instead of allowing every subsystem to observe keyboard interactions directly, Atlas Studio should expose a unified Shortcut Dispatcher.

# Responsibilities

The Shortcut Dispatcher is responsible for:

- Dispatching validated shortcut interactions.
- Managing shortcut consumers.
- Preserving dispatch order.
- Providing immutable dispatch metadata.
- Supporting future dispatcher extensions.

The Shortcut Dispatcher is **not** responsible for:

- Resolving shortcuts.
- Executing commands.
- Detecting key sequences.
- Managing editor state.
- Business logic.

# Public API

Example:

```ts
interface ShortcutDispatcher {
  register(consumer: ShortcutConsumer): void;

  unregister(id: string): void;

  dispatch(shortcut: ShortcutDispatch): void;
}
```

Example dispatch:

```ts
interface ShortcutDispatch {
  readonly id: string;

  readonly sequence: KeySequence;

  readonly timestamp: number;
}
```

The Shortcut Dispatcher should be:

- Strongly typed
- Event driven
- Lightweight
- Extensible

Dispatching should remain independent from command execution and shortcut resolution.
