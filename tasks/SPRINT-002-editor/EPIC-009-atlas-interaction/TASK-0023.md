# TASK-0023 — Implement Drag Preview

## Summary

Implement the `DragPreview`, a reusable abstraction responsible for representing the visual preview metadata associated with an active drag operation.

The Drag Preview provides a normalized description of what should be displayed during a drag session without performing any rendering. Rendering is delegated to higher-level packages such as `atlas-ui` or `atlas-renderer`.

The Drag Preview is responsible only for describing preview state and metadata. It does not draw, animate, or update visual elements.

---

# Goal

Provide a reusable, extensible, and framework-independent drag preview abstraction capable of describing drag preview information through a stable public API.

---

# Background

Modern editors typically display visual feedback while dragging.

Typical drag previews include:

- Ghost elements
- Floating previews
- Drag outlines
- Placeholder previews
- Multi-selection previews
- Plugin-defined previews

Rather than allowing every feature to define its own preview model, Atlas Studio should expose a unified Drag Preview abstraction.

# Responsibilities

The Drag Preview is responsible for:

- Representing drag preview metadata.
- Tracking preview lifecycle.
- Defining preview position offset.
- Exposing immutable preview information.
- Supporting future preview implementations.

The Drag Preview is **not** responsible for:

- Rendering previews.
- Animating previews.
- Updating layouts.
- Executing commands.
- Business logic.

# Public API

Example:

```ts
interface DragPreview {
  readonly dragSessionId: string;

  readonly state: DragPreviewState;

  readonly offsetX: number;

  readonly offsetY: number;

  readonly visible: boolean;
}
```

Example options:

```ts
interface DragPreviewOptions {

  readonly offsetX: number;

  readonly offsetY: number;
```
