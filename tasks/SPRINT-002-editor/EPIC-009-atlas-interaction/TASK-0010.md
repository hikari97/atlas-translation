# TASK-0010 — Implement Pointer Hit Test

## Summary

Implement the `PointerHitTest`, a reusable abstraction responsible for determining the interaction target associated with a normalized pointer event.

The Pointer Hit Test translates pointer coordinates into logical interaction targets without depending on rendering implementations, document models, or platform-specific APIs.

It provides the foundation for pointer selection, dragging, resizing, hovering, and other pointer-based interactions.

---

# Goal

Provide a reusable, extensible, and framework-independent hit-testing abstraction capable of resolving interaction targets through a stable public API.

---

# Background

Most pointer interactions begin by determining which object the pointer is currently interacting with.

Examples include:

- Selecting an editor block
- Clicking a toolbar button
- Resizing a panel
- Dragging an object
- Hovering a control
- Activating a custom interaction target

Rather than coupling interaction systems directly to rendering or DOM structures, Atlas Studio should expose a generic hit-testing abstraction.

# Responsibilities

The Pointer Hit Test is responsible for:

- Resolving interaction targets.
- Returning hit test results.
- Providing immutable target metadata.
- Supporting future target types.
- Remaining independent from rendering implementations.

The Pointer Hit Test is **not** responsible for:

- Rendering.
- Layout computation.
- Pointer routing.
- Pointer capture.
- Business logic.
