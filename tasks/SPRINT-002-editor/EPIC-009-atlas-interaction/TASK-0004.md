# TASK-0004 — Implement Interaction Context

## Summary

Implement the `InteractionContext`, a reusable context object that provides the environment required for an interaction session.

The Interaction Context acts as the bridge between the interaction layer and higher-level consumers by exposing the information required during an interaction without coupling the interaction system to specific implementations.

The Interaction Context must remain immutable during a single interaction session.

---

# Goal

Provide a reusable, extensible, and framework-independent context object that supplies interaction sessions with the information required to process user interactions.

---

# Background

Every interaction requires contextual information.

For example:

- Current pointer position
- Active viewport
- Active workspace
- Current interaction target
- Modifier keys
- Active input device

Rather than allowing interaction sessions to query these values directly from multiple systems, Atlas Studio should provide a single Interaction Context.

# Responsibilities

The Interaction Context is responsible for:

- Providing interaction metadata.
- Exposing the active interaction target.
- Providing viewport information.
- Providing input device information.
- Providing modifier key information.

The Interaction Context is **not** responsible for:

- Managing interaction sessions.
- Rendering.
- Executing commands.
- Editing documents.
- Business logic.

---


# Context Information

The context should provide:

- Session identifier
- Active target
- Input device
- Pointer position
- Viewport identifier
- Modifier keys
- Timestamp

Future context properties should remain backward compatible.

---

# Public API

Example:

```ts
interface InteractionContext {
  readonly sessionId: string;

  readonly targetId?: string;

  readonly device: string;

  readonly viewportId?: string;

  readonly timestamp: number;
}
```

---

# Design Principles

The Interaction Context should be:

- Immutable
- Serializable
- Framework independent
- Strongly typed
- Extensible

Interaction sessions should consume context instead of querying external systems directly.

---

# Technical Requirements

The implementation must:

- Preserve immutable public contracts.
- Support future context extensions.
- Avoid framework-specific dependencies.
- Remain lightweight.
- Preserve API stability.

---

# Accessibility Requirements

The Interaction Context must:

- Support accessibility workflows.
- Preserve interaction semantics.
- Remain presentation independent.
