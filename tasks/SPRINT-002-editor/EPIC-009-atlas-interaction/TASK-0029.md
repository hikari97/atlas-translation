# TASK-0029 — Implement Aspect Ratio Lock

## Summary

Implement the `AspectRatioLock`, a reusable abstraction responsible for preserving aspect ratios during resize interactions.

The Aspect Ratio Lock provides a consistent mechanism for maintaining proportional width and height while resizing graphical objects, panels, images, videos, and other resizable elements.

The Aspect Ratio Lock is responsible only for calculating constrained dimensions. It does not resize objects, update layouts, or perform rendering.

---

# Goal

Provide a reusable, extensible, and framework-independent aspect ratio constraint abstraction capable of preserving proportional dimensions through a stable public API.

---

# Background

Many resize operations require proportional scaling.

Typical examples include:

- Image resizing
- Video resizing
- Canvas resizing
- Shape resizing
- Selection bounding boxes
- Plugin-defined resize behavior

Instead of embedding aspect ratio calculations into every resize implementation, Atlas Studio should expose a unified Aspect Ratio Lock abstraction.

# Responsibilities

The Aspect Ratio Lock is responsible for:

- Preserving proportional dimensions.
- Calculating constrained width and height.
- Providing immutable ratio metadata.
- Supporting configurable ratio modes.
- Supporting future resize policies.

The Aspect Ratio Lock is **not** responsible for:

- Performing resize operations.
- Updating layouts.
- Rendering previews.
- Executing commands.
- Business logic.

# Public API

Example:

```ts
interface AspectRatioLock {
  readonly enabled: boolean;

  readonly ratio: AspectRatio;

  readonly state: AspectRatioLockState;

  apply(
    width: number,
    height: number,
  ): Readonly<{
    width: number;
    height: number;
  }>;
}
```

Example ratio:

```ts
interface AspectRatio {
  readonly width: number;

  readonly height: number;

  readonly value: number;
}
```

# Design Principles

The Aspect Ratio Lock should be:

- Framework independent
- Immutable
- Strongly typed
- Lightweight
- Extensible

Aspect ratio calculations should remain deterministic and independent from rendering or layout systems.
