# TASK-0015 — Implement Context Interaction

## Summary

Implement the `ContextInteraction`, a reusable abstraction responsible for representing context menu interactions initiated by user input.

The Context Interaction provides a unified model for context requests across supported input devices, including mouse secondary-click, keyboard context key, and platform-specific context gestures.

The Context Interaction is responsible only for identifying and describing a context interaction. It does not display menus, execute commands, or manage application state.

---

# Goal

Provide a reusable, extensible, and framework-independent context interaction abstraction capable of representing context menu requests through a stable public API.

---

# Background

Context interactions provide users with access to actions related to a specific object or location.

Typical examples include:

- Right-click on editor content
- Right-click on panels
- Right-click on toolbar items
- Keyboard Context Menu key
- Shift + F10
- Platform-specific context gestures

Rather than allowing each component to detect context interactions independently, Atlas Studio should expose a unified Context Interaction abstraction.

# Responsibilities

The Context Interaction is responsible for:

- Detecting context interaction requests.
- Identifying the interaction target.
- Recording the trigger source.
- Providing immutable interaction metadata.
- Supporting future context interaction sources.

The Context Interaction is **not** responsible for:

- Rendering context menus.
- Executing commands.
- Managing selections.
- Updating documents.
- Business logic.

# Supported Trigger Sources

The interaction should support:

- Mouse Secondary Button
- Keyboard Context Key
- Shift + F10
- Touch Long Press
- Platform-specific Context Gesture

Future trigger sources should remain backward compatible.

---

# Supported States

The Context Interaction should support:

- Idle
- Requested
- Completed
- Cancelled

# Public API

Example:

```ts
interface ContextInteraction {
  readonly pointerId?: string;

  readonly targetId?: string;

  readonly trigger: ContextTriggerType;

  readonly state: ContextInteractionState;

  readonly timestamp: number;
}
```

Example trigger type:

```ts
enum ContextTriggerType {
  Mouse,

  Keyboard,

  Touch,

  Platform,
}
```

# Design Principles

The Context Interaction should be:

- Framework independent
- Immutable
- Strongly typed
- Lightweight
- Extensible

Context interactions should describe user intent without being coupled to UI components or platform-specific menu implementations.

---

# Technical Requirements

The implementation must:

- Support multiple trigger sources.
- Preserve immutable interaction metadata.
- Avoid framework-specific dependencies.
- Support future context interaction extensions.
- Preserve API stability.

---

# Accessibility Requirements

The Context Interaction must:

- Support keyboard accessibility.
- Support assistive technologies.
- Preserve semantic interaction behavior.
- Remain presentation independent.
