# TASK-0030 — Implement Resize Anchors

## Summary

Implement the `ResizeAnchors`, a reusable abstraction responsible for defining and managing anchor points used during resize interactions.

Resize Anchors determine the fixed reference point while an object is being resized. They provide a consistent interaction model across panels, windows, canvases, graphical objects, and plugin-defined components.

The Resize Anchors abstraction is responsible only for describing anchor metadata and resize behavior. It does not resize objects, update layouts, or perform rendering.

---

# Goal

Provide a reusable, extensible, and framework-independent resize anchor abstraction capable of defining resize reference points through a stable public API.

---

# Background

Every resize operation requires one or more anchor points.

Examples include:

- Top
- Bottom
- Left
- Right
- Top Left
- Top Right
- Bottom Left
- Bottom Right
- Center

Rather than allowing every component to calculate anchor behavior independently, Atlas Studio should expose a unified Resize Anchors abstraction.

# Responsibilities

The Resize Anchors component is responsible for:

- Defining resize anchor positions.
- Identifying the active anchor.
- Providing immutable anchor metadata.
- Supporting future anchor extensions.
- Standardizing resize reference points.

The Resize Anchors component is **not** responsible for:

- Applying resize transformations.
- Rendering resize handles.
- Updating layouts.
- Executing commands.
- Business logic.
