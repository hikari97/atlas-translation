# TASK-0017 — Implement Modifier Keys

## Summary

Implement the `ModifierKeys` abstraction for Atlas Interaction.

The Modifier Keys component provides a normalized representation of keyboard modifier states such as **Shift**, **Control**, **Alt**, and **Meta**. It enables interaction systems to consistently determine active modifier combinations across operating systems and input devices.

The Modifier Keys abstraction is responsible only for representing modifier state. It does not execute shortcuts, dispatch commands, or perform platform-specific behavior.

---

# Goal

Provide a reusable, extensible, and framework-independent modifier key abstraction capable of representing keyboard modifier states through a stable public API.

---

# Background

Modifier keys influence the behavior of nearly every interaction within an editor.

Examples include:

- Multi-selection
- Copy and paste
- Range selection
- Duplicate objects
- Constrained dragging
- Aspect ratio locking
- Shortcut activation

Instead of querying modifier state directly from platform APIs, Atlas Studio should expose a unified Modifier Keys abstraction.

# Responsibilities

The Modifier Keys component is responsible for:

- Representing active modifier keys.
- Normalizing platform-specific modifier behavior.
- Exposing immutable modifier state.
- Supporting future modifier extensions.
- Providing a consistent API across operating systems.

The Modifier Keys component is **not** responsible for:

- Executing shortcuts.
- Dispatching commands.
- Detecting key sequences.
- Handling IME composition.
- Business logic.

# Public API

Example:

```ts
interface ModifierKeys {
```

The Modifier Keys component should be:

- Immutable
- Strongly typed
- Lightweight
- Extensible

Modifier key state should remain independent from shortcut processing and command execution.
