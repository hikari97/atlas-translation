# TASK-0033 — Implement Zoom Gesture

## Summary

Implement the `ZoomGesture`, a reusable abstraction responsible for representing scale-based gesture interactions within Atlas Studio.

The Zoom Gesture detects changes in distance between two or more active pointers and exposes normalized scale information that can be consumed by higher-level systems such as viewports, editors, canvases, and plugins.

The Zoom Gesture is responsible only for describing zoom state and scaling metadata. It does not modify viewport scale, transform objects, or perform rendering.

---

# Goal

Provide a reusable, extensible, and framework-independent zoom gesture abstraction capable of representing scaling gestures through a stable public API.

---

# Background

Zoom interactions are fundamental in modern graphical applications.

Typical examples include:

- Canvas zoom
- Image zoom
- Document zoom
- Timeline zoom
- Map zoom
- Pinch-to-zoom
- Trackpad zoom

Rather than allowing every subsystem to calculate scale independently, Atlas Studio should expose a unified Zoom Gesture abstraction.

# Responsibilities

The Zoom Gesture is responsible for:

- Detecting scaling gestures.
- Calculating scale factors.
- Calculating zoom center.
- Tracking gesture lifecycle.
- Providing immutable zoom metadata.

The Zoom Gesture is **not** responsible for:

- Updating viewport scale.
- Transforming objects.
- Rendering animations.
- Executing commands.
- Business logic.
