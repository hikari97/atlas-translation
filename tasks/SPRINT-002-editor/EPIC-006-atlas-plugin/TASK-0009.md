# TASK-0009 — Implement Plugin Manager

## Summary

Implement the Plugin Manager for Atlas Studio.

The Plugin Manager is responsible for coordinating all plugin-related operations during application startup and shutdown.

It orchestrates plugin discovery, registration, dependency validation, lifecycle execution, and runtime state while delegating specialized responsibilities to other components.

The Plugin Manager serves as the primary entry point of the Plugin System.

---

# Goal

Provide a centralized component that coordinates plugin management while remaining independent from plugin implementations.

---

# Background

The Plugin System consists of multiple independent components.

Rather than allowing every runtime component to communicate directly with each other, the Plugin Manager coordinates plugin operations through a single façade.

This simplifies runtime integration and provides a stable API for atlas-core.

# Responsibilities

Plugin Manager is responsible for:

- Coordinating plugin discovery.
- Coordinating plugin registration.
- Coordinating dependency validation.
- Coordinating lifecycle execution.
- Providing plugin lookup.
- Managing runtime plugin state.

Plugin Manager is **not** responsible for:

- Rendering UI.
- Dependency Injection.
- Runtime bootstrapping.
- Plugin implementation details.

# Public API

Example:

```ts
interface PluginManager {
  initialize(): Promise<void>;

  start(): Promise<void>;

  stop(): Promise<void>;

  register(plugin: PluginDefinition): void;

  unregister(pluginId: string): void;

  has(pluginId: string): boolean;

  get(pluginId: string): PluginDefinition | undefined;

  getAll(): readonly PluginDefinition[];
}
```

# Design Principles

The Plugin Manager should be:

- Deterministic
- Strongly typed
- Extensible
- Runtime independent
- Easy to test

The Plugin Manager should coordinate components rather than implement their logic directly.

---

# Technical Requirements

The implementation must:

- Use Plugin Registry.
- Use Plugin Dependency Graph.
- Use Plugin Discovery.
- Avoid duplicated responsibilities.
- Preserve public API stability.
