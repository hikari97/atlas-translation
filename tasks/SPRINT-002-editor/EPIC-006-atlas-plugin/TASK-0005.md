# TASK-0005 — Implement Plugin Builder

## Summary

Implement the Plugin Builder for Atlas Studio.

The Plugin Builder provides a fluent API for creating immutable Plugin Definitions before they are registered with the Plugin Registry.

The builder validates required properties during construction while remaining independent from runtime execution.

---

# Goal

Provide a strongly typed builder that simplifies plugin definition while ensuring consistency and validation before runtime registration.

---

# Background

Creating Plugin objects manually becomes difficult as plugins gain additional metadata and capabilities.

A dedicated builder provides:

- Fluent API
- Immutable output
- Required field validation
- Consistent plugin definitions

The builder does not instantiate or activate plugins.

---

# Responsibilities

Plugin Builder is responsible for:

- Building plugin definitions
- Validating required properties
- Producing immutable plugin definitions
- Providing fluent APIs

Plugin Builder is **not** responsible for:

- Registering plugins
- Loading plugins
- Managing lifecycle
- Dependency resolution

---

# Builder Flow

```text
PluginBuilder

↓

Set Metadata

↓

Set Dependencies

↓

Set Lifecycle

↓

Validate

↓

Build

↓

PluginDefinition
```

---

# Suggested API

Example:

```ts
const plugin = PluginBuilder.create('atlas.example')
  .name('Example Plugin')
  .version('1.0.0')
  .description('Example plugin')
  .build();
```

The final API may evolve during implementation but should remain fluent and strongly typed.

---

# Design Principles

Plugin Builder should be:

- Fluent
- Immutable
- Strongly typed
- Predictable
- Easy to use

The builder should fail fast when required properties are missing.

---

# Technical Requirements

The implementation must:

- Support method chaining.
- Validate required fields.
- Produce immutable output.
- Avoid runtime logic.
- Preserve public API stability.
