# TASK-0018 — Integrate Plugin Runtime

## Summary

Integrate the Atlas Plugin System with the Atlas Runtime (`atlas-core`).

This task establishes the runtime integration layer responsible for initializing the Plugin System during application startup and coordinating plugin execution throughout the application lifecycle.

The integration must use only the public APIs exposed by `atlas-core`.

---

# Goal

Provide seamless integration between `atlas-plugin` and `atlas-core` while preserving package isolation and architectural boundaries.

---

# Background

Previous tasks established the Plugin System independently.

This task connects the Plugin System to the Runtime so that plugins become part of the application lifecycle without introducing tight coupling.

The Runtime remains the application orchestrator while the Plugin System manages plugin behavior.

---

- Runtime Context Integration

# Responsibilities

Plugin Runtime Integration is responsible for:

- Connecting Plugin Manager to Runtime.
- Initializing plugins during startup.
- Stopping plugins during shutdown.
- Providing runtime plugin services.
- Coordinating plugin bootstrap.

Plugin Runtime Integration is **not** responsible for:

- Runtime implementation.
- Dependency Injection.
- Renderer lifecycle.
- UI lifecycle.
- Workspace lifecycle.

---

# Runtime Flow

```text
Application

↓

↓

↓

```

# Public API

Example:

```ts
interface PluginRuntime {
  initialize(): Promise<void>;

  start(): Promise<void>;

  stop(): Promise<void>;

  dispose(): Promise<void>;
}
```

---

# Design Principles

Plugin Runtime Integration should be:

- Modular
- Deterministic
- Strongly typed
- Runtime independent
- Easy to extend

The Runtime should communicate with plugins only through the Plugin Runtime abstraction.

---

# Technical Requirements

The implementation must:

- Use only public APIs from atlas-core.
- Avoid circular dependencies.
- Preserve lifecycle ordering.
- Support asynchronous startup.
- Preserve public API stability.
