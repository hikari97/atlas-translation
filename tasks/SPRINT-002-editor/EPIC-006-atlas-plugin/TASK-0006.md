# TASK-0006 — Implement Plugin Registry

## Summary

Implement the Plugin Registry for Atlas Studio.

The Plugin Registry acts as the central repository for plugin definitions. It is responsible for registering, storing, querying, and removing plugins while maintaining uniqueness and consistency.

The registry manages plugin definitions only and does not execute plugin lifecycle.

---

# Goal

Provide a strongly typed registry that manages plugin definitions and exposes a predictable API for runtime components.

---

# Background

As the number of plugins grows, the runtime requires a central registry to keep track of all registered plugins.

The Plugin Registry provides a single source of truth for plugin definitions and enables other runtime components such as the Plugin Manager and Dependency Resolver to retrieve plugin information consistently.

---

# Responsibilities

Plugin Registry is responsible for:

- Registering plugins
- Unregistering plugins
- Finding plugins
- Listing plugins
- Preventing duplicate registration

Plugin Registry is **not** responsible for:

- Activating plugins
- Loading plugins
- Resolving dependencies
- Executing lifecycle methods

---

# Public API

Example:

```ts
interface PluginRegistry {
  register(plugin: PluginDefinition): void;

  unregister(id: string): void;

  has(id: string): boolean;

  get(id: string): PluginDefinition | undefined;

  getAll(): readonly PluginDefinition[];

  clear(): void;
}
```

---

# Design Principles

The Plugin Registry should be:

- Deterministic
- Immutable from outside
- Strongly typed
- Fast lookup
- Independent from runtime lifecycle

Plugin definitions should be uniquely identified by their plugin identifier.

---

# Technical Requirements

The implementation must:

- Prevent duplicate registrations.
- Preserve registration order.
- Return immutable collections.
- Avoid runtime-specific APIs.
- Preserve public API stability.
