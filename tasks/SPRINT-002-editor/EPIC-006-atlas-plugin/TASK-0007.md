# TASK-0007 — Implement Plugin Dependency Graph

## Summary

Implement the Plugin Dependency Graph for Atlas Studio.

The Plugin Dependency Graph represents dependency relationships between registered plugins. It provides the runtime with a deterministic view of plugin dependencies without executing any plugin lifecycle.

The graph is responsible for dependency analysis only.

---

# Goal

Provide a strongly typed dependency graph that enables dependency validation and initialization ordering for plugins.

---

# Background

Plugins frequently depend on other plugins.

Instead of resolving dependencies dynamically every time the runtime starts, Atlas Studio maintains a dependency graph that describes the relationships between plugins.

This graph becomes the foundation for dependency validation and lifecycle ordering.

# Responsibilities

Plugin Dependency Graph is responsible for:

- Representing plugin dependencies.
- Building dependency relationships.
- Detecting missing dependencies.
- Detecting circular dependencies.
- Producing initialization order.

Plugin Dependency Graph is **not** responsible for:

- Loading plugins.
- Registering plugins.
- Executing lifecycle.
- Creating plugin instances.

---

# Dependency Model

```text
Plugin A

↓

Plugin B

↓

Plugin C
```

Initialization order:

```text
Plugin C

↓

Plugin B

↓

Plugin A
```

---

# Public API

Example:

```ts
interface PluginDependencyGraph {
  add(plugin: PluginDefinition): void;

  remove(pluginId: string): void;

  has(pluginId: string): boolean;

  dependenciesOf(pluginId: string): readonly string[];

  dependentsOf(pluginId: string): readonly string[];

  validate(): ValidationResult;

  sort(): readonly PluginDefinition[];
}
```

---

# Design Principles

The Plugin Dependency Graph should be:

- Deterministic
- Immutable from outside
- Strongly typed
- Fast to traverse
- Independent from runtime execution

Dependency ordering must always produce the same result for the same graph.

---

# Technical Requirements

The implementation must:

- Detect circular dependencies.
- Detect missing dependencies.
- Preserve deterministic ordering.
- Avoid duplicate nodes.
- Preserve public API stability.
