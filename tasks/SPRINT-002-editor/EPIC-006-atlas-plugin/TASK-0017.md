# TASK-0017 — Integrate Plugin Command System

## Summary

Integrate Atlas Plugin with the Atlas Command System.

This task enables plugins to execute commands and register command handlers through the public interfaces provided by `atlas-command`.

The integration must preserve package boundaries and avoid introducing direct dependencies on command implementations.

---

# Goal

Allow plugins to participate in the Atlas Command System while remaining fully decoupled from command execution internals.

---

# Background

Plugins often need to trigger application behavior.

Instead of invoking services directly, plugins should communicate through the Command System.

This approach provides:

- Loose coupling
- Better testability
- Consistent execution pipeline
- Future middleware support

---

# Scope

## Included

- Plugin Command Integration
- Command Execution
- Command Handler Registration
- Command Context Integration

---

## Excluded

- Command Bus implementation
- Command middleware
- Command persistence
- Distributed commands
- Command scheduling

# Responsibilities

Plugin Command Integration is responsible for:

- Executing commands.
- Registering command handlers.
- Connecting plugins to the Command System.
- Exposing command execution through Plugin Context.

Plugin Command Integration is **not** responsible for:

- Command Bus implementation.
- Command routing.
- Middleware execution.
- Retry policies.

---

# Integration Flow

```text
Plugin

↓

Plugin Context

↓

Plugin Command Integration

↓

atlas-command

↓

Command Bus
```

---

# Public API

Example:

```ts
interface PluginCommandIntegration {
  execute<TResult>(command: Command<TResult>): Promise<TResult>;

  register<TCommand>(handler: CommandHandler<TCommand>): void;
}
```

---

# Design Principles

Plugin Command Integration should be:

- Strongly typed
- Loosely coupled
- Runtime independent
- Extensible
- Easy to test

Plugins should interact only with public contracts from `atlas-command`.

---

# Technical Requirements

The implementation must:

- Use only public APIs from atlas-command.
- Avoid circular dependencies.
- Support asynchronous command execution.
- Preserve public API stability.
