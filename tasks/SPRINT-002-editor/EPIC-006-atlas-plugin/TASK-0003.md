# TASK-0003 — Implement Plugin Metadata

## Summary

Implement the Plugin Metadata contract for Atlas Studio.

Plugin Metadata provides descriptive information about a plugin. It is intended for runtime inspection, diagnostics, documentation, and developer tooling.

Metadata must remain immutable and independent from plugin implementation.

---

# Goal

Create a strongly typed metadata model that represents descriptive information about a plugin.

---

# Background

Besides its runtime behavior, every plugin should expose metadata that helps developers and the runtime understand what the plugin is, who maintains it, and which version is being used.

Plugin Metadata should never contain runtime state.

---

# Responsibilities

Plugin Metadata is responsible for describing:

- Plugin information
- Plugin documentation
- Plugin ownership
- Plugin version
- Plugin references

Plugin Metadata is **not** responsible for:

- Runtime execution
- Dependency resolution
- Plugin activation
- Configuration

---

# Suggested Structure

Example:

```ts
export interface PluginMetadata {
  readonly displayName: string;

  readonly description?: string;

  readonly version: string;

  readonly author?: string;

  readonly license?: string;

  readonly homepage?: string;

  readonly repository?: string;

  readonly keywords?: readonly string[];
}
```

---

# Design Principles

Plugin Metadata should be:

- Immutable
- Serializable
- Human readable
- Strongly typed
- Independent from runtime

---

# Technical Requirements

The implementation must:

- Use readonly properties.
- Avoid runtime logic.
- Preserve backward compatibility.
- Support future metadata extensions.
