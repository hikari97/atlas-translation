# TASK-0007 — Implement Workspace Manager

## Summary

Implement the `WorkspaceManager`, the central abstraction responsible for creating, registering, activating, suspending, and disposing Workspace instances throughout Atlas Studio.

The Workspace Manager coordinates the lifecycle of multiple workspaces while ensuring runtime isolation between projects. It provides deterministic workspace registration, active workspace switching, lookup, lifecycle coordination, and future extensibility without depending on rendering engines, browser APIs, operating system window managers, or application frameworks.

The Workspace Manager is responsible only for managing Workspace instances. It does not manage documents, render interfaces, process input, execute commands, or implement business logic.

---

# Goal

Provide a reusable, deterministic, immutable, and framework-independent Workspace Manager capable of coordinating every Workspace instance through a stable public API.

---

# Background

Modern creative applications commonly support:

- Multiple projects
- Multiple tabs
- Split workspaces
- Detached windows
- Temporary review workspaces
- Plugin workspaces
- Background workspaces

Each workspace should remain fully isolated while still being coordinated by a single runtime service.

Atlas Studio should expose one Workspace Manager abstraction.

# Responsibilities

The Workspace Manager is responsible for:

- Creating Workspace instances.
- Registering Workspaces.
- Tracking the active Workspace.
- Coordinating Workspace lifecycle.
- Supporting multiple Workspaces.
- Preserving deterministic runtime behavior.

The Workspace Manager is **not** responsible for:

- Editing documents.
- Rendering graphics.
- Executing commands.
- Processing input.
- Implementing business logic.

# Public API

Example:

```ts
interface WorkspaceManager {
  readonly state: WorkspaceManagerState;

  readonly activeWorkspace: Workspace | null;

  create(options: WorkspaceOptions): Workspace;
```
