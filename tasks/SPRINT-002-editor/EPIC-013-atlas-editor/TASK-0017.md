# TASK-0017 — Implement Active Tool

## Summary

Implement the `ActiveTool`, the runtime abstraction responsible for representing the currently active editor tool within a Workspace.

The Active Tool exposes the tool currently receiving user interaction and editor events. It coordinates runtime ownership of the active tool while preserving deterministic behavior across input processing, rendering, selection, translation, OCR, inpainting, layout, automation workflows, and future editor capabilities.

The Active Tool is responsible only for exposing the currently active tool. It does not implement tool behavior, render graphics, process input, execute commands, or implement business logic.

---

# Goal

Provide a reusable, immutable, deterministic, and framework-independent Active Tool abstraction capable of exposing the currently active editing tool through a stable public API.

---

# Background

Professional editors expose many editing tools.

Typical examples include:

- Selection
- Move
- Crop
- Brush
- Text
- Pen
- Shape
- OCR
- Translation
- Inpainting
- Batch Translation
- Export

Although many tools may be registered, only one primary tool should own editor interaction at any given moment.

Atlas Studio should expose one Active Tool abstraction.

# Responsibilities

The Active Tool is responsible for:

- Representing the active editor tool.
- Exposing immutable runtime references.
- Supporting runtime inspection.
- Supporting future tool categories.
- Preserving deterministic runtime behavior.

The Active Tool is **not** responsible for:

- Implementing tool behavior.
- Registering tools.
- Rendering.
- Processing input.
- Executing commands.
- Implementing business logic.
