# TASK-0003 — Implement Editor State

## Summary

Implement the `EditorState`, the immutable runtime abstraction responsible for representing the complete operational state of an Editor instance within Atlas Studio.

The Editor State provides a deterministic snapshot of the editor runtime, including lifecycle state, active workspace, active document, active tool, editing mode, session information, runtime flags, and future editor state extensions. It serves as the single source of truth for runtime state while remaining completely independent from rendering engines, browser APIs, operating system windowing systems, application frameworks, and editor implementations.

The Editor State is responsible only for representing runtime state. It does not modify editor behavior, dispatch events, execute commands, render interfaces, or implement business logic.

---

# Goal

Provide a reusable, immutable, deterministic, and framework-independent runtime state abstraction capable of representing the complete Editor state through a stable public API.

---

# Background

An editor runtime consists of multiple coordinated subsystems.

Examples include:

- Active workspace
- Active document
- Active tool
- Current editing mode
- Session information
- Runtime lifecycle
- Read-only state
- Busy state

Rather than storing runtime information across multiple services, Atlas Studio should expose one unified Editor State abstraction.

# Responsibilities

The Editor State is responsible for:

- Representing runtime state.
- Exposing immutable state snapshots.
- Tracking lifecycle status.
- Tracking active runtime references.
- Supporting future state extensions.
- Preserving deterministic state representation.

The Editor State is **not** responsible for:

- Changing runtime state.
- Executing commands.
- Rendering.
- Managing workspaces.
- Processing input.
- Implementing business logic.

# Public API

Example:

```ts
interface EditorState {
  readonly lifecycle: EditorLifecycleState;

  readonly activeWorkspace: Workspace | null;

  readonly activeDocument: Document | null;

  readonly activeMode: EditorMode | null;

  readonly activeSession: EditorSession | null;
```
