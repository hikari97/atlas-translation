# TASK-0028 — Implement Session Manager

## Summary

Implement the `SessionManager`, the central abstraction responsible for creating, registering, restoring, persisting, and disposing runtime sessions throughout Atlas Studio.

The Session Manager coordinates the complete session hierarchy, including Editor Sessions, Workspace Sessions, Document Sessions, AI runtime sessions, background task sessions, and future session providers while remaining completely independent from rendering engines, browser APIs, operating systems, storage implementations, and application frameworks.

The Session Manager is responsible only for coordinating runtime sessions. It does not render graphics, execute commands, manage editor behavior, or implement business logic.

---

# Goal

Provide a reusable, deterministic, immutable, and framework-independent Session Manager capable of coordinating every runtime session through a stable public API.

---

# Background

Atlas Studio maintains multiple session scopes.

Examples include:

- Editor Session
- Workspace Session
- Document Session
- OCR Session
- Translation Session
- Export Session
- Batch Session

Rather than allowing each subsystem to manage its own lifecycle independently, Atlas Studio should expose one unified Session Manager.

# Responsibilities

The Session Manager is responsible for:

- Registering sessions.
- Tracking runtime sessions.
- Coordinating session restoration.
- Coordinating persistence.
- Supporting future session providers.
- Preserving deterministic runtime behavior.

The Session Manager is **not** responsible for:

- Saving session data.
- Loading storage.
- Rendering.
- Executing commands.
- Implementing business logic.

# Public API

```ts
interface SessionManager {

  resolve<T extends Session>(id: string): T | undefined;
```
