# TASK-0023 — Implement Active Mode

## Summary

Implement the `ActiveMode`, the runtime abstraction responsible for representing the currently active Editor Mode within an Editor instance.

The Active Mode acts as the behavioral authority for the editor runtime. It exposes the currently active operational mode together with its capabilities, runtime policies, keyboard profile, interaction rules, workflow permissions, AI features, and future editor behaviors while remaining independent from rendering engines, browser APIs, operating systems, and application frameworks.

The Active Mode is responsible only for representing the currently active mode. It does not execute mode logic, render graphics, process user input, or implement business logic.

---

# Goal

Provide a reusable, immutable, deterministic, and framework-independent Active Mode abstraction capable of exposing the current editor behavior through a stable public API.

---

# Background

Although multiple modes may exist, only one mode governs editor behavior at a given time.

Typical examples include:

- Edit
- View
- Review
- OCR
- Translation
- Typesetting
- Export
- Batch

The editor runtime should always expose the active behavioral profile through a unified abstraction.

# Responsibilities

The Active Mode is responsible for:

- Representing the current editor behavior.
- Exposing immutable mode metadata.
- Exposing runtime capabilities.
- Supporting runtime inspection.
- Supporting future mode providers.
- Preserving deterministic runtime behavior.

The Active Mode is **not** responsible for:

- Executing mode logic.
- Switching modes.
- Rendering.
- Processing input.
- Implementing business logic.
