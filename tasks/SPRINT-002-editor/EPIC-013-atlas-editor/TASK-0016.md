# TASK-0016 — Implement Tool Manager

## Summary

Implement the `ToolManager`, the central abstraction responsible for registering, activating, deactivating, and coordinating every editing tool available within an Atlas Studio Workspace.

The Tool Manager provides deterministic runtime management for all editor tools including selection, crop, brush, pen, shape, text, OCR, translation, inpainting, measurement, annotation, and future plugin-defined tools while remaining completely independent from rendering engines, browser APIs, operating system APIs, and application frameworks.

The Tool Manager is responsible only for tool orchestration. It does not implement tool behavior, render graphics, process input, or execute business logic.

---

# Goal

Provide a reusable, deterministic, immutable, and framework-independent Tool Manager capable of coordinating every editor tool through a stable public API.

---

# Background

Professional editors expose many tools.

Examples include:

- Selection

- Hand
- Zoom
- Crop
- Brush
- Eraser
- Shape
- Text
- Eyedropper

- Measure

Only one primary tool should receive editor input at a given moment.

Atlas Studio should expose one unified Tool Manager abstraction.

# Responsibilities

The Tool Manager is responsible for:

- Registering editor tools.
- Tracking the active tool.
- Activating tools.
- Deactivating tools.
- Supporting plugin-defined tools.
- Preserving deterministic runtime behavior.

The Tool Manager is **not** responsible for:

- Implementing tool logic.
- Rendering.
- Processing input.
- Executing commands.
- Implementing business logic.
