# TASK-0022 — Implement Mode Manager

## Summary

Implement the `ModeManager`, the central abstraction responsible for registering, activating, deactivating, and coordinating every Editor Mode within Atlas Studio.

The Mode Manager provides deterministic runtime management for editor modes including Edit, View, Review, OCR, Translation, Typesetting, Batch Processing, Export, and future plugin-defined modes. It coordinates behavioral policies without depending on rendering engines, browser APIs, operating systems, or application frameworks.

The Mode Manager is responsible only for mode orchestration. It does not execute mode behavior, process user input, render graphics, or implement business logic.

---

# Goal

Provide a reusable, deterministic, immutable, and framework-independent Mode Manager capable of coordinating editor modes through a stable public API.

---

# Background

Only one Editor Mode should govern editor behavior at any given moment.

Examples include:

- Edit Mode
- Review Mode
- OCR Mode
- Translation Mode
- Export Mode
- Plugin-defined Modes

Changing mode should consistently update runtime capabilities without affecting unrelated editor services.

Atlas Studio should expose one unified Mode Manager abstraction.

# Responsibilities

The Mode Manager is responsible for:

- Registering modes.
- Activating modes.
- Tracking the active mode.
- Supporting plugin-defined modes.
- Validating mode transitions.
- Preserving deterministic runtime behavior.

The Mode Manager is **not** responsible for:

- Executing mode logic.
- Rendering.
- Processing input.
- Executing commands.
- Implementing business logic.

# Public API

```ts
interface ModeManager {
  readonly active: EditorMode | null;

  readonly modes: readonly EditorMode[];

  register(mode: EditorMode): void;

  unregister(id: string): void;

  activate(id: string): Promise<void>;

  deactivate(): Promise<void>;

  get(id: string): EditorMode | undefined;
}
```
