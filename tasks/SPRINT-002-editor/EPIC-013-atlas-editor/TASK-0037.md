# TASK-0037 — Implement Diagnostics Manager

## Summary

Implement the `DiagnosticsManager`, the runtime abstraction responsible for collecting, aggregating, classifying, and exposing diagnostics information across every Atlas Studio subsystem.

The Diagnostics Manager provides deterministic runtime diagnostics for editor services, workspaces, documents, rendering, AI execution, plugins, sessions, lifecycle coordination, event processing, and future runtime components while remaining independent from UI frameworks, rendering engines, operating systems, and storage providers.

The Diagnostics Manager is responsible only for collecting and exposing diagnostics. It does not modify runtime behavior, render diagnostics UI, or implement business logic.

---

# Goal

Provide a reusable, deterministic, immutable, and framework-independent Diagnostics Manager capable of exposing runtime diagnostics through a stable public API.

---

# Background

Professional creative applications require continuous runtime diagnostics.

Typical diagnostics include:

- Missing services
- Invalid lifecycle transitions
- Plugin failures
- Renderer warnings
- Session inconsistencies
- OCR failures
- AI execution errors
- Invalid configuration
- Performance degradation

Diagnostics should be collected centrally to simplify debugging and troubleshooting.

# Responsibilities

The Diagnostics Manager is responsible for:

- Collecting diagnostics.
- Categorizing diagnostics.
- Filtering diagnostics.
- Publishing immutable snapshots.
- Supporting runtime inspection.
- Preserving deterministic behavior.

The Diagnostics Manager is **not** responsible for:

- Recovering from failures.
- Rendering diagnostics.
- Uploading telemetry.
- Implementing business logic.
