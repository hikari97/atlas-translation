# TASK-0019 — Implement Tool Context

## Summary

Implement the `ToolContext`, the runtime abstraction responsible for exposing every resource, service, and execution environment required by an active Tool.

The Tool Context provides a deterministic execution environment shared by built-in tools, AI-powered tools, workflow tools, and plugin-defined tools. It exposes immutable access to the current Workspace, active Document, renderer, history, input, selection, translation, OCR, layout, export services, and future runtime capabilities while remaining completely independent from rendering engines, browser APIs, operating systems, and application frameworks.

The Tool Context is responsible only for exposing execution context. It does not execute tool logic, process user input, render graphics, or implement business logic.

---

# Goal

Provide a reusable, immutable, deterministic, and framework-independent Tool Context capable of supplying every runtime dependency required by an editor tool through a stable public API.

---

# Background

Every editor tool requires access to shared runtime resources.

Examples include:

- Active Workspace
- Active Document
- Renderer
- History
- Selection
- Input
- OCR
- Translation
- Layout
- Export

Without a unified Tool Context, every tool must resolve dependencies individually, increasing coupling and reducing extensibility.

Atlas Studio should expose one Tool Context abstraction.

# Responsibilities

The Tool Context is responsible for:

- Providing tool execution context.
- Exposing immutable runtime services.
- Supporting dependency injection.
- Supporting runtime inspection.
- Supporting future runtime modules.
- Preserving deterministic dependency resolution.

The Tool Context is **not** responsible for:

- Executing tools.
- Rendering.
- Processing input.
- Managing tool lifecycle.
- Implementing business logic.

# Public API

Example:

```ts
interface ToolContext {
  readonly workspace: Workspace;

  readonly document: Document | null;

  readonly services: ToolServices;

  readonly metadata: ToolContextMetadata;
}
```

Example service container:

```ts
interface ToolServices {
  readonly renderer: RendererPort;

  readonly history: HistoryPort;

  readonly selection: SelectionPort;

  readonly input: InputPort;
```
