# TASK-0020 — Implement Tool State

## Summary

Implement the `ToolState`, the immutable runtime abstraction responsible for representing the complete operational state of an editor Tool.

The Tool State represents every runtime aspect of a Tool, including lifecycle, activation state, execution state, configuration, temporary runtime data, AI execution state, asynchronous operations, progress information, cancellation state, diagnostics, and future tool extensions while remaining completely independent from rendering engines, browser APIs, operating systems, application frameworks, and editor implementations.

The Tool State is responsible only for representing runtime state. It does not execute tools, render graphics, process user input, or implement business logic.

---

# Goal

Provide a reusable, immutable, deterministic, and framework-independent Tool State abstraction capable of representing every runtime aspect of an editor tool through a stable public API.

---

# Background

Modern editor tools are no longer simple UI actions.

Examples include:

- Selection Tool
- Brush Tool
- Crop Tool
- OCR Tool
- AI Translation Tool
- Auto Layout Tool
- Inpainting Tool
- Batch Translation Tool
- Export Tool

Many of these tools execute asynchronously and require progress tracking, diagnostics, cancellation, and recovery.

Atlas Studio should expose one immutable Tool State abstraction.

# Responsibilities

The Tool State is responsible for:

- Representing runtime state.
- Tracking execution status.
- Exposing immutable snapshots.
- Supporting runtime inspection.
- Supporting future runtime extensions.
- Preserving deterministic runtime behavior.

The Tool State is **not** responsible for:

- Executing tools.
- Rendering.
- Processing input.
- Executing commands.
- Implementing business logic.

# Public API

Example:

```ts
interface ToolState {
  readonly lifecycle: ToolLifecycleState;

  readonly execution: ToolExecutionState;

  readonly progress: number;

  readonly isBusy: boolean;

  readonly isCancelled: boolean;

  readonly diagnostics: ToolDiagnostics | null;

  readonly snapshot: ToolStateSnapshot;
}
```
