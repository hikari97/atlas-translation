# TASK-0011 — Implement Active Document

## Summary

Implement the `ActiveDocument`, the runtime abstraction responsible for representing the currently focused document within a Workspace.

The Active Document provides a deterministic reference to the document currently being edited while coordinating with selection, history, renderer, input, translation, tools, and future runtime services. It exposes the active editing target without owning document data or implementing document behavior.

The Active Document is responsible only for exposing the currently active document reference. It does not load documents, save documents, render content, execute commands, or implement business logic.

---

# Goal

Provide a reusable, deterministic, immutable, and framework-independent Active Document abstraction capable of exposing the current editing target through a stable public API.

---

# Background

A Workspace may contain multiple documents.

Examples include:

- Manga Chapter
- Novel Chapter
- Image
- PSD Project
- SVG Document
- PDF
- Temporary OCR Result
- Translation Preview

Only one document should receive editing input at any given moment.

Atlas Studio should expose one Active Document abstraction.

# Responsibilities

The Active Document is responsible for:

- Representing the active editing target.
- Exposing immutable document references.
- Supporting runtime inspection.
- Supporting future document types.
- Preserving deterministic runtime behavior.

The Active Document is **not** responsible for:

- Opening documents.
- Closing documents.
- Saving documents.
- Rendering.
- Implementing document behavior.
- Implementing business logic.
