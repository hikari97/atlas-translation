# TASK-0032 — Implement Event Bus

## Summary

Implement the `EventBus`, the central abstraction responsible for dispatching, routing, subscribing, and coordinating runtime events across every Atlas Studio package.

The Event Bus provides a deterministic, asynchronous, and framework-independent communication mechanism between independent subsystems including editor, workspace, document, renderer, history, input, selection, translation, OCR, plugins, automation workflows, and future runtime modules.

The Event Bus is responsible only for event distribution. It does not execute business logic, render graphics, instantiate services, or coordinate runtime lifecycles.

---

# Goal

Provide a reusable, deterministic, strongly typed, framework-independent Event Bus capable of supporting all cross-package communication within Atlas Studio.

---

# Background

Atlas Studio is composed of numerous independent packages.

Without an Event Bus:

- packages become tightly coupled
- circular dependencies appear
- testing becomes difficult
- plugins become difficult to integrate

Every package should communicate through events rather than direct references whenever possible.

# Responsibilities

The Event Bus is responsible for:

- Publishing events.
- Dispatching events.
- Managing subscribers.
- Supporting event filtering.
- Supporting event priorities.
- Supporting future event transports.
- Preserving deterministic event ordering.

The Event Bus is **not** responsible for:

- Executing business logic.
- Persisting events.
- Network communication.
- Rendering.
- Service discovery.

# Public API

```ts
interface EventBus {
  publish<T extends Event>(event: T): Promise<void>;

  subscribe<T extends Event>(
    type: EventType,
    handler: EventHandler<T>,
  ): Disposable;

  unsubscribe(handler: EventHandler): void;

  hasSubscribers(type: EventType): boolean;
}
```

---

# Event Rules

The implementation should support:

- Strongly typed events.
- Immutable payloads.
- Deterministic ordering.
- Async handlers.
- Event filtering.
- Priority dispatch.
- Correlation tracking.

Publishing an event must never mutate the original event object.

# Design Principles

The Event Bus should be:

- Framework independent
- Strongly typed
- Lightweight
- Deterministic
- Extensible
- Testable

Events must represent facts that have already occurred.

Commands must never be implemented as events.
