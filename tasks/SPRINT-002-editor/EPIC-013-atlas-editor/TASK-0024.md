# TASK-0024 — Implement Mode Context

## Summary

Implement the `ModeContext`, the runtime abstraction responsible for exposing every dependency, policy, capability, and runtime service required by the currently active Editor Mode.

The Mode Context provides a deterministic execution environment for editor modes. It exposes immutable references to the current Workspace, active Document, active Tool, runtime services, policies, capability providers, keyboard profiles, workflow definitions, and future runtime extensions while remaining completely independent from rendering engines, browser APIs, operating systems, and application frameworks.

The Mode Context is responsible only for exposing runtime dependencies. It does not execute mode logic, process input, render graphics, or implement business logic.

---

# Goal

Provide a reusable, immutable, deterministic, and framework-independent Mode Context capable of supplying every dependency required by an Editor Mode through a stable public API.

---

# Background

An Editor Mode requires access to numerous runtime services.

Examples include:

- Workspace
- Active Document
- Active Tool
- Tool Manager
- Renderer
- History
- Selection
- Input
- Commands
- Translation
- OCR
- Plugin System

Passing every dependency individually increases coupling and makes plugin-defined modes difficult to implement.

Atlas Studio should expose one Mode Context abstraction.

# Responsibilities

The Mode Context is responsible for:

- Providing mode-scoped services.
- Providing runtime capabilities.
- Providing runtime policies.
- Supporting dependency injection.
- Supporting runtime inspection.
- Preserving deterministic dependency resolution.

The Mode Context is **not** responsible for:

- Executing modes.
- Switching modes.
- Rendering.
- Processing input.
- Implementing business logic.
