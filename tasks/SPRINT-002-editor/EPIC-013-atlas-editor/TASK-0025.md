# TASK-0025 — Implement Mode State

## Summary

Implement the `ModeState`, the immutable runtime abstraction responsible for representing the complete operational state of the currently active Editor Mode.

Mode State represents every runtime aspect of an editor mode including lifecycle state, activation state, applied policies, active capabilities, keyboard profile, workflow state, AI runtime state, runtime flags, diagnostics, and future mode extensions while remaining independent from rendering engines, browser APIs, operating systems, and application frameworks.

Mode State is responsible only for representing runtime state. It does not execute mode logic, process input, render graphics, or implement business logic.

---

# Goal

Provide a reusable, immutable, deterministic, and framework-independent Mode State abstraction capable of representing the complete runtime state of an Editor Mode through a stable public API.

---

# Background

Editor modes are dynamic runtime objects.

Examples include:

- Edit Mode
- OCR Mode
- Translation Mode
- Typesetting Mode
- Review Mode
- Export Mode
- Plugin-defined Modes

Each mode may expose different runtime capabilities, restrictions, workflow stages, and AI availability.

Atlas Studio should expose one immutable Mode State abstraction.

# Responsibilities

The Mode State is responsible for:

- Representing runtime state.
- Tracking active capabilities.
- Tracking applied policies.
- Exposing immutable snapshots.
- Supporting runtime inspection.
- Supporting future runtime extensions.

The Mode State is **not** responsible for:

- Executing mode logic.
- Switching modes.
- Rendering.
- Processing input.
- Implementing business logic.

# Public API

```ts
interface ModeState {
  readonly lifecycle: ModeLifecycleState;

  readonly capabilities: readonly Capability[];

  readonly policies: readonly Policy[];

  readonly isBusy: boolean;

  readonly isReadOnly: boolean;

  readonly diagnostics: ModeDiagnostics | null;

  readonly snapshot: ModeStateSnapshot;
}
```
