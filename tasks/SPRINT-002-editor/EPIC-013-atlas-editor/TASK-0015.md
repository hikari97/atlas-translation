# TASK-0015 — Implement Document Context

## Summary

Implement the `DocumentContext`, the runtime abstraction responsible for exposing every document-scoped dependency required while editing a document inside a Workspace.

The Document Context serves as the composition boundary for document-scoped runtime services. It exposes immutable references to renderer, history, selection, input, translation, OCR, layout, commands, plugins, export services, and future document-scoped providers without owning their lifecycle or implementation.

The Document Context is responsible only for exposing runtime dependencies. It does not create services, render graphics, execute commands, process input, or implement business logic.

---

# Goal

Provide a reusable, immutable, deterministic, and framework-independent Document Context capable of exposing every document-scoped runtime service through a stable public API.

---

# Background

Editing a document requires coordination across many independent services.

Examples include:

- Renderer
- History
- Selection
- Input
- Translation
- OCR
- Layout
- Commands
- Export
- Plugins

Passing every service individually throughout the editor introduces unnecessary coupling.

Atlas Studio should expose one Document Context abstraction.

# Responsibilities

The Document Context is responsible for:

- Exposing document-scoped services.
- Supporting dependency injection.
- Preserving immutable runtime references.
- Supporting runtime inspection.
- Supporting future runtime extensions.
- Maintaining deterministic dependency resolution.

The Document Context is **not** responsible for:

- Creating services.
- Managing service lifecycles.
- Rendering.
- Executing commands.
- Implementing business logic.

# Public API

Example:

```ts
interface DocumentContext {
  readonly metadata: DocumentContextMetadata;

  readonly services: DocumentServices;
}
```

Example service container:

```ts

  readonly renderer: RendererPort;

  readonly history: HistoryPort;

  readonly selection: SelectionPort;

  readonly input: InputPort;

  readonly ocr: OCRPort;

  readonly translation: TranslationPort;
```
