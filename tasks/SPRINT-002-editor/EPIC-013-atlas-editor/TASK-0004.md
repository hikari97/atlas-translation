# TASK-0004 — Implement Editor Context

## Summary

Implement the `EditorContext`, the dependency composition abstraction responsible for exposing every runtime service required by an Editor instance.

The Editor Context acts as the runtime composition root for Atlas Studio. It aggregates references to all editor-related services—including document management, renderer, history, input, commands, selection, plugins, workspaces, sessions, tools, and future runtime services—through a deterministic, immutable, and framework-independent interface.

The Editor Context is responsible only for providing runtime dependencies. It does not own their lifecycle, execute business logic, render interfaces, process input, or implement subsystem behavior.

---

# Goal

Provide a reusable, immutable, deterministic, and framework-independent dependency container capable of exposing every service required by an Editor instance through a stable public API.

---

# Background

An Editor coordinates many independent packages.

Examples include:

- Document Manager
- Renderer
- History
- Input
- Selection
- Commands
- Plugins
- Workspace
- Sessions
- Tool Manager

Without a unified context object, every subsystem would require numerous constructor parameters or global service lookups.

Atlas Studio should expose one Editor Context abstraction.

# Responsibilities

The Editor Context is responsible for:

- Providing runtime dependencies.
- Exposing runtime services.
- Supporting dependency injection.
- Preserving immutable runtime composition.
- Supporting future runtime extensions.
- Maintaining deterministic dependency resolution.

The Editor Context is **not** responsible for:

- Creating services.
- Owning service lifecycles.
- Rendering.
- Executing commands.
- Implementing business logic.

# Public API

Example:

```ts
interface EditorContext {
  readonly metadata: EditorContextMetadata;

  readonly services: EditorServices;

  has(service: string): boolean;

  resolve<T>(service: string): T;
}
```

Example service container:

```ts
interface EditorServices {
  readonly document: DocumentPort;

  readonly renderer: RendererPort;

  readonly history: HistoryPort;

  readonly input: InputPort;

  readonly selection: SelectionPort;

  readonly command: CommandPort;

  readonly plugin: PluginPort;
```
