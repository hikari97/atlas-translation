# TASK-0007 — Implement Snapshot Builder

## Summary

Implement the `SnapshotBuilder`, a reusable abstraction responsible for constructing immutable history snapshots within Atlas Studio.

The Snapshot Builder transforms application state into deterministic, immutable snapshot objects that can be consumed by the History Manager, transaction system, diagnostics, serialization, and future history extensions. It provides a consistent construction pipeline while remaining independent from rendering, persistence, editor implementations, and application frameworks.

The Snapshot Builder is responsible only for constructing immutable snapshot objects. It does not perform snapshot restoration, serialization, rendering, or business logic.

---

# Goal

Provide a reusable, extensible, and framework-independent snapshot builder capable of constructing immutable history snapshots through a stable public API.

---

# Background

History systems require a deterministic mechanism for converting application state into immutable snapshots.

Typical examples include:

- Document snapshot construction
- Selection snapshot construction
- Viewport snapshot construction
- Tool state snapshot construction
- Plugin-defined snapshot construction
- Transaction checkpoint generation

Instead of allowing every subsystem to construct snapshots independently, Atlas Studio should expose a unified Snapshot Builder abstraction.

# Responsibilities

The Snapshot Builder is responsible for:

- Constructing immutable snapshots.
- Producing deterministic snapshot structures.
- Attaching snapshot metadata.
- Supporting custom snapshot builders.
- Preserving snapshot consistency.

The Snapshot Builder is **not** responsible for:

- Restoring snapshots.
- Serializing snapshots.
- Rendering snapshots.
- Persisting snapshots.
- Business logic.
