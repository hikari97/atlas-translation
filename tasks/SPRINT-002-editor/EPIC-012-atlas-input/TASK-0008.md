# TASK-0008 — Implement Keyboard State

## Summary

Implement the `KeyboardState`, the immutable abstraction responsible for representing the current runtime state of keyboard input within Atlas Studio.

The Keyboard State abstraction provides a unified representation of all active keyboard information, including pressed keys, modifier keys, key repeat state, focus ownership, lock keys, and runtime metadata. It serves as the canonical source of truth for keyboard interactions while remaining completely independent from browser APIs, operating system event models, rendering engines, application frameworks, and editor implementations.

The Keyboard State abstraction is responsible only for representing keyboard state. It does not dispatch keyboard events, execute shortcuts, interpret commands, or implement business logic.

---

# Goal

Provide a reusable, immutable, and framework-independent keyboard state abstraction capable of representing the complete runtime keyboard state through a stable public API.

---

# Background

Professional editors require continuous knowledge of keyboard state rather than reacting solely to individual events.

Typical examples include:

- Currently pressed keys
- Active modifier keys
- Lock key status
- Repeat state
- Keyboard focus
- Active keyboard device
- Accessibility keyboard state
- Runtime metadata

Rather than allowing each subsystem to maintain its own keyboard state, Atlas Studio should expose a unified Keyboard State abstraction.

# Responsibilities

The Keyboard State abstraction is responsible for:

- Representing immutable keyboard state.
- Tracking pressed keys.
- Tracking modifier keys.
- Tracking lock keys.
- Providing runtime metadata.
- Supporting future keyboard providers.

The Keyboard State abstraction is **not** responsible for:

- Dispatching keyboard events.
- Executing shortcuts.
- Resolving key mappings.
- Rendering.
- Business logic.

# Public API

Example:

```ts
interface KeyboardState {
  readonly pressedKeys: ReadonlySet<string>;

  readonly modifiers: ModifierState;

  readonly repeat: boolean;

  readonly focused: boolean;

  readonly timestamp: number;

  readonly metadata: KeyboardStateMetadata;
}
```

Example metadata:

```ts
interface KeyboardStateMetadata {
  readonly capsLock: boolean;

  readonly numLock: boolean;

  readonly scrollLock: boolean;

  readonly activeDeviceId?: string;
}
```
