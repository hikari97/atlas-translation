# TASK-0022 — Implement Shortcut Context

## Summary

Implement the `ShortcutContext`, the abstraction responsible for representing logical shortcut scopes within Atlas Studio.

The Shortcut Context abstraction enables identical keyboard shortcuts to resolve differently depending on the active application context. It provides deterministic context-aware shortcut resolution for editors, canvases, inspectors, dialogs, overlays, command palettes, plugins, and future workspace modules while remaining completely independent from rendering engines, browser APIs, operating system implementations, application frameworks, and editor logic.

The Shortcut Context abstraction is responsible only for describing shortcut scopes. It does not execute commands, resolve shortcuts, dispatch keyboard events, render interfaces, or implement business logic.

---

# Goal

Provide a reusable, immutable, and framework-independent shortcut context abstraction capable of representing logical shortcut scopes through a stable public API.

---

# Background

Modern desktop applications contain multiple interactive areas that require different shortcut behavior.

Typical examples include:

- Document Editor
- Canvas
- Layer Panel
- Inspector
- Timeline
- Modal Dialog
- Context Menu
- Command Palette
- Plugin Window

For example:

```text
Ctrl+C

↓

Editor
→ Copy text

↓

Canvas
→ Copy selected objects

↓

Layer Panel
→ Duplicate layer

↓

Plugin
→ Plugin-defined action
```

Rather than hard-coding shortcut behavior into every subsystem, Atlas Studio should expose one unified Shortcut Context abstraction.

# Responsibilities

The Shortcut Context abstraction is responsible for:

- Representing logical shortcut scopes.
- Maintaining shortcut hierarchy.
- Tracking active shortcut context.
- Supporting nested shortcut contexts.
- Supporting plugin-defined contexts.

The Shortcut Context abstraction is **not** responsible for:

- Resolving shortcuts.
- Executing commands.
- Rendering.
- Business logic.
