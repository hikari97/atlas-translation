# TASK-0003 — Implement Input Router

## Summary

Implement the `InputRouter`, the central abstraction responsible for routing immutable input events from registered input devices to the appropriate consumers within Atlas Studio.

The Input Router provides deterministic event routing across keyboard, pointer, gesture, wheel, accessibility, and future input providers while preserving event order, routing context, and immutable event flow. It enables flexible routing strategies without introducing dependencies on rendering engines, application frameworks, editor implementations, or operating system APIs.

The Input Router abstraction is responsible only for routing input events. It does not generate input events, execute commands, recognize gestures, render interfaces, or implement business logic.

---

# Goal

Provide a reusable, deterministic, and framework-independent routing abstraction capable of delivering immutable input events through a stable public API.

---

# Background

Modern editing systems receive input from multiple devices simultaneously.

Typical examples include:

- Keyboard events
- Pointer events
- Wheel events
- Touch events
- Stylus events
- Accessibility devices
- Virtual input providers
- Plugin-defined input devices

Instead of allowing each subsystem to consume raw events independently, Atlas Studio should expose a unified Input Router abstraction.

# Responsibilities

The Input Router abstraction is responsible for:

- Routing immutable input events.
- Maintaining deterministic routing order.
- Resolving active routes.
- Supporting context-aware routing.
- Supporting future routing providers.

The Input Router abstraction is **not** responsible for:

- Producing input events.
- Executing commands.
- Recognizing gestures.
- Rendering.
- Business logic.
