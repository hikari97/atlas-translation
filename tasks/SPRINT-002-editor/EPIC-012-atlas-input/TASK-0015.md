# TASK-0015 — Implement Pointer State

## Summary

Implement the `PointerState`, the immutable abstraction responsible for representing the complete runtime state of pointer interactions within Atlas Studio.

The Pointer State abstraction provides a unified representation of all active pointer information, including active pointers, pointer positions, pressed buttons, pointer capture ownership, pressure, tilt, hover state, contact state, and runtime metadata. It serves as the canonical source of truth for every pointer-based subsystem while remaining completely independent from browser APIs, operating system event models, rendering engines, application frameworks, and editor implementations.

The Pointer State abstraction is responsible only for representing pointer state. It does not dispatch pointer events, recognize gestures, manipulate documents, render interfaces, or implement business logic.

---

# Goal

Provide a reusable, immutable, and framework-independent pointer state abstraction capable of representing the complete runtime pointer state through a stable public API.

---

# Background

Professional editing software continuously tracks pointer state rather than responding only to isolated events.

Typical examples include:

- Active pointer positions
- Pressed buttons
- Hover state
- Pointer capture ownership
- Stylus pressure
- Tilt information
- Contact state
- Active pointer device
- Runtime metadata

Rather than allowing individual subsystems to maintain separate pointer state, Atlas Studio should expose one unified Pointer State abstraction.

# Responsibilities

The Pointer State abstraction is responsible for:

- Representing immutable pointer state.
- Tracking active pointers.
- Tracking button states.
- Tracking pointer capture ownership.
- Tracking pressure and tilt information.
- Supporting future pointer providers.

The Pointer State abstraction is **not** responsible for:

- Dispatching pointer events.
- Recognizing gestures.
- Executing commands.
- Rendering.
- Business logic.

# Public API

Example:

```ts
interface PointerState {
  readonly activePointers: readonly ActivePointer[];

  readonly capturedPointers: ReadonlyMap<number, string>;

  readonly primaryPointer: ActivePointer | null;

  readonly timestamp: number;

  readonly metadata: PointerStateMetadata;
}
```

Example active pointer:

```ts
interface ActivePointer {
  readonly id: number;

  readonly type: 'mouse' | 'touch' | 'pen' | 'custom';

  readonly position: Point2D;

  readonly buttons: number;

  readonly pressure: number;

  readonly tiltX: number;

  readonly tiltY: number;
}
```

Example metadata:

```ts
interface PointerStateMetadata {
  readonly hovering: boolean;

  readonly activeCaptureCount: number;

  readonly activeDeviceId?: string;
}
```

# Design Principles

The Pointer State abstraction should be:

- Framework independent
- Immutable
- Strongly typed
- Lightweight
- Deterministic
- Extensible

State snapshots should remain deterministic regardless of pointer hardware or operating system.
