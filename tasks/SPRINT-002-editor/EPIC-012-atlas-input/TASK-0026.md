# TASK-0026 — Implement Input Mapping

## Summary

Implement the `InputMapping`, the central abstraction responsible for translating normalized physical input into logical application actions throughout Atlas Studio.

The Input Mapping abstraction receives normalized input from keyboard, pointer, touch, stylus, wheel, accessibility devices, gamepads, and future input providers. It translates device-specific interactions into device-independent logical actions that can be consumed consistently by Atlas Studio packages without requiring knowledge of the originating hardware.

The Input Mapping abstraction is responsible only for translating input into logical actions. It does not execute commands, modify application state, render interfaces, or implement business logic.

---

# Goal

Provide a reusable, deterministic, immutable, and framework-independent input mapping abstraction capable of translating every supported physical input into logical actions through a stable public API.

---

# Background

Professional editing applications should separate **physical input** from **logical behavior**.

Examples:

```text
Ctrl + Z

↓

Keyboard

↓

Undo
```

```text
Two Finger Swipe Left

↓

Trackpad

↓

Undo
```

```text
Stylus Button + Drag

↓

Stylus

↓

Pan Canvas
```

```text
Middle Mouse Button

↓

Mouse

↓

Pan Canvas
```

Although the physical interactions differ, they should produce the same logical action.

Atlas Studio should expose a unified Input Mapping abstraction to eliminate device-specific logic throughout the application.

# Responsibilities

The Input Mapping abstraction is responsible for:

- Translating physical input.
- Producing logical actions.
- Supporting multiple input devices.
- Supporting runtime mapping updates.
- Supporting future mapping providers.
- Preserving deterministic translation.

The Input Mapping abstraction is **not** responsible for:

- Executing actions.
- Rendering.
- Editing documents.
- Business logic.

The Input Mapping abstraction should be:

- Framework independent
- Immutable
- Strongly typed
- Lightweight
- Deterministic
- Extensible

The same input under identical runtime conditions should always produce the same logical action.
