# TASK-0019 — Implement Rotate Recognition

## Summary

Implement the `RotateRecognition`, the abstraction responsible for detecting multi-pointer rotation gestures within Atlas Studio.

The Rotate Recognition abstraction analyzes normalized pointer events from two or more active pointers to determine rotational movement around a shared center point. It provides deterministic recognition for touchscreens, precision trackpads, styluses, pen tablets, and future multi-pointer devices while remaining completely independent from rendering engines, browser APIs, operating system gesture APIs, application frameworks, and editor implementations.

The Rotate Recognition abstraction is responsible only for recognizing rotation gestures. It does not rotate documents, modify viewports, render interfaces, execute commands, or implement business logic.

---

# Goal

Provide a reusable, deterministic, and framework-independent rotation recognition abstraction capable of recognizing multi-pointer rotation gestures through a stable public API.

---

# Background

Many professional creative applications support rotation gestures.

Typical examples include:

- Canvas rotation
- Object rotation
- Image manipulation
- Diagram rotation
- CAD navigation
- Touchscreen rotation
- Precision trackpad rotation
- Plugin-defined rotation gestures

Rather than allowing each subsystem to detect rotation independently, Atlas Studio should expose a unified Rotate Recognition abstraction.

# Responsibilities

The Rotate Recognition abstraction is responsible for:

- Detecting rotation gestures.
- Tracking active rotation.
- Calculating rotation angle.
- Calculating rotation center.
- Supporting future rotation providers.

The Rotate Recognition abstraction is **not** responsible for:

- Rotating objects.
- Rotating viewports.
- Rendering gesture feedback.
- Executing commands.
- Business logic.
