# TASK-0037 — Implement Performance Profiler

## Summary

Implement the `PerformanceProfiler`, the abstraction responsible for profiling execution behavior throughout the Atlas Studio input subsystem.

The Performance Profiler provides deterministic runtime profiling for every major component of the input pipeline, including keyboard processing, pointer processing, gesture recognition, shortcut resolution, event dispatching, queue processing, and future input providers. It measures execution duration, call frequency, nested operations, and profiling sessions while remaining completely independent from rendering engines, browser APIs, operating system profiling tools, application frameworks, and editor implementations.

The Performance Profiler is responsible only for collecting execution profiling data. It does not optimize execution automatically, execute commands, render interfaces, or implement business logic.

---

# Goal

Provide a reusable, deterministic, immutable, and framework-independent profiling abstraction capable of analyzing runtime execution behavior through a stable public API.

---

# Background

Performance metrics alone cannot identify execution bottlenecks.

Typical profiling targets include:

- Input Manager
- Keyboard Manager
- Pointer Manager
- Gesture Manager
- Shortcut Resolver
- Event Dispatcher
- Event Queue
- Plugin input providers

Developers require execution traces to determine where processing time is spent.

Atlas Studio should expose one unified Performance Profiler abstraction.

# Responsibilities

The Performance Profiler is responsible for:

- Measuring execution duration.
- Recording nested scopes.
- Aggregating execution statistics.
- Producing immutable profile sessions.
- Supporting future profiling providers.
- Preserving deterministic measurements.

The Performance Profiler is **not** responsible for:

- Optimizing runtime performance.
- Executing commands.
- Rendering.
- Business logic.
