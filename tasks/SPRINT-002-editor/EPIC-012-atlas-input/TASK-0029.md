# TASK-0029 — Implement Device Mapping

## Summary

Implement the `DeviceMapping`, the abstraction responsible for translating device-specific capabilities into normalized input actions throughout Atlas Studio.

The Device Mapping abstraction provides a unified mechanism for associating physical input devices—including keyboards, mice, touchscreens, styluses, trackpads, gamepads, accessibility devices, and future custom hardware—with normalized device capabilities used by the Input Mapping subsystem. It isolates hardware-specific behavior from logical application behavior while remaining completely independent from rendering engines, browser APIs, operating system implementations, application frameworks, and editor logic.

The Device Mapping abstraction is responsible only for describing device mappings. It does not dispatch input events, execute commands, recognize gestures, or implement business logic.

---

# Goal

Provide a reusable, immutable, deterministic, and framework-independent device mapping abstraction capable of translating hardware capabilities into normalized device representations through a stable public API.

---


```text

Single Touch

```

```text

↓

```

Rather than exposing these differences throughout Atlas Studio, every device should be normalized through a unified Device Mapping abstraction.

# Responsibilities

The Device Mapping abstraction is responsible for:

- Mapping physical devices.
- Normalizing device capabilities.
- Associating devices with profiles.
- Supporting runtime registration.
- Supporting future device providers.
- Preserving deterministic mapping behavior.

The Device Mapping abstraction is **not** responsible for:

- Dispatching events.
- Executing commands.
- Rendering.
- Business logic.

# Public API

Example:

```ts
interface DeviceMapping {
  register(profile: DeviceProfile): void;

  unregister(id: string): void;

  resolve(device: InputDevice): DeviceProfile | null;

  list(): readonly DeviceProfile[];
}
```

Example device profile:

```ts
interface DeviceProfile {
  readonly id: string;

  readonly deviceType: InputDeviceType;

  readonly capabilities: readonly DeviceCapability[];

  readonly enabled: boolean;
```
