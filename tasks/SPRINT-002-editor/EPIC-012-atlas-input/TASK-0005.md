# TASK-0005 — Implement Input State

## Summary

Implement the `InputState`, the central abstraction responsible for representing the immutable runtime state of the Atlas Studio input subsystem.

The Input State abstraction provides a unified representation of the current input environment, including registered devices, active contexts, routing status, focus ownership, and interaction metadata. It serves as the canonical source of truth for the atlas-input package while remaining completely independent from rendering engines, application frameworks, operating system APIs, and editor implementations.

The Input State abstraction is responsible only for describing the current input state. It does not generate events, dispatch input, recognize gestures, execute shortcuts, or implement business logic.

---

# Goal

Provide a reusable, immutable, and framework-independent state abstraction capable of representing the complete runtime status of the input subsystem through a stable public API.

---

# Background

Every input system requires a consistent representation of its current state.

Typical examples include:

- Connected devices
- Active device
- Active input context
- Focus owner
- Pointer capture
- Current modifier keys
- Active gesture
- Current routing state

Instead of allowing each subsystem to maintain its own mutable state, Atlas Studio should expose a unified Input State abstraction.

# Responsibilities

The Input State abstraction is responsible for:

- Representing immutable runtime state.
- Tracking active input devices.
- Tracking active input contexts.
- Maintaining routing metadata.
- Preserving focus information.
- Supporting future state providers.

The Input State abstraction is **not** responsible for:

- Dispatching events.
- Executing shortcuts.
- Recognizing gestures.
- Rendering interfaces.
- Business logic.

# Public API

Example:

```ts
interface InputState {
  readonly devices: readonly InputDevice[];

  readonly activeContext: InputContext | null;

  readonly focusedDevice: InputDevice | null;

  readonly timestamp: number;

  readonly metadata: InputStateMetadata;
}
```

Example metadata:

```ts
interface InputStateMetadata {
  readonly routingEnabled: boolean;

  readonly activePointers: number;

  readonly activeModifiers: readonly string[];
}
```
