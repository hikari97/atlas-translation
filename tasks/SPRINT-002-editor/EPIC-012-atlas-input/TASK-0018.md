# TASK-0018 — Implement Pinch Recognition

## Summary

Implement the `PinchRecognition`, the abstraction responsible for detecting multi-pointer pinch interactions within Atlas Studio.

The Pinch Recognition abstraction analyzes normalized pointer events from two or more active pointers to determine pinch gestures used for scaling, zooming, and other distance-based interactions. It provides deterministic recognition for touchscreens, trackpads, styluses, and future multi-pointer devices while remaining completely independent from rendering engines, application frameworks, browser APIs, operating system gesture APIs, and editor implementations.

The Pinch Recognition abstraction is responsible only for recognizing pinch gestures. It does not zoom viewports, modify documents, render interfaces, execute commands, or implement business logic.

---

# Goal

Provide a reusable, deterministic, and framework-independent pinch recognition abstraction capable of recognizing multi-pointer pinch gestures through a stable public API.

---

# Background

Pinch gestures are fundamental to modern touch-enabled applications.

Typical examples include:

- Canvas zoom
- Image scaling
- Viewport scaling
- Timeline zoom
- Diagram scaling
- Touchpad pinch gestures
- Multi-touch interactions
- Plugin-defined scaling gestures

Rather than allowing every subsystem to detect pinch gestures independently, Atlas Studio should expose a unified Pinch Recognition abstraction.

# Responsibilities

The Pinch Recognition abstraction is responsible for:

- Detecting pinch gestures.
- Tracking active touch points.
- Calculating scale factors.
- Maintaining gesture lifecycle.
- Supporting future multi-pointer providers.

The Pinch Recognition abstraction is **not** responsible for:

- Zooming viewports.
- Scaling documents.
- Rendering gesture feedback.
- Executing commands.
- Business logic.
