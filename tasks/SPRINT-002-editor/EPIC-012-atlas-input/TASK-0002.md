# TASK-0002 — Implement Input Device

## Summary

Implement the `InputDevice`, the foundational abstraction responsible for representing every physical and virtual input device supported by Atlas Studio.

The Input Device abstraction provides a unified interface for keyboards, mice, touchscreens, styluses, trackpads, gamepads, accessibility devices, and future plugin-defined input sources. It encapsulates device identity, capabilities, lifecycle, metadata, and state while remaining completely independent from rendering engines, application frameworks, operating system APIs, and editor implementations.

The Input Device abstraction is responsible only for describing input devices. It does not dispatch events, interpret gestures, execute shortcuts, or implement business logic.

---

# Goal

Provide a reusable, immutable, and framework-independent input device abstraction capable of representing every supported input source through a stable public API.

---

# Background

Modern editing environments interact with many different hardware and virtual input devices.

Typical examples include:

- Keyboard
- Mouse
- Trackpad
- Touchscreen
- Stylus
- Pen tablet
- Wheel devices
- Gamepad
- Assistive technologies
- Plugin-defined devices

Rather than exposing platform-specific implementations, Atlas Studio should provide a single device abstraction for all input sources.

# Responsibilities

The Input Device abstraction is responsible for:

- Representing immutable device information.
- Describing device capabilities.
- Managing lifecycle metadata.
- Providing device identifiers.
- Supporting future device extensions.

The Input Device abstraction is **not** responsible for:

- Dispatching events.
- Routing input.
- Executing shortcuts.
- Rendering.
- Business logic.

# Public API

Example:

```ts
interface InputDevice {
  readonly id: string;

  readonly type: InputDeviceType;

  readonly state: InputDeviceState;

  readonly capabilities: InputDeviceCapabilities;

  readonly metadata: Readonly<Record<string, unknown>>;
}
```

Example device types:

```ts
type InputDeviceType =
  | 'keyboard'
  | 'mouse'
  | 'touch'
  | 'stylus'
  | 'trackpad'
  | 'wheel'
  | 'gamepad'
  | 'accessibility'
  | 'custom';
```
