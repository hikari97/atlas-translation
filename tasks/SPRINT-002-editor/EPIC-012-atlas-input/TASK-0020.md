# TASK-0020 — Implement Gesture State

## Summary

Implement the `GestureState`, the immutable abstraction responsible for representing the complete runtime state of gesture interactions within Atlas Studio.

The Gesture State abstraction provides a unified representation of all active gesture information, including active gestures, gesture lifecycle, participating pointers, gesture metrics, recognition confidence, and runtime metadata. It serves as the canonical source of truth for every gesture-related subsystem while remaining completely independent from rendering engines, browser APIs, operating system gesture APIs, application frameworks, and editor implementations.

The Gesture State abstraction is responsible only for representing gesture state. It does not recognize gestures, manipulate documents, execute commands, render interfaces, or implement business logic.

---

# Goal

Provide a reusable, immutable, and framework-independent gesture state abstraction capable of representing the complete runtime gesture state through a stable public API.

---

# Background

Professional editing applications continuously maintain gesture state while users interact with touch devices, styluses, or multiple pointers.

Typical examples include:

- Active drag
- Active pinch
- Active rotation
- Gesture duration
- Participating pointers
- Gesture confidence
- Gesture cancellation
- Runtime metadata

Rather than allowing individual recognizers to maintain isolated mutable state, Atlas Studio should expose one unified Gesture State abstraction.

# Responsibilities

The Gesture State abstraction is responsible for:

- Representing immutable gesture state.
- Tracking active gestures.
- Tracking participating pointers.
- Preserving gesture lifecycle.
- Supporting future gesture providers.

The Gesture State abstraction is **not** responsible for:

- Detecting gestures.
- Dispatching gesture events.
- Rendering.
- Executing commands.
- Business logic.

# Public API

Example:

```ts
interface GestureState {
  readonly activeGestures: readonly ActiveGesture[];

  readonly primaryGesture: ActiveGesture | null;

  readonly timestamp: number;

  readonly metadata: GestureStateMetadata;
}
```

Example active gesture:

```ts
interface ActiveGesture {
  readonly id: string;

  readonly type: GestureType;

  readonly state: GestureLifecycleState;

  readonly pointerIds: readonly number[];

  readonly startedAt: number;
}
```

Example metadata:

```ts
interface GestureStateMetadata {
  readonly activeCount: number;

  readonly recognizerCount: number;

  readonly lastUpdated: number;
}
```

# Design Principles

The Gesture State abstraction should be:

- Framework independent
- Immutable
- Strongly typed
- Lightweight
- Deterministic
- Extensible

State snapshots should remain deterministic regardless of hardware, browser, or operating system.
