# TASK-0032 — Implement Event Dispatcher

## Summary

Implement the `EventDispatcher`, the central abstraction responsible for delivering normalized input events throughout Atlas Studio.

The Event Dispatcher receives immutable input events and coordinates deterministic delivery to registered listeners, subscribers, plugins, and future event consumers. It supports event propagation, listener priority, event cancellation, propagation control, and event filtering while remaining completely independent from rendering engines, browser APIs, operating system event systems, application frameworks, and editor implementations.

The Event Dispatcher is responsible only for delivering events. It does not generate events, execute commands, modify application state, render interfaces, or implement business logic.

---

# Goal

Provide a reusable, deterministic, immutable, and framework-independent event dispatching abstraction capable of distributing every supported input event through a stable public API.

---

# Background

Modern applications contain numerous components interested in the same input event.

Typical consumers include:

- Editor
- Canvas
- Viewport
- History
- Selection
- Plugins
- Accessibility
- Developer tools

Instead of tightly coupling producers and consumers, Atlas Studio should distribute events through a unified Event Dispatcher abstraction.

# Responsibilities

The Event Dispatcher is responsible for:

- Dispatching events.
- Managing listeners.
- Preserving listener priority.
- Supporting propagation control.
- Supporting future event consumers.
- Preserving deterministic dispatch order.

The Event Dispatcher is **not** responsible for:

- Generating events.
- Queueing events.
- Executing commands.
- Rendering.
- Business logic.

# Dispatch Rules

The implementation should support:

- Listener priorities
- Propagation cancellation
- Event filtering
- Listener isolation
- Multiple listeners
- Plugin listeners

---

# Design Principles

The Event Dispatcher should be:

- Framework independent
- Immutable event objects
- Strongly typed
- Lightweight
- Deterministic
- Extensible

The same event should always be dispatched in the same order under identical runtime conditions.
