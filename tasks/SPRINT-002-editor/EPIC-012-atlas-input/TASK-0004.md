# TASK-0004 — Implement Input Context

## Summary

Implement the `InputContext`, the abstraction responsible for representing logical input scopes within Atlas Studio.

The Input Context abstraction provides deterministic context-aware input routing by defining where and when input should be interpreted. It enables editors, dialogs, overlays, plugins, inspectors, and future application modules to receive input according to the currently active context while remaining completely independent from rendering engines, operating system APIs, application frameworks, and editor implementations.

The Input Context abstraction is responsible only for describing logical input contexts. It does not route events, execute shortcuts, recognize gestures, render interfaces, or implement business logic.

---

# Goal

Provide a reusable, immutable, and framework-independent context abstraction capable of representing logical input scopes through a stable public API.

---

# Background

Modern editors often contain multiple interactive regions.

Typical examples include:

- Document editor
- Canvas
- Inspector panel
- Layer panel
- Modal dialog
- Context menu
- Command palette
- Plugin window

The same keyboard shortcut or pointer interaction may produce different behavior depending on which context is active.

Atlas Studio should expose a unified Input Context abstraction to isolate input behavior from application layout.

# Responsibilities

The Input Context abstraction is responsible for:

- Representing logical input scopes.
- Maintaining context hierarchy.
- Managing active context metadata.
- Supporting nested contexts.
- Supporting future context providers.

The Input Context abstraction is **not** responsible for:

- Routing events.
- Executing shortcuts.
- Rendering interfaces.
- Managing focus.
- Business logic.
