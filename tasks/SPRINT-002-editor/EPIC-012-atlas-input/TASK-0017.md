# TASK-0017 — Implement Drag Recognition

## Summary

Implement the `DragRecognition`, the abstraction responsible for detecting and managing drag interactions within Atlas Studio.

The Drag Recognition abstraction analyzes normalized pointer events to determine when a drag interaction begins, progresses, and ends. It provides deterministic drag recognition for object manipulation, selection, resizing, canvas interaction, node movement, and future drag-based workflows while remaining completely independent from rendering engines, application frameworks, browser APIs, operating system gesture APIs, and editor implementations.

The Drag Recognition abstraction is responsible only for recognizing drag gestures. It does not modify documents, manipulate viewports, render interfaces, execute commands, or implement business logic.

---

# Goal

Provide a reusable, deterministic, and framework-independent drag recognition abstraction capable of recognizing drag gestures through a stable public API.

---

# Background

Dragging is one of the most common interactions in graphical editing applications.

Typical examples include:

- Object movement
- Canvas panning
- Node repositioning
- Resize handles
- Selection boxes
- Layer reordering
- Drag and drop
- Plugin-defined drag interactions

Rather than allowing every subsystem to detect drag independently, Atlas Studio should expose a unified Drag Recognition abstraction.

# Responsibilities

The Drag Recognition abstraction is responsible for:

- Detecting drag initiation.
- Tracking active drags.
- Determining drag completion.
- Supporting configurable thresholds.
- Preserving deterministic drag recognition.

The Drag Recognition abstraction is **not** responsible for:

- Moving objects.
- Rendering drag previews.
- Executing commands.
- Editing documents.
- Business logic.
