# TASK-0004 — Implement Theme Tokens

## Summary

Implement the Theme Tokens subsystem for Atlas UI.

Theme Tokens define the design primitives used throughout Atlas Studio. They provide a consistent vocabulary for colors, typography, spacing, sizing, borders, shadows, animation, and elevation without coupling components to concrete styling implementations.

Theme Tokens are consumed by the Theme System and should never contain rendering or component logic.

---

# Goal

Provide a strongly typed, immutable, and extensible collection of design tokens that can be shared across all Atlas UI components.

---

# Background

Directly embedding visual values inside components leads to inconsistency and makes customization difficult.

By introducing Theme Tokens, every visual decision is centralized, allowing themes to change appearance without modifying component implementations.

# Responsibilities

Theme Tokens are responsible for:

- Defining reusable design primitives.
- Providing immutable token values.
- Supporting multiple themes.
- Standardizing visual values.
- Serving as the foundation for component styling.

Theme Tokens are **not** responsible for:

- Theme management.
- Rendering.
- Component behavior.
- Layout calculation.
- Accessibility logic.

# Public API

Example:

```ts
interface ThemeTokens {
  readonly colors: ColorTokens;

  readonly typography: TypographyTokens;

  readonly spacing: SpacingTokens;

  readonly radius: RadiusTokens;

  readonly shadows: ShadowTokens;

  readonly motion: MotionTokens;
}
```

---

# Design Principles

Theme Tokens should be:

- Immutable
- Strongly typed
- Framework independent
- Reusable
- Versionable

Components should consume tokens rather than hard-coded values.

---

# Technical Requirements

The implementation must:

- Avoid duplicate token definitions.
- Support future token categories.
- Preserve API stability.
- Allow theme overrides.
- Compile under TypeScript strict mode.
