# TASK-0021 — Implement Caret Component

## Summary

Implement the Caret Component for Atlas UI.

The Caret Component represents the visual insertion point inside the editor. It integrates with Atlas Renderer, Atlas Document, UI Context, Accessibility Layer, and Editor View while remaining independent from any rendering framework.

The Caret Component is responsible only for visualization and interaction feedback. Cursor movement logic and document editing remain outside this component.

---

# Goal

Provide a reusable, renderer-independent Caret component capable of accurately representing the current insertion position.

---

# Background

The caret is one of the most fundamental editor components.

It must remain synchronized with:

- Document Position
- Renderer Layout
- Selection State
- Viewport
- Input Events

The Caret should render smoothly without affecting document rendering performance.

# Responsibilities

The Caret Component is responsible for:

- Displaying the insertion point.
- Synchronizing with renderer layout.
- Managing blink state.
- Tracking viewport position.
- Supporting accessibility.

The Caret Component is **not** responsible for:

- Editing text.
- Document navigation.
- Selection rendering.
- Input processing.
- Business logic.

# Design Principles

The Caret Component should be:

- Lightweight
- Renderer-independent
- Accessible
- Strongly typed
- Extensible

The Caret should never modify the document.

---

# Technical Requirements

The implementation must:

- Support high-frequency position updates.
- Support viewport scrolling.
- Preserve immutable props.
- Avoid hard-coded rendering.
- Preserve API stability.

---

# Accessibility Requirements

The Caret must:

- Reflect keyboard focus.
- Integrate with screen readers.
- Respect accessibility settings.
- Support high-contrast themes.
- Respect reduced-motion preferences.
