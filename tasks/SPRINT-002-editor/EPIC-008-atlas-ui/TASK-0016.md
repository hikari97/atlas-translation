# TASK-0016 — Implement Tooltip Component

## Summary

Implement the Tooltip Component for Atlas UI.

The Tooltip Component provides contextual information when users interact with UI elements through hover, focus, or keyboard navigation. It integrates with the Theme System, Overlay System, Accessibility Layer, and UI Context while remaining framework-independent.

Tooltip behavior should be lightweight, non-blocking, and entirely presentation-oriented.

---

# Goal

Provide an accessible, reusable, and extensible Tooltip component that delivers contextual assistance without interrupting user workflows.

---

# Background

Tooltips are used extensively throughout Atlas Studio to explain:

- Toolbar actions
- Icons
- Buttons
- Form controls
- Menu items
- Keyboard shortcuts
- Plugin actions

A unified Tooltip abstraction ensures consistent behavior across the entire application.

# Responsibilities

The Tooltip Component is responsible for:

- Displaying contextual information.
- Managing tooltip visibility.
- Supporting keyboard interaction.
- Supporting pointer interaction.
- Integrating with Theme System.
- Providing accessibility metadata.

The Tooltip Component is **not** responsible for:

- Overlay positioning engine.
- Modal interaction.
- Business logic.
- Application navigation.
- Animation implementation.

---

# Supported Placements

The component should support:

- Top
- Bottom
- Left
- Right
- Top Start
- Top End
- Bottom Start
- Bottom End

Future placements should remain backward compatible.

---

# Supported States

The component should support:

- Hidden
- Visible
- Delayed
- Disabled

# Public API

Example:

```ts
interface Tooltip extends UIComponent {

```

Example props:

```ts
interface TooltipProps {
  readonly content: UIElement;

  readonly placement: TooltipPlacement;

  readonly delay?: number;

  readonly disabled?: boolean;
}
```

---

# Design Principles

The Tooltip Component should be:

- Accessible
- Lightweight
- Theme-aware
- Framework independent
- Strongly typed

Tooltips should never receive keyboard focus directly.

---

# Technical Requirements

The implementation must:

- Support configurable delays.
- Support keyboard focus.
- Preserve immutable props.
- Avoid hard-coded styling.
- Preserve API stability.

---

# Accessibility Requirements

The Tooltip must:

- Associate itself with its trigger element.
- Support screen readers.
- Not trap keyboard focus.
- Be dismissible.
- Respect reduced-motion preferences.
