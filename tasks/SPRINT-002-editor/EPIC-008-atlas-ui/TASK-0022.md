# TASK-0022 — Implement Selection UI

## Summary

Implement the Selection UI for Atlas UI.

The Selection UI provides the visual representation of document selections within Atlas Studio. It integrates with Atlas Renderer, Atlas Document, Editor View, Accessibility Layer, and UI Context while remaining independent from any rendering framework.

The Selection UI is responsible only for visualizing selection ranges. Selection logic and document manipulation remain outside this component.

---

# Goal

Provide a reusable, accessible, and renderer-independent Selection UI capable of visualizing single and multiple selection ranges.

---

# Background

Selection is a fundamental part of every editor.

The visual representation must remain synchronized with:

- Document Selection
- Renderer Layout
- Viewport
- Caret
- Scrolling
- Zoom

Selection rendering must remain performant even for large documents.

# Responsibilities

The Selection UI is responsible for:

- Displaying selection highlights.
- Rendering multiple selection ranges.
- Synchronizing with renderer layout.
- Tracking viewport movement.
- Supporting accessibility.

The Selection UI is **not** responsible for:

- Creating selections.
- Editing documents.
- Clipboard operations.
- Caret movement.
- Business logic.

# Design Principles

The Selection UI should be:

- Lightweight
- Renderer-independent
- Accessible
- Strongly typed
- Extensible

Selection rendering should never modify the document.
