# TASK-0036 — Implement Split View

## Summary

Implement the Split View component for Atlas UI.

The Split View provides a reusable layout mechanism that allows multiple editor views or workspace panels to be displayed simultaneously. It integrates with the Workspace Layout, Docking System, Panel Manager, Theme System, Accessibility Layer, and Layout Services while remaining framework-independent.

The Split View is responsible only for arranging resizable panes. It must not manage editor state or document synchronization.

---

# Goal

Provide a reusable, accessible, and extensible Split View capable of supporting horizontal and vertical workspace partitioning through a stable public API.

---

# Background

Split views are fundamental to professional IDEs and editors.

Typical scenarios include:

- Side-by-side editors
- Vertical editor split
- Horizontal editor split
- Preview panel
- Diff viewer
- Plugin-defined layouts

Atlas UI should expose a generic Split View abstraction reusable by all workspace components.

# Responsibilities

The Split View is responsible for:

- Arranging panes.
- Supporting nested layouts.
- Managing divider interaction.
- Integrating with Theme System.
- Providing accessibility metadata.

The Split View is **not** responsible for:

- Docking panels.
- Synchronizing documents.
- Managing editor lifecycle.
- Workspace persistence.
- Business logic.

# Public API

Example:

```ts
interface SplitView extends UIComponent {
  readonly props: SplitViewProps;

  split(direction: SplitDirection): void;

  collapse(paneId: string): void;

  expand(paneId: string): void;
}
```

Example pane:

```ts
interface SplitPane {
  readonly id: string;

  readonly size: number;

  readonly minimumSize?: number;

  readonly maximumSize?: number;
}
```
