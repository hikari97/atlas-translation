# TASK-0030 — Implement Editor Status Bar

## Summary

Implement the Editor Status Bar component for Atlas UI.

The Editor Status Bar provides a persistent interface for displaying editor state, document information, cursor position, language mode, zoom level, background tasks, and plugin status. It integrates with the Editor View, Theme System, UI Context, Accessibility Layer, and Status Services while remaining framework-independent.

The Status Bar is responsible only for presenting status information. It must not own or modify application state.

---

# Goal

Provide a reusable, extensible, and accessible Status Bar capable of presenting editor and workspace status through a stable public API.

---

# Background

Status bars are a standard component in modern editors.

Typical information includes:

- Cursor Position
- Selection Information
- Document Encoding
- Line Ending
- Language Mode
- Zoom Level
- Read-only Status
- Plugin Status
- Background Tasks

Rather than embedding status widgets into the editor, Atlas UI should expose a unified Status Bar abstraction.

# Responsibilities

The Status Bar is responsible for:

- Displaying editor status.
- Organizing status items.
- Supporting dynamic providers.
- Integrating with Theme System.
- Providing accessibility metadata.

The Status Bar is **not** responsible for:

- Computing status values.
- Executing background tasks.
- Managing plugins.
- Updating documents.
- Business logic.

# Design Principles

The Status Bar should be:

- Lightweight
- Accessible
- Theme-aware
- Framework independent
- Strongly typed

Status information should always originate from providers rather than internal state.

The implementation must:

- Respect accessibility settings.
