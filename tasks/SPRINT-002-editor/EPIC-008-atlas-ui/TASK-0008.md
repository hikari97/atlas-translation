# TASK-0008 — Implement Checkbox Component

## Summary

Implement the Checkbox Component for Atlas UI.

The Checkbox Component provides a reusable control for representing binary and tri-state selections across Atlas Studio. It integrates with the Theme System, UI Context, Accessibility Layer, and Form Components while remaining framework-independent.

The Checkbox Component should expose a stable public API suitable for forms, settings panels, dialogs, toolbars, and plugin interfaces.

---

# Goal

Provide a reusable, accessible, and strongly typed Checkbox component that supports binary and indeterminate selection states.

---

# Background

Checkboxes are widely used throughout Atlas Studio for configuration, preferences, filters, and multi-selection interfaces.

Rather than creating multiple implementations, Atlas UI should provide a single extensible Checkbox abstraction.

# Responsibilities

The Checkbox Component is responsible for:

- Representing selection state.
- Managing checked transitions.
- Supporting indeterminate state.
- Integrating with Theme System.
- Providing accessibility metadata.

The Checkbox Component is **not** responsible for:

- Form submission.
- Business validation.
- Checkbox grouping.
- Tree management.
- Application state.

---

# Supported States

The component should support:

- Checked
- Unchecked
- Indeterminate
- Hover
- Focus
- Active
- Disabled

---

# Public API

Example:

```ts
interface Checkbox extends UIComponent {
  readonly props: CheckboxProps;

  checked(): boolean;

  indeterminate(): boolean;

  toggle(): void;

  check(): void;

  uncheck(): void;
}
```

Example props:

```ts
interface CheckboxProps {
  readonly label?: string;

  readonly checked?: boolean;

  readonly indeterminate?: boolean;

  readonly disabled?: boolean;
}
```

# Design Principles

The Checkbox should be:

- Accessible
- Theme-aware
- Framework independent
- Strongly typed
- Extensible

Checkbox behavior should remain identical across every rendering adapter.

# Technical Requirements

The implementation must:
