# TASK-0031 — Implement Breadcrumb Navigation

## Summary

Implement the Breadcrumb Navigation component for Atlas UI.

The Breadcrumb Navigation provides hierarchical navigation for documents, editor structures, workspaces, and plugin-defined resources. It integrates with the Theme System, Navigation Services, Accessibility Layer, and UI Context while remaining framework-independent.

The Breadcrumb component is responsible only for presenting navigation paths. Navigation logic and route resolution remain outside this component.

---

# Goal

Provide a reusable, accessible, and extensible Breadcrumb Navigation component capable of representing hierarchical structures through a stable public API.

---

# Background

Breadcrumb navigation helps users understand their current location within complex hierarchies.

Typical examples include:

- Workspace hierarchy
- Folder hierarchy
- Document outline
- Nested editor blocks
- Plugin resources
- Asset browsers

Atlas UI should expose a single Breadcrumb abstraction reusable throughout the application.

# Responsibilities

The Breadcrumb Navigation is responsible for:

- Displaying hierarchical paths.
- Organizing breadcrumb items.
- Supporting overflow behavior.
- Integrating with Theme System.
- Providing accessibility metadata.

The Breadcrumb Navigation is **not** responsible for:

- Resolving routes.
- Managing navigation state.
- Loading resources.
- Updating documents.
- Business logic.

# Public API

Example:

```ts
interface Breadcrumb extends UIComponent {
  readonly props: BreadcrumbProps;

  items(): readonly BreadcrumbItem[];

  refresh(): void;
```

Example item:

```ts
interface BreadcrumbItem {
  readonly id: string;

  readonly label: string;

  readonly icon?: UIElement;
```

# Design Principles

The Breadcrumb Navigation should be:

- Lightweight
- Accessible
- Theme-aware
- Framework independent
- Strongly typed

Breadcrumb items should be provided externally rather than generated internally.

---

# Technical Requirements

The implementation must:

- Support dynamic updates.
- Support overflow collapsing.
- Preserve immutable item definitions.
- Avoid hard-coded styling.
- Preserve API stability.

---

# Accessibility Requirements

The Breadcrumb Navigation must:

- Expose semantic navigation role.
- Support keyboard navigation.
- Support screen readers.
- Respect accessibility settings.
- Preserve logical navigation order.
