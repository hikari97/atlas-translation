# TASK-0011 — Implement Menu Component

## Summary

Implement the Menu Component for Atlas UI.

The Menu Component provides a reusable hierarchical navigation and action system for Atlas Studio. It supports nested menus, separators, shortcuts, icons, disabled items, and accessibility while remaining independent from any UI framework.

The Menu Component should serve as the foundation for application menus, context menus, command menus, and plugin-contributed menus.

---

# Goal

Provide a reusable, extensible, and accessible Menu component capable of representing complex menu hierarchies through a stable public API.

---

# Background

Menus appear throughout Atlas Studio in various forms, including:

- Main Menu
- Context Menu
- Toolbar Menu
- Dropdown Menu
- Plugin Menu

Rather than implementing each independently, Atlas UI should expose one composable Menu abstraction.

# Responsibilities

The Menu Component is responsible for:

- Displaying hierarchical actions.
- Managing menu navigation.
- Supporting nested menus.
- Displaying keyboard shortcuts.
- Supporting icons.
- Providing accessibility metadata.

The Menu Component is **not** responsible for:

- Executing commands.
- Overlay positioning.
- Application navigation.
- Plugin discovery.
- Business logic.

---

# Supported Elements

The component should support:

- Menu Item
- Menu Group
- Separator
- Nested Menu
- Disabled Item
- Checkable Item
- Radio Item

Future menu elements should remain backward compatible.

---

# Supported States

The component should support:

- Open
- Closed
- Hover
- Focus
- Active
- Disabled

---

# Public API

Example:

```ts
interface Menu extends UIComponent {
  readonly props: MenuProps;

  open(): void;

  close(): void;

  items(): readonly MenuItem[];
}
```

Example menu item:

```ts
interface MenuItem {
  readonly id: string;

  readonly label: string;

  readonly icon?: UIElement;

  readonly shortcut?: string;

  readonly disabled?: boolean;

  readonly children?: readonly MenuItem[];
}
```

---

# Design Principles

The Menu Component should be:

- Accessible
- Theme-aware
- Framework independent
- Strongly typed
- Extensible

Menus should describe available actions without executing them directly.

---

# Technical Requirements

The implementation must:

- Support nested menus.
- Support immutable menu definitions.
- Preserve menu ordering.
- Avoid hard-coded styling.
- Preserve API stability.

---

# Accessibility Requirements

The Menu Component must:

- Support keyboard navigation.
- Support arrow-key traversal.
- Expose semantic menu roles.
- Announce focused items.
- Support associated shortcuts.
- Respect disabled items.
