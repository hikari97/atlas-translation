# TASK-0017 — Implement Popover Component

## Summary

Implement the Popover Component for Atlas UI.

The Popover Component provides a reusable floating container for displaying contextual and interactive content that is anchored to another UI element. It integrates with the Theme System, Overlay System, Accessibility Layer, and UI Context while remaining framework-independent.

Unlike Tooltip, Popover may contain interactive elements such as buttons, inputs, menus, and forms.

---

# Goal

Provide a reusable, accessible, and extensible Popover component suitable for interactive contextual interfaces.

---

# Background

Popovers are widely used throughout Atlas Studio for:

- Color Picker
- Emoji Picker
- Date Picker
- User Profile Menu
- Quick Settings
- Inline Editor Controls
- Plugin Panels

Rather than implementing separate floating panels, Atlas UI should expose a unified Popover abstraction.

# Responsibilities

The Popover Component is responsible for:

- Displaying contextual interactive content.
- Managing popover visibility.
- Supporting keyboard interaction.
- Managing focus within the popover.
- Integrating with Theme System.
- Providing accessibility metadata.

The Popover Component is **not** responsible for:

- Overlay positioning engine.
- Command execution.
- Business logic.
- Application navigation.
- Animation implementation.

---

# Supported Placements

The component should support:

- Top
- Bottom

- Top Start
- Top End
- Bottom Start
- Bottom End
- Auto

# Supported States

The component should support:

# Public API

Example:

```ts
interface Popover extends UIComponent {
  readonly props: PopoverProps;

  open(): void;

  close(): void;

  toggle(): void;

  isOpen(): boolean;
}
```

Example props:

```ts
interface PopoverProps {
  readonly trigger: UIElement;

  readonly content: UIElement;

  readonly placement: PopoverPlacement;

  readonly offset?: number;

  readonly persistent?: boolean;
}
```

# Design Principles

The Popover Component should be:

- Accessible
- Theme-aware
- Framework independent
- Strongly typed
- Composable

Popover content should support arbitrary UI components.

---

# Technical Requirements

The implementation must:

- Support focus management.
- Support configurable placement.
- Preserve immutable props.
- Avoid hard-coded styling.
- Preserve API stability.

---

# Accessibility Requirements

The Popover must:

- Associate with its trigger.
- Support keyboard navigation.
- Restore focus when closed.
- Support Escape key.
- Respect screen readers.
- Support interactive content.
