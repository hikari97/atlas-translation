# TASK-0026 — Implement Drag Handle

## Summary

Implement the Drag Handle component for Atlas UI.

The Drag Handle provides a reusable interaction component that enables users to initiate drag operations on document blocks and editor objects. It integrates with the Editor View, Interaction Layer, Theme System, Accessibility Layer, and Drag & Drop infrastructure while remaining framework-independent.

The Drag Handle is responsible only for initiating drag interactions. It must not perform drag operations or modify document state.

---

# Goal

Provide a reusable, accessible, and extensible Drag Handle component capable of initiating drag operations through a stable public API.

---

# Background

Modern editors commonly provide drag handles for block manipulation.

Typical scenarios include:

- Reordering blocks
- Moving images
- Moving tables
- Moving embedded widgets
- Dragging plugin blocks

Atlas UI should expose a generic Drag Handle abstraction reusable across all draggable editor objects.

# Responsibilities

The Drag Handle is responsible for:

- Displaying drag affordances.
- Receiving drag initiation.
- Tracking hover state.
- Integrating with Theme System.
- Providing accessibility metadata.

The Drag Handle is **not** responsible for:

- Moving blocks.
- Computing drop targets.
- Rendering drop indicators.
- Editing documents.
- Business logic.
