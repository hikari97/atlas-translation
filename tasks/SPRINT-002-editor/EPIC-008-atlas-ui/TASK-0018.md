# TASK-0018 — Implement Context Menu

## Summary

Implement the Context Menu Component for Atlas UI.

The Context Menu provides contextual actions associated with a specific UI element, document region, or editor object. It integrates with the Menu Component, Overlay System, Command System, Accessibility Layer, and UI Context while remaining framework-independent.

The Context Menu should reuse the Menu abstraction rather than introducing a separate menu implementation.

---

# Goal

Provide a reusable, accessible, and extensible Context Menu component capable of displaying context-sensitive actions through a stable public API.

---

# Background

Context menus are used extensively throughout Atlas Studio for:

- Editor Blocks
- Files
- Folders
- Tabs
- Canvas Objects
- Selections
- Plugin Extensions

Rather than implementing multiple context menu variants, Atlas UI should expose one composable Context Menu abstraction.

# Responsibilities

The Context Menu is responsible for:

- Displaying contextual actions.
- Hosting a Menu instance.
- Resolving context-specific menu entries.
- Managing visibility.
- Supporting accessibility.

The Context Menu is **not** responsible for:

- Implementing Menu behavior.
- Executing commands.
- Managing overlay positioning.
- Business logic.
- Plugin lifecycle.

# Design Principles

The Context Menu should be:

- Accessible
- Theme-aware
- Framework independent
- Strongly typed
- Composable

The Context Menu should reuse the Menu Component instead of redefining menu behavior.

---

# Technical Requirements

The implementation must:

- Reuse Menu abstractions.
- Support dynamic menu generation.
- Preserve immutable menu definitions.
- Avoid hard-coded styling.
- Preserve API stability.

---

# Accessibility Requirements

The Context Menu must:

- Support keyboard invocation.
- Support pointer invocation.
- Expose semantic menu role.
- Restore focus when closed.
- Respect disabled state.
- Support screen readers.
