# TASK-0012 — Implement Toolbar Component

## Summary

Implement the Toolbar Component for Atlas UI.

The Toolbar Component provides a reusable container for organizing commands, actions, menus, and editor tools. It integrates with the Theme System, UI Context, Accessibility Layer, and Command System while remaining independent from any rendering framework.

The Toolbar should act purely as a presentation layer and must not execute application logic directly.

---

# Goal

Provide a reusable, configurable, and accessible Toolbar component that supports command-oriented user interfaces across Atlas Studio.

---

# Background

Toolbars are one of the primary interaction surfaces within Atlas Studio.

They are used in:

- Main Editor
- Floating Toolbars
- Block Toolbars
- Property Panels
- Plugin Toolbars

A unified Toolbar abstraction ensures consistent layout and interaction across the application.

# Responsibilities

The Toolbar Component is responsible for:

- Organizing UI actions.
- Managing toolbar layout.
- Supporting grouped actions.
- Providing overflow behavior.
- Supporting accessibility.
- Integrating with Theme System.

The Toolbar Component is **not** responsible for:

- Executing commands.
- Editor state.
- Plugin management.
- Layout engine.
- Business logic.

---

# Supported Elements

The component should support:

- Toolbar Button
- Toolbar Menu
- Toolbar Separator
- Toolbar Group
- Flexible Spacer
- Overflow Menu

Future toolbar elements should remain backward compatible.

```ts
interface ToolbarItem {
  readonly id: string;

  readonly commandId: string;

  readonly icon?: UIElement;

  readonly label?: string;

  readonly disabled?: boolean;
}
```

---

# Design Principles

The Toolbar should be:

- Accessible
- Theme-aware
- Framework independent
- Strongly typed
- Extensible

Toolbar items should reference commands rather than implementing behavior directly.

---

# Technical Requirements

The implementation must:

- Support grouped items.
- Support overflow handling.
- Preserve immutable item definitions.
- Avoid hard-coded styling.
- Preserve API stability.

---

# Accessibility Requirements

The Toolbar must:

- Support keyboard navigation.
- Expose semantic toolbar role.
- Support arrow-key traversal.
- Respect disabled items.
- Preserve logical focus order.
