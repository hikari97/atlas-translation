# TASK-0006 — Implement Button Component

## Summary

Implement the Button Component for Atlas UI.

The Button is the fundamental interactive component used throughout Atlas Studio. It provides a consistent interface for user actions while integrating with the Theme System, Accessibility Layer, Command System, and UI Context.

The Button implementation must remain framework-independent and expose only public contracts.

---

# Goal

Provide a reusable, accessible, and strongly typed Button component that serves as the foundation for interactive UI elements.

---

# Background

Nearly every application workflow requires buttons.

Rather than creating multiple button implementations, Atlas UI provides a single configurable Button abstraction that supports multiple visual variants and interaction states.

# Responsibilities

The Button Component is responsible for:

- Rendering button content.
- Managing interaction states.
- Supporting icons.
- Supporting accessibility attributes.
- Integrating with Theme System.

The Button Component is **not** responsible for:

- Command execution.
- Navigation.
- Business logic.
- State management.
- Layout management.

---

The component should support:

# Public API

Example:

```ts
interface Button extends UIComponent {
  readonly props: ButtonProps;

  click(): void;

  focus(): void;
}
```

Example props:

```ts
interface ButtonProps {
  readonly label: string;

  readonly variant: ButtonVariant;

  readonly size: ButtonSize;

  readonly disabled?: boolean;

  readonly loading?: boolean;

  readonly icon?: UIElement;
}
```
