# TASK-0009 — Implement Radio Component

## Summary

Implement the Radio Component for Atlas UI.

The Radio Component provides a reusable mechanism for selecting a single option from a predefined group. It integrates with the Theme System, UI Context, Accessibility Layer, and Form Components while remaining framework-independent.

Radio components should always operate within a Radio Group, ensuring only one option is selected at any given time.

---

# Goal

Provide an accessible, reusable, and strongly typed Radio Component with deterministic single-selection behavior.

---

# Background

Radio buttons are commonly used throughout Atlas Studio for mutually exclusive selections such as settings, preferences, editor modes, and configuration panels.

Instead of implementing selection logic in every feature, Atlas UI should provide a unified Radio abstraction.

# Responsibilities

The Radio Component is responsible for:

- Representing a selectable option.
- Managing selected state.
- Participating in a Radio Group.
- Supporting accessibility.
- Integrating with Theme System.

The Radio Component is **not** responsible for:

- Form submission.
- Business validation.
- Multi-selection.
- Application state.
- Navigation.

---

# Supported States

The component should support:

- Selected
- Unselected
- Hover
- Focus
- Active
- Disabled

---

# Public API

Example:

```ts
interface Radio extends UIComponent {
  readonly props: RadioProps;

  selected(): boolean;

  select(): void;

  deselect(): void;

  group(): RadioGroup;
}
```

Example group:

```ts
interface RadioGroup {
  readonly value: string;

  select(value: string): void;

  clear(): void;
}
```

---

# Design Principles

The Radio Component should be:

- Accessible
- Theme-aware
- Framework independent
- Strongly typed
- Extensible

Only one Radio within the same Radio Group may be selected at a time.

---

# Technical Requirements

The implementation must:

- Support keyboard navigation.
- Support group navigation.
- Preserve immutable props.
- Avoid hard-coded styling.
- Preserve API stability.

---

# Accessibility Requirements

The Radio Component must:

- Support keyboard navigation.
- Expose semantic radio role.
- Announce selected state.
- Respect disabled mode.
- Support associated labels.
- Support arrow-key navigation within a Radio Group.
