# TASK-0025 — Implement Block Toolbar

## Summary

Implement the Block Toolbar component for Atlas UI.

The Block Toolbar provides contextual actions associated with an individual document block, such as paragraphs, headings, images, tables, lists, code blocks, and custom plugin blocks. It integrates with the Editor View, Overlay System, Theme System, Command System, and Block Model while remaining framework-independent.

The Block Toolbar is responsible only for presenting block-specific actions. It must not execute commands or modify document state directly.

---

# Goal

Provide a reusable, accessible, and extensible Block Toolbar capable of presenting block-specific actions through a stable public API.

---

# Background

Modern editors commonly display a toolbar beside the active block.

Typical actions include:

- Insert Block
- Duplicate Block
- Delete Block
- Move Up
- Move Down
- Convert Block
- Plugin Actions

Rather than embedding these actions into every block implementation, Atlas UI should expose one reusable Block Toolbar abstraction.

# Responsibilities

The Block Toolbar is responsible for:

- Displaying block-specific actions.
- Tracking the active block.
- Updating its anchor position.
- Integrating with Theme System.
- Providing accessibility metadata.

The Block Toolbar is **not** responsible for:

- Executing commands.
- Manipulating document blocks.
- Managing overlay positioning.
- Editing documents.
- Business logic.

- Accessible

- Framework independent
