# TASK-0027 — Implement Drop Indicator

## Summary

Implement the Drop Indicator component for Atlas UI.

The Drop Indicator provides visual feedback during drag-and-drop operations by indicating where a dragged object will be inserted or positioned. It integrates with the Interaction Layer, Editor View, Theme System, Accessibility Layer, and Drag & Drop infrastructure while remaining framework-independent.

The Drop Indicator is responsible only for visual feedback. It must not determine drop targets or perform document mutations.

---

# Goal

Provide a reusable, accessible, and renderer-independent Drop Indicator component capable of representing insertion targets consistently across Atlas Studio.

---

# Background

Drop indicators are essential for drag-and-drop interactions.

Typical scenarios include:

- Block insertion
- Block reordering
- Table row insertion
- Table column insertion
- Image positioning
- Plugin-defined insertion points

Atlas UI should expose a single Drop Indicator abstraction reusable by all interaction systems.

# Responsibilities

The Drop Indicator is responsible for:

- Displaying insertion feedback.
- Updating indicator position.
- Supporting different indicator styles.
- Integrating with Theme System.
- Providing accessibility metadata.

The Drop Indicator is **not** responsible for:

- Determining insertion positions.
- Handling drag operations.
- Editing documents.
- Business logic.
- Overlay positioning.
