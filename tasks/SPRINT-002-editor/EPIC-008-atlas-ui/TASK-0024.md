# TASK-0024 — Implement Floating Toolbar

## Summary

Implement the Floating Toolbar component for Atlas UI.

The Floating Toolbar provides contextual editing actions that appear near the current selection or active editor element. It integrates with the Editor View, Selection UI, Caret Component, Overlay System, Theme System, and Command System while remaining framework-independent.

The Floating Toolbar is responsible only for presenting contextual actions. It must not contain editing logic or execute commands directly.

---

# Goal

Provide a reusable, accessible, and extensible Floating Toolbar capable of presenting context-aware editing actions through a stable public API.

---

# Background

Floating toolbars are a common interaction pattern in modern editors.

They are typically used for:

- Text formatting
- Link editing
- Inline commands
- Media actions
- Table actions
- Plugin-provided actions

The toolbar should appear only when relevant and follow the active selection.

# Responsibilities

The Floating Toolbar is responsible for:

- Displaying contextual editing actions.
- Tracking the active selection.
- Updating its anchor position.
- Integrating with Theme System.
- Providing accessibility metadata.

The Floating Toolbar is **not** responsible for:

- Executing commands.
- Managing selections.
- Overlay positioning.
- Editing documents.
- Business logic.

# Design Principles

The Floating Toolbar should be:

- Context-aware
- Accessible
- Theme-aware
- Framework independent
- Strongly typed
