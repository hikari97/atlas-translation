# TASK-0004 — Implement Render Context

## Summary

Implement the Render Context for Atlas Studio.

The Render Context provides the execution environment for the rendering pipeline. It exposes the resources, services, configuration, and runtime information required during rendering while isolating renderer implementations from the underlying Runtime.

The Render Context serves as the primary communication boundary between the Render Engine and the rest of Atlas Studio.

---

# Goal

Provide a stable, strongly typed, and platform-independent context that contains all information required during a rendering session.

---

# Background

A renderer should not communicate directly with Runtime, Plugin Manager, or other internal systems.

Instead, every rendering operation receives a Render Context containing the required rendering resources.

This keeps renderer implementations portable and easy to test.

# Responsibilities

Render Context is responsible for providing:

- Render configuration
- Current render session
- Runtime services
- Render cache access
- Environment information

Render Context is **not** responsible for:

- Executing rendering
- Scheduling rendering
- Building Render Tree
- Applying patches
- Managing runtime lifecycle

---

# Suggested Structure

Example:

```ts
export interface RenderContext {
  readonly session: RenderSession;

  readonly environment: RenderEnvironment;

  readonly configuration: RendererConfiguration;

  readonly cache: RenderCache;

  getService<T>(token: ServiceIdentifier<T>): T;
}
```

---

# Design Principles

Render Context should be:

- Immutable
- Strongly typed
- Lightweight
- Platform independent
- Easy to extend

The context should expose only the services required during rendering.

---

# Technical Requirements

The implementation must:

- Avoid runtime implementation details.
- Support dependency injection through abstractions.
- Preserve backward compatibility.
- Support future rendering backends.
- Maintain public API stability.
