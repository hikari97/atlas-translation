# TASK-0005 — Implement Renderer Builder

## Summary

Implement the Renderer Builder for Atlas Studio.

The Renderer Builder provides a fluent API for constructing immutable renderer definitions before they are registered within the Atlas Runtime.

It simplifies renderer configuration while ensuring consistency, validation, and extensibility.

---

# Goal

Provide a strongly typed builder that enables developers to construct renderer definitions through a fluent, readable, and predictable API.

---

# Background

Renderer definitions are expected to grow over time as Atlas Studio supports additional rendering targets, capabilities, and optimization options.

Constructing renderer definitions manually can become error-prone.

The Renderer Builder centralizes construction, validation, and default initialization.

# Responsibilities

Renderer Builder is responsible for:

- Building renderer definitions.
- Validating required properties.
- Producing immutable renderer definitions.
- Providing fluent configuration APIs.

Renderer Builder is **not** responsible for:

- Rendering documents.
- Registering renderers.
- Scheduling rendering.
- Runtime lifecycle.

---

# Builder Flow

```text
RendererBuilder

↓

Metadata

↓

↓

↓

RendererDefinition
```

---

# Suggested API

Example:

```ts
const renderer = RendererBuilder.create('atlas.html')
  .name('HTML Renderer')
  .version('1.0.0')
  .target('html')
  .capability('incremental-render')
  .build();
```

The final API should remain fluent, immutable, and strongly typed.

---

# Design Principles

Renderer Builder should be:

- Fluent
- Immutable
- Predictable
- Strongly typed
- Easy to extend

Construction should fail fast whenever required information is missing.

---

# Technical Requirements

The implementation must:

- Support method chaining.
- Validate required fields.
- Produce immutable definitions.
- Avoid runtime logic.
- Preserve public API stability.
