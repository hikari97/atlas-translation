# TASK-0013 — Implement Diff Engine

## Summary

Implement the Diff Engine for Atlas Studio.

The Diff Engine compares two Render Trees and produces a minimal set of differences required to transform the previous tree into the new tree.

This component enables incremental rendering by ensuring only modified portions of the Render Tree are processed by the Patch Engine.

---

# Goal

Provide a deterministic, efficient, and backend-independent mechanism for detecting structural and property changes between Render Trees.

---

# Background

Rebuilding the entire rendered output after every document change is inefficient.

Instead, Atlas Renderer compares the previous Render Tree with the newly generated Render Tree and identifies only the changes required to update the rendered result.

The Diff Engine is a core component of incremental rendering.

# Responsibilities

Diff Engine is responsible for:

- Comparing Render Trees.
- Detecting node additions.
- Detecting node removals.
- Detecting node updates.
- Detecting reordered nodes.
- Producing immutable diff results.

Diff Engine is **not** responsible for:

- Applying changes.
- Rendering.
- Scheduling.
- Runtime lifecycle.
- Backend synchronization.

---

# Diff Flow

```text
Previous Render Tree

↓

Current Render Tree

↓

Diff Engine

↓

Diff Result

↓

Patch Engine
```

---

# Supported Operations

The Diff Engine should detect:

- Create Node
- Remove Node
- Replace Node
- Update Attributes
- Update Properties
- Move Node

Future operations should remain backward compatible.

---

# Public API

Example:

```ts
interface DiffEngine {
  compare(previous: RenderTree, current: RenderTree): DiffResult;
}
```

---

# Design Principles

The Diff Engine should be:

- Deterministic
- Stateless
- Strongly typed
- Backend independent
- Easy to optimize

The same input trees must always produce the same Diff Result.

---

# Technical Requirements

The implementation must:

- Preserve node ordering.
- Avoid duplicate operations.
- Produce immutable diff results.
- Support future optimization strategies.
- Preserve public API stability.
