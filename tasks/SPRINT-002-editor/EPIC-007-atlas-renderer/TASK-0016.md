# TASK-0016 — Integrate Renderer with Runtime

## Summary

Integrate Atlas Renderer with the Atlas Runtime (`atlas-core`).

This task establishes the runtime integration layer that allows Atlas Runtime to initialize, execute, update, and dispose renderer instances through a stable public interface.

The Renderer remains independent from the Runtime implementation and communicates only through public contracts.

---

# Goal

Provide seamless integration between `atlas-renderer` and `atlas-core` while preserving package boundaries and architectural separation.

---

# Background

Previous tasks implemented the Renderer as an isolated subsystem.

This task connects the Renderer to the Runtime so that rendering becomes part of the application lifecycle.

The Runtime orchestrates execution while Atlas Renderer remains responsible for rendering behavior.

---

# Responsibilities

Renderer Runtime Integration is responsible for:

- Registering the Renderer with Runtime.
- Initializing Renderer services.
- Starting rendering lifecycle.
- Stopping rendering lifecycle.
- Providing runtime renderer access.

Renderer Runtime Integration is **not** responsible for:

- Runtime implementation.
- Plugin lifecycle.
- Event dispatching.
- UI rendering.
- Backend rendering.

# Design Principles

Renderer Runtime should be:

- Modular
- Deterministic
- Strongly typed
- Runtime independent
- Easy to extend

The Runtime communicates with the Renderer only through the Renderer Runtime abstraction.

---

# Technical Requirements

The implementation must:

- Use only public APIs from atlas-core.
- Avoid circular dependencies.
- Preserve lifecycle ordering.
- Support asynchronous startup.
- Preserve public API stability.
