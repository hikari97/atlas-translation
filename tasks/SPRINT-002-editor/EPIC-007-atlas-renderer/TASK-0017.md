# TASK-0017 — Integrate Renderer Event System

## Summary

Integrate Atlas Renderer with the Atlas Event System.

This task enables the Renderer to publish rendering events and subscribe to runtime events through the public interfaces exposed by `atlas-events`.

The integration must remain loosely coupled and rely exclusively on the public contracts of the Event System.

---

# Goal

Allow the Renderer to participate in Atlas Studio's event-driven architecture without introducing direct dependencies on event implementation details.

---

# Background

Rendering is influenced by various runtime events, including:

- Document changes
- Plugin updates
- Runtime lifecycle events
- User interactions
- System notifications

Likewise, the Renderer should publish events when significant rendering activities occur.

Using the Event System ensures components remain decoupled while communicating efficiently.

---

# Scope

## Included

- Renderer Event Publisher
- Renderer Event Subscriber
- Render Event Registration
- Render Event Dispatch
- Render Context Integration

---

## Excluded

- Event Bus implementation
- Event persistence
- Distributed events
- Event replay
- Logging

# Responsibilities

Renderer Event Integration is responsible for:

- Publishing rendering events.
- Subscribing to runtime events.
- Registering render event handlers.
- Connecting the Renderer to the Atlas Event System.

Renderer Event Integration is **not** responsible for:

- Event Bus implementation.
- Event storage.
- Event transport.
- Event replay.

---

# Integration Flow

```text
Renderer

↓

```

---

# Event Examples

The integration should support events such as:

- Render Started
- Render Completed
- Render Failed
- Render Cancelled
- Render Updated
- Render Invalidated

Additional event types may be introduced without breaking the public API.

---

# Public API

Example:

```ts
interface RendererEventIntegration {
  publish(event: RenderEvent): Promise<void>;

  subscribe<T extends Event>(
    type: EventType<T>,
    handler: EventHandler<T>,
  ): void;
}
```

---

# Design Principles

Renderer Event Integration should be:

- Event-driven
- Strongly typed
- Loosely coupled
- Extensible
- Backend independent

Renderer components must communicate through events rather than direct component references.

---

# Technical Requirements

The implementation must:

- Use only public APIs from atlas-events.
- Avoid circular dependencies.
- Support asynchronous event publishing.
- Support multiple subscribers.
- Preserve public API stability.
