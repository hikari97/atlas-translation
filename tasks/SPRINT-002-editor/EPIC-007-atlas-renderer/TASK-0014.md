# TASK-0014 — Implement Patch Engine

## Summary

Implement the Patch Engine for Atlas Studio.

The Patch Engine consumes the Diff Result generated by the Diff Engine and applies the required changes to the rendering backend.

Rather than rebuilding the entire rendered output, the Patch Engine performs incremental updates by executing only the required patch operations.

The Patch Engine remains backend-independent by operating on abstract patch operations.

---

# Goal

Provide an efficient and deterministic mechanism for applying Render Tree changes incrementally.

---

# Background

The Diff Engine identifies what changed.

The Patch Engine determines how those changes are applied.

Separating these responsibilities allows the renderer to support multiple rendering backends while keeping the diffing algorithm reusable.

# Responsibilities

Patch Engine is responsible for:

- Executing patch operations.
- Applying incremental updates.
- Validating patch sequences.
- Recording patch statistics.
- Producing patch results.

Patch Engine is **not** responsible for:

- Diff calculation.
- Render scheduling.
- Tree construction.
- Layout calculation.
- Backend-specific rendering logic.

---

# Patch Flow

```text
Render Tree

↓

Diff Result

↓

Patch Engine

↓

Patch Executor

↓

Updated Render Output
```

---

# Supported Patch Operations

The Patch Engine should support:

- Create Node
- Remove Node
- Replace Node
- Move Node
- Update Attributes
- Update Properties

Patch execution order must remain deterministic.

---

# Public API

Example:

```ts
interface PatchEngine {
  apply(diff: DiffResult, context: PatchContext): Promise<PatchResult>;
}
```

---

# Design Principles

Patch Engine should be:

- Deterministic
- Incremental
- Stateless
- Strongly typed
- Backend independent

Patch execution should rely only on abstract operations.

---

# Technical Requirements

The implementation must:

- Execute operations in order.
- Stop on unrecoverable failures.
- Produce immutable PatchResult.
- Record execution statistics.
- Preserve public API stability.
